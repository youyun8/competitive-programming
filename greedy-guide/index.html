0導讀與使用方式

貪心演算法（Greedy Algorithm）：在每一個決策點，只根據「當下看起來最好」的準則做選擇，不回頭、不全域搜尋，期望最終累積成全域最優解。

用決策樹的視角比較三種範式：

  • 暴力搜尋：走遍整棵決策樹 — 永遠正確，但通常指數時間。
  • 動態規劃：發現大量子樹等價（重疊子問題），把樹壓縮成 DAG — 多項式時間。
  • 貪心：證明「在每個節點只需走一條特定的邊」— 整棵樹退化成一條鏈，往往 O(n log n)（排序主導）甚至 O(n)。
⚠ 貪心最大的風險

它「看起來對」的機率遠高於「真的對」的機率。競賽中錯誤貪心是 WA 的最大來源之一。因此本指南把正確性證明與反例構造放在與策略同等重要的位置 — 見第 1、3、4 章。

1理論基礎：什麼時候貪心是對的

一個問題能用貪心正確求解，教科書上的標準條件是以下兩個性質同時成立：

1.1 貪心選擇性質（Greedy Choice Property）

存在一個全域最優解包含第一步的貪心選擇。換句話說：不管最優解長什麼樣，我們總能把它「改造」成第一步就採用貪心選擇、而且不變差的解。這個「改造」正是交換論證的核心動作。

1.2 最優子結構（Optimal Substructure）

做完第一步貪心選擇後，剩下的子問題的最優解，拼上這一步，就是原問題的最優解。DP 也需要這條；貪心額外需要的是上面那條「第一步就能貪」。

💡 實戰翻譯

競賽裡你不會真的寫證明，但你腦中應該完成這個 30 秒檢查：「假設神給我一個最優解，它第一步和我的貪心不一樣 — 我能不能把它換成我的選擇而不變差？」若你能在腦中完成這個交換且理由紮實，貪心大概率是對的；若你發現交換會牽動後面一連串東西且理不清，請立刻警戒（考慮 DP 或反悔貪心）。

1.3 四大證明技巧

① 交換論證（Exchange Argument）— 使用率 80%

假設最優解 OPT 與貪心解 G 不同，找到第一個不同處，把 OPT 中的元素換成 G 的選擇，證明交換後仍是可行解且目標值不變差。重複交換，OPT 被逐步改造成 G，故 G 最優。排序類貪心則常用相鄰交換版本：證明任意相鄰逆序對交換後不變差 ⇒ 排序後最優（類似冒泡排序的正確性）。完整五步驟模板見 第 4 章。

② 領先論證（Greedy Stays Ahead）

用歸納法證明：貪心解在每一步都「不落後」於任何解 — 例如活動選擇中，貪心選的第 i 個活動結束時間 ≤ 任何解的第 i 個活動結束時間。步步領先 ⇒ 最終不劣。

③ 反證法與極端原理

假設貪心不是最優，取「第一個出錯的位置」或「差異最小的最優解」等極端對象，推出矛盾。常與前兩者混用。

④ 擬陣（Matroid）— 專家級

若問題的可行解族構成擬陣（遺傳性 + 交換公理），則按權重由大到小、能加就加的貪心必然得到最大權基（Rado–Edmonds 定理）。Kruskal 正確性的最深刻解釋。詳見 2.11。

1.4 如何「懷疑」一個貪心：反例構造的系統方法

  • 小資料窮舉對拍：寫一個 O(2^n) 或 O(n!) 暴力，隨機生成 n ≤ 8 的小測資對拍 200 次 — 5 分鐘寫完，能擋掉 95% 的錯誤貪心。模板見 4.3。
  • 攻擊排序鍵：構造兩個元素，讓你的排序鍵認為 A 優於 B，但實際上先選 B 更好（利用「後效性」— A 的選擇影響了未來）。
  • 攻擊邊界：全部相等、極大極小混搭、恰好卡滿容量。
  • 經典警訊：目標函數含權重（帶權區間排程）、資源不可分割（0/1 背包）、非 canonical 硬幣系統 — 見 第 3 章。

2策略分類

以下 11 個分類按「思維模式」劃分：每類都有固定的核心想法、適用特徵（如何辨識）、證明要點與例題。競賽中辨識題型 → 對應到分類 → 套用該類的思考框架，是最高效的解題路徑。

2.1 排序貪心（Sort & Sweep）

💡 核心想法

找到正確的排序鍵，排序後從頭到尾線性掃描，每一步的決策變得顯然。80% 的貪心題第一步都是排序 — 問題只剩「按什麼排」。

🔎 適用特徵
  • 元素之間的「處理順序」影響答案，但單一元素的貢獻可獨立衡量。
  • 存在一個單調的「性價比」「緊迫度」或「結束時間」可比較。
  • 排序鍵的推導方法：對任兩個相鄰元素寫出「先 A 後 B」與「先 B 後 A」的成本，令前者 ≤ 後者，解出比較式 — 這就是相鄰交換論證。

2.1.1 單鍵排序

★部分背包 Fractional Knapsack排序性價比

物品可切割，容量 W，最大化總價值。

思路

按「單位重量價值 v/w」由大到小排序，依序裝滿，最後一件裝部分。與 0/1 背包的對比是理解貪心邊界的第一課：可分割使得「先拿性價比最高的」永遠不會後悔 — 交換論證中把最優解裡任何一單位重量換成更高性價比的都不變差。0/1 背包不可分割，交換會破壞容量可行性，貪心失效（見第 3 章反例）。

程式碼（C++） 
sort(items.begin(), items.end(), [](auto& a, auto& b){
    return a.v * b.w > b.v * a.w;   // v/w 大者在前，交叉相乘避免浮點
});
double ans = 0, cap = W;
for (auto& [w, v] : items) {
    double take = min(cap, (double)w);
    ans += take * v / w;
    cap -= take;
    if (cap == 0) break;
}
★LC 455 分發餅乾排序雙指針

每個孩子有胃口 g[i]，每塊餅乾大小 s[j]，一塊只能給一人，最多滿足幾個孩子？

思路

兩邊都排序，用最小的能滿足的餅乾去餵胃口最小的孩子（雙指針）。證明（領先論證）：任何最優方案中，把餵給最小孩子的餅乾換成「最小的可行餅乾」，剩餘餅乾集合只會更「寬鬆」，不會變差。

★★POJ 1328 Radar Installation排序區間選點

海島要被 x 軸上的雷達（半徑 d）覆蓋，求最少雷達數。

提示

把「島」轉化成「雷達可以放置的 x 軸區間」— 幾何問題化為區間選點問題。

思路

每個島對應區間 [x−√(d²−y²), x+√(d²−y²)]。問題變成：最少的點刺穿所有區間。按右端點排序，維護目前雷達位置；若下一個區間左端點 > 雷達位置，則在其右端點放新雷達。這是 2.2 區間家族的「區間選點」模型。

2.1.2 自訂比較函數（用交換論證推導 comparator）

★★LC 179 最大數排序交換論證

把一組非負整數串接成最大的數。

思路

比較兩個字串 a、b：若 a+b > b+a（字串串接後字典序）則 a 排前面。正確性：相鄰交換論證 — 任意相鄰逆序對交換後結果變大，故完全排序後最大。陷阱：必須驗證這個比較關係是嚴格弱序（可證其可傳遞，因 a+b>b+a ⟺ 把字串視為 k 進位數後 a/(10^|a|−1) > b/(10^|b|−1)，是一個實數鍵值），否則 std::sort 是 UB。

程式碼（C++） 
sort(strs.begin(), strs.end(), [](const string& a, const string& b){
    return a + b > b + a;
});
string ans = accumulate(strs.begin(), strs.end(), string());
return ans[0] == '0' ? "0" : ans;   // 全零特判
★★★洛谷 P1080 國王遊戲排序交換論證高精度

n 位大臣每人左手 a、右手 b。大臣獲得的金幣 = 排在他前面所有人左手乘積 ÷ 他自己的右手。安排順序使獲得最多金幣的大臣拿的金幣最少。

提示

對相鄰兩位大臣 i、j，分別寫出「i 在前」與「j 在前」時兩人獎金的最大值，比較之。

思路

相鄰交換論證得排序鍵：按 a×b 遞增排序。設前綴乘積為 S，i 在 j 前時兩人的較大值是 max(S/b_i, S·a_i/b_j)；交換後為 max(S/b_j, S·a_j/b_i)。通分比較可得 a_i·b_i ≤ a_j·b_j 時 i 在前不劣。注意本題需高精度乘除。同型題：洛谷 P1842 奶牛玩雜技（按 w+s 排序）、UVa 10026 Shoemaker's Problem。

2.1.3 雙指針配對

★★LC 881 救生艇排序雙指針

每艘船限乘 2 人、限重 limit，最少幾艘船載走所有人？

思路

排序後最重配最輕：若最重 + 最輕 ≤ limit 則同船，否則最重者單獨一船。交換論證：最重的人反正要一艘船，讓他帶走「還能帶的人中任意一個」都不會比帶最輕的更好 — 因為最輕的人與任何人都相容，留下他最保值。

程式碼（C++） 
sort(p.begin(), p.end());
int i = 0, j = p.size() - 1, boats = 0;
while (i <= j) {
    if (p[i] + p[j] <= limit) i++;
    j--; boats++;
}
★★★POJ 2287 田忌賽馬排序雙指針博弈

田忌與齊王各 n 匹馬，贏一場 +200、輸 −200、平 0，田忌知道齊王出馬順序（由強到弱），求最大收益。

思路

雙方馬按速度排序後貪心：若田忌目前最快 > 齊王最快，直接贏；若最慢 > 齊王最慢，贏下這場；否則用最慢的馬去消耗齊王最快的馬（打不贏就墊）。正確性用交換論證逐 case 驗證，或退而用區間 DP O(n²) 保底 — 這題是「貪心可行但 case 多，DP 更穩」的典型例子。

2.2 區間問題全家桶（Interval Scheduling Family）

💡 核心想法

區間貪心的一切糾結都在「按左端點還是右端點排」。記住對應關係：選最多不重疊 → 右端點；完全覆蓋 → 左端點；算最少資源 → 掃描線/小根堆。

🔎 適用特徵
  • 輸入是一堆 [l, r]，或能轉化成區間（如 POJ 1328 的幾何轉化、「站點+半徑」轉化）。
  • 目標是「最多選幾個 / 最少用幾個 / 最少資源數 / 能否覆蓋」。
  • ⚠ 若區間帶權且目標是權和 — 貪心失效，轉 DP（見 2.2.5）。

2.2.1 不重疊區間最多選幾個（活動選擇）— 按右端點排

★★LC 435 無重疊區間區間右端點排序

移除最少的區間，使其餘互不重疊 ⟺ 選出最多互不重疊的區間。

思路

按右端點遞增排序，能選就選（左端點 ≥ 上一個選中的右端點）。證明用領先論證：貪心第 k 個選中區間的右端點 ≤ 任意解第 k 個的右端點（歸納），所以任何解能塞下的，貪心都塞得下。為什麼不是左端點：一個很早開始但很長的區間會堵死後面 — 反例 [1,100],[2,3],[4,5]。

程式碼（C++） 
sort(iv.begin(), iv.end(), [](auto& a, auto& b){ return a[1] < b[1]; });
int keep = 0; long long lastEnd = LLONG_MIN;
for (auto& v : iv)
    if (v[0] >= lastEnd) { keep++; lastEnd = v[1]; }
return iv.size() - keep;

2.2.2 完全覆蓋最少段數 — 按左端點排 + 貪心延伸

★★★LC 1024 視頻拼接區間左端點排序覆蓋

用最少段影片覆蓋 [0, T]。同型：LC 45 跳躍遊戲 II、LC 1326 灌溉花園。

思路

按左端點排序。維護目前已覆蓋到 cur；在所有左端點 ≤ cur 的區間中，選右端點最遠的一段接上（這一步是貪心選擇性質：反正都要有一段跨過 cur，選伸最遠的不會變差），cur 跳到該右端點。跳躍遊戲 II 是同一個模型的陣列形式，可 O(n) 一次掃描。

程式碼（C++，跳躍遊戲 II 形式） 
int jumps = 0, curEnd = 0, farthest = 0;
for (int i = 0; i + 1 < n; i++) {
    farthest = max(farthest, i + nums[i]);
    if (i == curEnd) { jumps++; curEnd = farthest; }  // 必須起跳
}

2.2.3 最少資源數（區間分組）— 掃描線 / 小根堆

★★★LC 253 會議室 II區間堆掃描線

最少需要幾間會議室容納所有會議？同型：洛谷 P2887、CF 手機基地台類。

思路

兩種等價視角：掃描線 — 把開始 +1、結束 −1 排成事件，答案是前綴和最大值（= 最大同時重疊數，這是下界，也構造性地可達）；小根堆 — 按開始時間排序，堆存每間房目前的結束時間，新會議若能接最早空出的房就接，否則開新房。

程式碼（C++，小根堆） 
sort(iv.begin(), iv.end());
priority_queue<int, vector<int>, greater<>> rooms; // 各房結束時間
for (auto& [s, e] : iv) {
    if (!rooms.empty() && rooms.top() <= s) rooms.pop();
    rooms.push(e);
}
return rooms.size();

2.2.4 區間選點 / 最少箭射氣球

★★LC 452 用最少數量的箭引爆氣球區間右端點排序
思路

「最少的點使每個區間至少含一點」— 與活動選擇對偶。按右端點排序，箭射在目前區間的右端點（能捅穿的最晚位置，捅到最多後續區間），左端點超過箭位置就補新箭。答案 = 最多不相交區間數（LP 對偶的離散體現）。

2.2.5 邊界討論：帶權區間 → 貪心失效 → DP

當每個區間帶權 w、目標最大化選中權和：交換論證失敗 — 「結束早」與「權大」不可比。此時按右端點排序後做 dp[i] = max(dp[i−1], w_i + dp[prev(i)])（prev 用二分找），O(n log n)。辨識訓練：看到區間題先問「有沒有權重」，這一問決定貪心/DP 的分岔。

2.3 交換論證排程（Scheduling by Exchange Argument）

💡 核心想法

單機/多機排程問題的統一解法：假設最優序列中相鄰兩個任務 i、j 交換，寫出兩種順序的目標值，令「i 在前不劣」解出不等式 — 這個不等式就是排序鍵。

🧾 推導模板

設 i 在 j 前、其餘不動。因為只有 i、j 的完成時刻改變，比較兩種順序下這兩項的目標貢獻即可。得到形如 f(i,j) ≤ f(j,i) 的可傳遞關係後，直接 sort。務必檢查關係可傳遞（通常能改寫成單一實數鍵值比較，如 p/w、min(a,b) 分類）。

★★★最小化加權完成時間（Smith's Rule）排程交換論證

單機 n 個任務，任務 i 耗時 p_i、權 w_i，最小化 Σ w_i × C_i（C_i 為完成時刻）。對應：洛谷 P1248 加工生產調度前置、AtCoder 各 ABC 排程題。

推導

相鄰交換 i、j：i 在前時兩者貢獻差為 w_j·p_i − w_i·p_j。令 i 在前不劣 ⇒ p_i/w_i ≤ p_j/w_j。所以按 p/w 遞增（性價比：短且重要優先）排序。特例 w 全等 → SJF 最短工時優先；p 全等 → 權大優先。

★★★★洛谷 P1248 加工生產調度（Johnson's Rule）排程交換論證

每個工件先過機器 A（耗時 a_i）再過機器 B（耗時 b_i），兩台機器各自序列處理，最小化總完工時間（makespan）。

思路

Johnson's Rule：min(a_i, b_j) ≤ min(a_j, b_i) 時 i 排 j 前。實作：把工件分兩組 — a < b 的按 a 遞增排在前，a ≥ b 的按 b 遞減排在後。直覺：A 快 B 慢的先上（讓 B 早點有活幹且不斷流），A 慢 B 快的殿後（結尾時 B 不用等）。這是三機以上 NP-hard、兩機卻有優美貪心的經典分界。

★★★★AtCoder DP Contest X - Tower排程交換論證貪心+DP

每個箱子有重量 w、承重 s、價值 v，疊塔使塔上每箱之上的總重 ≤ 其承重，最大化總價值。

提示

價值使純貪心失效（要選子集）— 但「選定子集後的最佳疊放順序」可以先用交換論證定死，再對這個順序做背包 DP。

思路

相鄰交換：i 疊在 j 下方可行性更寬 ⟺ w_i + s_i ≥ w_j + s_j。按 w+s 遞增排序（小的在上/先考慮），然後 dp[承重餘量] = 最大價值 做 0/1 背包。這是「交換論證定順序 + DP 選子集」的代表套路，競賽出現率極高（同型：CF 開會安排、洛谷 P1417 烹調方案）。

2.4 反悔貪心（Regret Greedy）

💡 核心想法

普通貪心不敢貪（怕後悔）的地方，反悔貪心先貪了再說，同時把「撤銷這次選擇的代價」丟進堆裡；未來遇到更好的選項時，彈出堆頂完成撤銷。等價於在時間軸上動態維護「目前最優的選擇集合」。

🔎 適用特徵
  • 依時間/期限順序處理，選擇之間互相排擠（容量、期限、次數限制）。
  • 「不選白不選、選錯可退貨」— 撤銷一個已選項不影響其他已選項的可行性。
  • 直接貪心能構造出反例、DP 狀態又太大時，優先想反悔堆。
★★★LC 630 課程表 III反悔貪心堆

第 i 門課耗時 d_i、截止 e_i，從時刻 0 開始連續上課，最多修幾門？

思路

按截止時間遞增處理，維護大根堆（已選課程的耗時）與目前總耗時 T。每門課先選上（T += d）；若 T 超過截止時間，反悔：彈掉堆中耗時最大的課（撤銷最虧的決策）。正確性：交換論證 — 修課數相同時總耗時越小越好，彈最大者使 T 最小化，為未來保留最大餘裕。

程式碼（C++） 
sort(c.begin(), c.end(), [](auto& a, auto& b){ return a[1] < b[1]; });
priority_queue<int> pq;           // 已選課程的耗時，大根堆
long long T = 0;
for (auto& [d, e] : c) {
    T += d; pq.push(d);
    if (T > e) { T -= pq.top(); pq.pop(); }   // 反悔：踢掉最耗時的
}
return pq.size();
★★★LC 871 最低加油次數反悔貪心堆

路上有加油站，車開到哪算哪，最少加幾次油到終點？

思路

「路過先不加，把油量存入大根堆；油不夠時再反悔 — 取出堆中最大的油量視為當時加過」。次數相同時油越多越好，故每次補堆頂。這是把「在哪加油」的決策延遲到必要時刻的教科書範例。

程式碼（C++） 
priority_queue<int> pq;
long long fuel = startFuel; int cnt = 0, i = 0;
stations.push_back({target, 0});
for (auto& [pos, gas] : stations) {
    while (fuel < pos) {                 // 到不了，反悔補油
        if (pq.empty()) return -1;
        fuel += pq.top(); pq.pop(); cnt++;
    }
    pq.push(gas);
}
return cnt;
★★★★洛谷 P4053 [JSOI2007] 建築搶修反悔貪心堆

n 棟建築，修第 i 棟耗時 t_i、必須在 d_i 前修完，一次修一棟，最多修幾棟？（LC 630 的原型競賽題，USACO 2009 Work Scheduling 為「最大化報酬」變體）

思路

同 LC 630 反悔堆。USACO Work Scheduling 變體：按期限處理、堆存報酬最小的已接工作，時間不夠且新工作報酬更高時換掉堆頂 — 反悔的對象從「耗時」變成「報酬」，體會反悔鍵值隨目標函數變化。

★★★★CF 865D Buy Low Sell High反悔貪心堆費用流思想

已知未來 n 天股價，每天限買一股或賣一股或不動，最大化最終利潤（結束時不持股要求可自動滿足）。

思路

經典反悔交換：每天把 p_i 推入小根堆；若 p_i > 堆頂，利潤 += p_i − 堆頂，彈出堆頂，再把 p_i 推回去一次。第二次推入的 p_i 代表「反悔券」：若未來有更高價 p_j 用它成交，總效果 (p_i − p_x) + (p_j − p_i) = p_j − p_x，等價於當初直接在 x 買 j 賣 — 中間商 p_i 無縫退出。這正是模擬費用流中「殘量網路反向邊」的化身，是通往 2.11.5 的橋樑。

程式碼（C++） 
priority_queue<long long, vector<long long>, greater<>> pq;
long long profit = 0;
for (int i = 0; i < n; i++) {
    long long p; cin >> p;
    if (!pq.empty() && pq.top() < p) {
        profit += p - pq.top(); pq.pop();
        pq.push(p);              // 反悔券：允許未來把這次交易升級
    }
    pq.push(p);
}

2.5 堆積貪心（Priority-Queue Greedy）

💡 核心想法

貪心策略是「每一步取全域最小（或最大）的元素合併/處理」，且處理後會產生新元素回到候選池 — 用 heap 維護候選池，總複雜度 O(n log n)。

★★洛谷 P1090 合併果子（Huffman）堆Huffman

每次合併兩堆果子，代價為兩堆重量和，求合併成一堆的最小總代價。

思路

每次取最小的兩堆合併 — 即 Huffman 樹構造。證明（交換論證）：最小的兩個元素必然可以是最優樹中深度最深的一對兄弟 — 若不是，把它們與最深處的元素交換，總代價（Σ 重量×深度）不增。合併次序對應樹形，貪心構造即最優前綴碼，這也是資訊理論中 Huffman 編碼最優性的來源。

程式碼（C++） 
priority_queue<long long, vector<long long>, greater<>> pq(w.begin(), w.end());
long long cost = 0;
while (pq.size() > 1) {
    long long a = pq.top(); pq.pop();
    long long b = pq.top(); pq.pop();
    cost += a + b; pq.push(a + b);
}
★★★★洛谷 P2168 [NOI2015] 荷馬史詩（k 叉 Huffman）堆Huffman構造

k 進位前綴編碼，最小化總長度，並在最小總長下最小化最長編碼。

思路

k 叉 Huffman：每次合併 k 個最小的。關鍵技巧：若 (n−1) mod (k−1) ≠ 0，先補權值為 0 的虛擬節點湊齊，保證每次都能滿 k 合併（否則根附近會浪費短編碼）。第二關鍵字：堆中同權值時取「樹高最小」者合併，即可同時最小化最深深度。

★★★LC 295 數據流中位數（對頂堆）堆中位數
思路

大根堆存小的一半、小根堆存大的一半，兩堆大小差 ≤ 1，中位數在堆頂。這個「對頂堆」結構是許多貪心題的載體（如 2.7 的中位數貪心需要動態中位數、slope trick 的可合併堆雛形）。滑動視窗版需要延遲刪除或 multiset。

★★★洛谷 P1631 序列合併（k 路歸併）堆多路歸併

兩個長度 n 的遞增序列，n² 個和中最小的 n 個。

思路

堆中放 (a_i + b_1, i, 1)，彈出 (i, j) 時推入 (i, j+1) — 把 n² 個候選組織成 n 條有序鏈做 k 路歸併。同型：LC 373、Super Ugly Number、CF「第 k 小的 XXX」系列。核心辨識：候選集合呈笛卡兒積/指數大，但可以分層有序生成。

2.6 字典序貪心與單調堆疊構造

💡 核心想法

字典序目標自帶貪心性質：第一位越小/大，後面全不用比。所以逐位確定「當前位能放的最好字元」，配一個 O(1)/O(log) 的可行性檢查（剩餘資源夠不夠完成約束）。實作上大量以單調堆疊出現：新字元把棧頂「還有機會再來的較差字元」彈掉。

★★LC 402 移掉 K 位數字字典序單調棧

刪 k 位使剩下的數最小。

思路

維護遞增棧：新數字比棧頂小且還有刪除額度時，彈棧頂（高位變小永遠更優 — 這就是字典序的貪心選擇性質）。掃完還有額度就從尾巴刪。

程式碼（C++） 
string st;
for (char c : num) {
    while (!st.empty() && k && st.back() > c) { st.pop_back(); k--; }
    st.push_back(c);
}
st.resize(st.size() - k);
st.erase(0, st.find_first_not_of('0'));      // 去前導零
return st.empty() ? "0" : st;
★★★LC 316 去除重複字母字典序單調棧可行性檢查

刪去重複字母使每個字母恰出現一次，且結果字典序最小。

思路

單調棧 + 可行性檢查：彈棧頂的條件除了「棧頂 > 新字元」，還必須「棧頂字元在後面還會出現」（否則彈了就湊不齊了）。這是字典序貪心的完整形態：貪 + 保證還能完成。用剩餘計數陣列與 in-stack 標記各一個即可。

★★★★LC 321 拼接最大數字典序單調棧歸併

從兩個陣列中共取 k 個數（各自保持相對順序），拼成最大的數。

思路

三層貪心組合：枚舉從第一個陣列取 i 個 — 各自用單調棧取出「保留 i 個的最大子序列」（LC 402 的最大版）— 再貪心歸併兩個序列（比較整段字典序決定下一個取誰，注意相等時要往後比，故歸併是 O(n²) 最壞）。展示「字典序貪心可作為零件組裝」的層次感。

★★★字典序最小拓撲序字典序圖堆
思路

Kahn 演算法中把普通 queue 換成小根堆：每次在所有入度 0 的點中取編號最小者。正確性：第一位可以是任何目前入度 0 的點，取最小者是字典序的貪心選擇性質；取走後子問題同構。⚠ 對比陷阱：「字典序最小拓撲序」≠「每個點盡量早出現」— HDU 4857 反向建圖才是後者，辨識題意的細微差異是這型題的坑。

2.7 數學貪心與調整法

💡 核心想法

決策由一條不等式或極值性質一錘定音：排序不等式、中位數最小化絕對偏差、均值不等式。證明手法是調整法 — 對任何解做微小擾動，證明往某方向調整必不劣，推到極端即最優。

★★LC 462 最小操作次數使陣列相等 II中位數調整法

每次 ±1，使全部相等的最小步數 = Σ|a_i − x| 的最小值。

思路

x 取中位數。調整法證明：設 x 左側有 L 個點、右側有 R 個，x 右移一單位成本變化 L − R — 只要 L < R 就該右移，平衡點即中位數。倉庫選址、士兵站隊（洛谷 P1889，二維可分離：x、y 各自取中位數，x 座標需先排序後減去下標再取中位數）都是此模型。

★★★★洛谷 P2512 [HAOI2008] 糖果傳遞（環形均分紙牌）中位數前綴和

環上 n 人傳糖果使人人相等，傳一顆代價 1，最小總代價。

思路

設 x_i 為 i 傳給 i−1 的淨流量，列方程消元後答案 = Σ|x_1 − c_i|，其中 c_i 是可預先算出的前綴常數 — 問題化為「數軸上選點最小化絕對值和」，x_1 取 c 的中位數。線性版（均分紙牌，洛谷 P1031）直接前綴和貪心。展示「代數變形 → 化歸為中位數模型」的威力。

★★★硬幣問題：什麼時候找零貪心是對的數學反例意識
思路

「每次拿最大面額」只對 canonical coin system 正確（如 1/5/10/50 與各國流通幣制 — 這是設計出來的），對 {1,3,4} 湊 6 就錯（貪心 4+1+1 三枚，最優 3+3 兩枚）。判定一般幣制是否 canonical 可用 Kozen–Zaks 定理：最小反例必出現在小範圍內，可 O(面額²) 檢查。競賽守則：面額任意給的找零題，一律 DP。

★★LC 343 整數拆分（均值不等式）數學
思路

拆成盡量多的 3（餘 1 時改一個 3+1 → 2+2）。由均值不等式，固定段數時各段相等最優；比較 e 附近的整數，3 的冪勝出。數學貪心常見終局：答案結構被不等式限定成幾種 case，逐一比較。

★★★排序不等式應用：僱傭計畫 / 加權匹配數學排序不等式
思路

排序不等式：兩序列同序點積最大、反序最小。應用如「n 個工人 n 件事，效率×難度求和最小」→ 效率大配難度小（反序）。與 2.1.2 的 comparator 推導殊途同歸，但直接引用不等式可跳過推導。

2.8 圖上貪心

💡 核心想法

MST 與 Dijkstra 本質上都是貪心，正確性由兩條圖論性質保證：Cut Property（跨越任意割的最小邊必在某棵 MST 中）與非負權下最近未定點的距離已收斂。把它們當作貪心來理解，才能在變體題中正確遷移。

★★★Kruskal / Prim 與 Cut & Cycle Property圖MST交換論證
思路

Cut Property 的交換論證：設跨割最小邊 e ∉ MST T，把 e 加入 T 形成環，環上必有另一條跨割邊 f，換掉（w(f) ≥ w(e)）得不劣的生成樹 — 標準 exchange argument。Cycle Property（環上最大邊必不在 MST）是對偶。變體遷移：最小瓶頸生成樹 = MST（Cut Property 直接推）；「最大化路徑最小邊」= 在最大生成樹上走；Kruskal 重構樹處理「≤k 邊權連通性」詢問。

★★★Dijkstra 的貪心正確性邊界圖最短路
思路

歸納不變量：每次從堆中取出的最近未確定點 u，其 dist 已是最短 — 因為任何經過「未確定區」的替代路徑，其前綴已 ≥ dist[u]（非負權在此處使用）。負權邊破壞這條歸納 ⇒ 貪心失效（見第 3 章）。同樣的貪心框架換目標可解：最大化路徑最小值（「瓶頸 Dijkstra」）、乘積最短路（取 log）。

★★★★洛谷 P4180 [BJWC2010] 嚴格次小生成樹圖MST倍增
思路

結論（由交換性質保證）：次小生成樹與 MST 恰差一條邊。枚舉每條非樹邊 (u,v,w)，替換樹上 u→v 路徑中的最大邊（嚴格次小需同時維護路徑最大與嚴格次大值，用倍增）。「MST 家族題 = Cut/Cycle Property 的花式應用」的代表。

★★★★★CF 888G Xor-MST（Boruvka）圖MST01-Trie

完全圖，邊權 = a_u XOR a_v，求 MST。n ≤ 2×10⁵，邊數 O(n²) 不可枚舉。

思路

Boruvka：每輪為每個連通塊找最小出邊、批量合併，輪數 O(log n)。「找最小 XOR 出邊」用 01-Trie 完成。Boruvka 的價值正在於此類隱式完全圖（邊權由函數定義）：每輪的「找最小出邊」換成資料結構查詢。分治視角：按最高位把點分兩半，跨半只需一條最小 XOR 邊。

2.9 貪心 × 資料結構

💡 核心想法

貪心策略本身簡單，但每一步的決策需要「快速找到／更新某個極值或空位」— 瓶頸在資料結構：並查集跳過已佔用位置、BIT/線段樹維護區間資訊、multiset 支援任意刪除。

★★★★截止期限任務調度（Job Sequencing）+ 並查集DSU排程

每個任務一單位時間、截止 d_i、報酬 p_i，最大化總報酬。

思路

按報酬遞減排序，每個任務安排在「≤ d_i 的最晚空閒時槽」（越晚越不擋別人）。找最晚空槽用並查集：find(d) 返回 ≤ d 的最晚空槽，佔用後 fa[t] = t−1，均攤近 O(1)。正確性可用擬陣描述：「可行任務子集」構成 transversal matroid，按權降序能加就加必最優 — 2.11.1 的具體實例。

程式碼（C++） 
sort(jobs.begin(), jobs.end(), [](auto& a, auto& b){ return a.p > b.p; });
vector<int> fa(maxD + 1); iota(fa.begin(), fa.end(), 0);
function<int(int)> find = [&](int x){ return fa[x]==x ? x : fa[x]=find(fa[x]); };
long long ans = 0;
for (auto& j : jobs) {
    int t = find(min(j.d, maxD));
    if (t > 0) { ans += j.p; fa[t] = t - 1; }   // 佔用時槽 t
}
★★★★洛谷 P1250 種樹區間差分約束→貪心

區間 [l_i, r_i] 內至少要種 t_i 棵樹，每個位置至多一棵，求最少總棵數。

思路

按右端點遞增排序，每個約束不足時從右端點往左補種（種得越右，越可能被後面的區間重複利用）。交換論證：把任何解中較左的樹右移到區間右端，不影響當前約束、只會幫助後續。加速：BIT 查區間和 + 並查集找「最右空位」。同型進階：洛谷 P2894、CF 志願者/餵牛系列。

★★★CF 1526C2 Potions（multiset/堆反悔）反悔貪心multiset

從左到右喝藥水（可跳過），生命值不得為負，最多喝幾瓶？

思路

全喝，血量為負時反悔吐掉「已喝中最毒的一瓶」（小根堆取最小負值）。與 LC 630 同構 — 反悔貪心配上不同容器（heap / multiset）反覆出現在 CF Div2 C/D，是性價比最高的訓練題型之一。

2.10 貪心 × 二分答案（參數化貪心）

💡 核心想法

「最大化最小值 / 最小化最大值」的答案具單調性：二分答案 x，把最優化問題降級成判定問題「能否達成 x」— 而判定問題往往有顯然的貪心解。貪心不必解決原題，只需服務 check。

🔎 適用特徵
  • 題面出現「最大的最小」「最小的最大」「至少/至多 k 次操作內」。
  • 驗證一個候選答案，比直接構造最優答案容易得多。
★★★洛谷 P2678 [NOIP2015] 跳石頭二分+貪心

移走至多 M 塊石頭，最大化相鄰石頭最短跳躍距離。（= POJ 2456 Aggressive Cows）

思路

二分最短距離 x；check：從起點掃過去，與上一塊保留石頭距離 < x 的就移走，數移走數是否 ≤ M。check 的貪心正確性：碰到近的就移，絕不提前移遠的 — 領先論證。

程式碼（C++） 
auto ok = [&](long long x){
    long long last = 0; int removed = 0;
    for (int i = 1; i <= n; i++)
        if (d[i] - last < x) removed++;      // 移走第 i 塊
        else last = d[i];
    if (L - last < x) return false;          // 終點是牆，不可移
    return removed <= m;
};
long long lo = 0, hi = L, ans = 0;
while (lo <= hi) {
    long long mid = (lo + hi) / 2;
    if (ok(mid)) { ans = mid; lo = mid + 1; } else hi = mid - 1;
}
★★★★CF 1486D Max Median二分+貪心中位數

找長度 ≥ k 的子陣列，最大化其中位數。

思路

二分中位數 x，把 ≥ x 記 +1、< x 記 −1：「存在長 ≥ k 子陣列中位數 ≥ x」⟺「存在長 ≥ k 子陣列和 > 0」，前綴和 + 前綴最小值 O(n) 判定。「二分 + ±1 轉化」是中位數最值題的萬能鑰匙（同型：POJ 3579、各類「第 k 大最大化」）。

★★★★★01 分數規劃（最優比率生成樹/環）二分+貪心分數規劃

最大化 Σa_i·x_i / Σb_i·x_i（選擇 x ∈ 可行集）。例：POJ 2728 最優比率生成樹、洛谷 P3199 最小圈。

思路

二分比率 λ，判定「是否存在解使 Σ(a_i − λ·b_i)x_i ≥ 0」— 每個元素的權重變為 a − λb，判定問題退化成普通的最優化（做一次 MST／找負環／取前 k 大），可用貪心或標準演算法。Dinkelbach 迭代可加速。參數化思想的巔峰：把非線性目標「線性化」後餵給貪心。

2.11 專家專題 ★★★★★

2.11.1 擬陣（Matroid）：貪心正確性的統一理論

擬陣 M = (E, I)：元素全集 E 與「獨立集」族 I，滿足（1）遺傳性：獨立集的子集獨立；（2）交換公理：|A| < |B| 且皆獨立 ⇒ 存在 x ∈ B\A 使 A∪{x} 獨立。

🧾 Rado–Edmonds 定理

在擬陣上「按權重降序、能加就加」的貪心必得最大權基；反之，若一個獨立系統對所有權重函數貪心都正確，它必是擬陣。— 這給了「貪心何時必然正確」的完整刻畫。

  • 圖形擬陣（無環邊集）→ Kruskal 正確性。
  • 分割擬陣 / 橫貫擬陣 → Job Sequencing（2.9）按報酬降序能排就排的正確性。
  • 擬陣交：兩個擬陣的公共獨立集最大化（如彩色生成樹 = 圖形擬陣 ∩ 分割擬陣），有多項式演算法（增廣路），但已超出貪心 — 知道邊界在哪本身就是專家素養。CF 話題題：CF 1284G。

2.11.2 Slope Trick：凸函數 DP 的貪心壓縮

★★★★★CF 713C Sonya and Problem Wihtout a Legendslope trick堆

每次操作 ±1，把序列改成嚴格遞增，最小總操作數。

提示

嚴格遞增先做變換 a_i −= i 化為非嚴格。定義 f_i(x) = 前 i 項、第 i 項改為 ≤ x 時的最小代價 — 它是關於 x 的凸的分段線性函數。

思路

轉移 f_i(x) = min_{y≤x} [f_{i−1}(y)] + |a_i − x|。凸函數的「前綴 min」= 砍掉右半上升段；「+|a−x|」= 左右各加一個斜率轉折點。用大根堆存左半段的斜率轉折點集合：每步 push a_i 兩次、若堆頂 > a_i 則答案累加 top − a_i 並彈出替換。整套操作把 O(n·V) 的 DP 壓成 O(n log n) 貪心。

程式碼（C++） 
priority_queue<long long> L;    // 凸函數左半段斜率轉折點
long long ans = 0;
for (int i = 0; i < n; i++) {
    long long x = a[i] - i;      // 嚴格遞增 → 非嚴格
    L.push(x);
    if (L.top() > x) {           // 出現下降段，付出代價抹平
        ans += L.top() - x;
        L.pop(); L.push(x);
    }
}

進階應用：APIO 2016 烟花表演（樹上 slope trick，用可並堆合併子樹的凸函數）、ABC 217H。辨識特徵：代價為絕對值/分段線性、狀態值域大、DP 關於值域是凸的。

2.11.3 交換論證 × DP 的一般化

2.3 的「Tower」模式推廣：當問題 =「選子集 + 定順序」時，先用相鄰交換論證證明存在最優解按某鍵有序，把順序維度消掉，再對排序後的序列 DP。訓練重點是推導鍵值的代數功力：把「i 在 j 前可行/更優」的條件化成 g(i) ≤ g(j) 的形式。同型：洛谷 P1417 烹調方案（鍵 b_i·c_j 交叉比較）、CF 1203F。

2.11.4 對抗式貪心（博弈中的貪心）

輪流行動的博弈裡，「雙方都貪心拿當前最大」在可加性目標下常是最優（如 CF 談判分糖果：排序後輪流取），證明用策略交換。但存在後手結構（取走影響未來選項）時貪心失效需要博弈 DP — 分辨兩者的試金石：你的選擇會不會改變對手未來選項的「相對」價值。例：LC 1690、CF 1539E。

2.11.5 反悔貪心 ⇄ 模擬費用流

★★★★★洛谷 P6122 [NOI Online #1] 隊長的技能 / CF 802O April Fools' Problem模擬費用流反悔貪心二分

CF 802O：n 天中選 k 天準備（費 a_i）、k 天列印（費 b_j），準備日 ≤ 對應列印日，最小總費。

思路

費用流建模顯然但跑不動（n ≤ 5×10⁵）。模擬費用流：逐天掃描，遇 b_j 時與「堆中最小的未配對 a」成交；成交後把 −(a_i)+b_j 類型的「反悔券」推回堆，允許未來更便宜的 a 替換掉這次的 a、或更貴的 b 被換掉 — 每一張反悔券精確對應殘量網路的一條反向邊。k 的限制用 wqs 二分（帶懲罰的拉格朗日鬆弛）處理。反悔貪心（2.4）是本技術的單反向邊特例；掌握「彈出成交 → 推回反悔券」的對應，即可自行推導新題的模擬費用流。

3經典「假貪心」陷阱

每個陷阱都是一面鏡子：照出貪心成立所依賴的、平時看不見的前提。

陷阱 10/1 背包按性價比貪心

反例：容量 10，物品 (重6, 值9)、(重5, 值7)、(重5, 值7)。性價比貪心先拿 1.5 的第一件，剩 4 裝不下 → 得 9；最優是兩件 1.4 的 → 14。

為什麼失敗：不可分割 ⇒ 交換「一單位重量」時會破壞容量可行性，交換論證的「交換後仍可行」這步斷裂。正解：背包 DP。部分背包（2.1.1）之所以能貪，正因可分割補上了這一步。

陷阱 2非 canonical 硬幣系統找零

反例：面額 {1, 3, 4} 湊 6 — 貪心 4+1+1（3 枚），最優 3+3（2 枚）。

為什麼失敗：大面額不再「支配」小面額組合（4 不能被替換分析覆蓋 3+3 的情形）。正解：完全背包 DP；理論判定見 2.7 的 Kozen–Zaks。

陷阱 3帶權區間排程按右端點貪心

反例：[1,10] 權 100 vs. [1,2],[3,4] 各權 1。按右端點貪心先選小區間 → 得 2；最優選大區間 → 100。

為什麼失敗：「結束早」與「權大」是不可比的兩個維度，任何單一排序鍵都會被另一維度的極端值擊穿。正解：排序 + 二分 + DP（2.2.5）。

陷阱 4負權邊上跑 Dijkstra／用 Dijkstra 求最長路

為什麼失敗：Dijkstra 的歸納不變量「取出的點距離已收斂」依賴邊權非負（2.8）；最長路即使非負權也 NP-hard（一般圖），DAG 上才可 DP。正解：負權用 Bellman-Ford/SPFA；最長路在 DAG 上拓撲 DP。

陷阱 5「交換相鄰論證」推出的 comparator 不滿足嚴格弱序

症狀：本地 AC、提交 RE/WA — std::sort 對不合法 comparator 是未定義行為。

檢查法：確認 cmp(a,a)==false、不可同時 cmp(a,b)&&cmp(b,a)、可傳遞。最保險的做法是把比較化成單一數值鍵（如 2.1.2 的 a+b vs b+a 其實是實數鍵）。含 0 或相等元素時特別容易翻車。

4證明方法實戰模板

4.1 交換論證五步驟模板

🧾 模板
  1. 設：設 OPT 為任一最優解，G 為貪心解，且 OPT ≠ G。
  2. 找：找出第一個分歧點 —— OPT 與 G 第一個不同的決策（或 OPT 中第一對「違反貪心排序鍵」的相鄰元素）。
  3. 換：把 OPT 在該處的選擇換成 G 的選擇（或交換那對相鄰元素），得到 OPT′。
  4. 證：證明 OPT′ 仍可行、且目標值不比 OPT 差。← 全部的工作量在這一步。
  5. 推：交換有限次後 OPT 變成 G 且始終不變差 ⇒ G 最優。∎

完整範例（活動選擇）：設 OPT 第一個活動為 j，G 第一個為 g（結束最早）。因 end(g) ≤ end(j)，把 OPT 中 j 換成 g：g 結束更早，不與 OPT 其餘活動衝突 ⇒ OPT′ 可行且數量相同。之後對子問題歸納。∎

4.2 領先論證模板

🧾 模板

定義「進度量」m(·)（如：前 i 個選擇後的結束時間 / 剩餘資源）。歸納證明：對每個 i，貪心解的 m(G, i) 不劣於任何解的 m(S, i)。步步領先 ⇒ 若 S 可行/達到 k，G 也能。適用於「貪心與對手逐步並排比較」自然的問題（活動選擇、跳躍遊戲、分發餅乾）。
