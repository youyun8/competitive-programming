2.5 堆積貪心（Priority-Queue Greedy）

💡 核心想法

貪心策略是「每一步取全域最小（或最大）的元素合併/處理」，且處理後會產生新元素回到候選池 — 用 heap 維護候選池，總複雜度 O(n log n)。

★★洛谷 P1090 合併果子（Huffman）堆Huffman

每次合併兩堆果子，代價為兩堆重量和，求合併成一堆的最小總代價。

思路

每次取最小的兩堆合併 — 即 Huffman 樹構造。證明（交換論證）：最小的兩個元素必然可以是最優樹中深度最深的一對兄弟 — 若不是，把它們與最深處的元素交換，總代價（Σ 重量×深度）不增。合併次序對應樹形，貪心構造即最優前綴碼，這也是資訊理論中 Huffman 編碼最優性的來源。

程式碼（C++） 
priority_queue<long long, vector<long long>, greater<>> pq(w.begin(), w.end());
long long cost = 0;
while (pq.size() > 1) {
    long long a = pq.top(); pq.pop();
    long long b = pq.top(); pq.pop();
    cost += a + b; pq.push(a + b);
}
★★★★洛谷 P2168 [NOI2015] 荷馬史詩（k 叉 Huffman）堆Huffman構造

k 進位前綴編碼，最小化總長度，並在最小總長下最小化最長編碼。

思路

k 叉 Huffman：每次合併 k 個最小的。關鍵技巧：若 (n−1) mod (k−1) ≠ 0，先補權值為 0 的虛擬節點湊齊，保證每次都能滿 k 合併（否則根附近會浪費短編碼）。第二關鍵字：堆中同權值時取「樹高最小」者合併，即可同時最小化最深深度。

★★★LC 295 數據流中位數（對頂堆）堆中位數
思路

大根堆存小的一半、小根堆存大的一半，兩堆大小差 ≤ 1，中位數在堆頂。這個「對頂堆」結構是許多貪心題的載體（如 2.7 的中位數貪心需要動態中位數、slope trick 的可合併堆雛形）。滑動視窗版需要延遲刪除或 multiset。

★★★洛谷 P1631 序列合併（k 路歸併）堆多路歸併

兩個長度 n 的遞增序列，n² 個和中最小的 n 個。

思路

堆中放 (a_i + b_1, i, 1)，彈出 (i, j) 時推入 (i, j+1) — 把 n² 個候選組織成 n 條有序鏈做 k 路歸併。同型：LC 373、Super Ugly Number、CF「第 k 小的 XXX」系列。核心辨識：候選集合呈笛卡兒積/指數大，但可以分層有序生成。
