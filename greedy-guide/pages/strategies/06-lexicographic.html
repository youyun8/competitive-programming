2.6 字典序貪心與單調堆疊構造

💡 核心想法

字典序目標自帶貪心性質：第一位越小/大，後面全不用比。所以逐位確定「當前位能放的最好字元」，配一個 O(1)/O(log) 的可行性檢查（剩餘資源夠不夠完成約束）。實作上大量以單調堆疊出現：新字元把棧頂「還有機會再來的較差字元」彈掉。

★★LC 402 移掉 K 位數字字典序單調棧

刪 k 位使剩下的數最小。

思路

維護遞增棧：新數字比棧頂小且還有刪除額度時，彈棧頂（高位變小永遠更優 — 這就是字典序的貪心選擇性質）。掃完還有額度就從尾巴刪。

程式碼（C++） 
string st;
for (char c : num) {
    while (!st.empty() && k && st.back() > c) { st.pop_back(); k--; }
    st.push_back(c);
}
st.resize(st.size() - k);
st.erase(0, st.find_first_not_of('0'));      // 去前導零
return st.empty() ? "0" : st;
★★★LC 316 去除重複字母字典序單調棧可行性檢查

刪去重複字母使每個字母恰出現一次，且結果字典序最小。

思路

單調棧 + 可行性檢查：彈棧頂的條件除了「棧頂 > 新字元」，還必須「棧頂字元在後面還會出現」（否則彈了就湊不齊了）。這是字典序貪心的完整形態：貪 + 保證還能完成。用剩餘計數陣列與 in-stack 標記各一個即可。

★★★★LC 321 拼接最大數字典序單調棧歸併

從兩個陣列中共取 k 個數（各自保持相對順序），拼成最大的數。

思路

三層貪心組合：枚舉從第一個陣列取 i 個 — 各自用單調棧取出「保留 i 個的最大子序列」（LC 402 的最大版）— 再貪心歸併兩個序列（比較整段字典序決定下一個取誰，注意相等時要往後比，故歸併是 O(n²) 最壞）。展示「字典序貪心可作為零件組裝」的層次感。

★★★字典序最小拓撲序字典序圖堆
思路

Kahn 演算法中把普通 queue 換成小根堆：每次在所有入度 0 的點中取編號最小者。正確性：第一位可以是任何目前入度 0 的點，取最小者是字典序的貪心選擇性質；取走後子問題同構。⚠ 對比陷阱：「字典序最小拓撲序」≠「每個點盡量早出現」— HDU 4857 反向建圖才是後者，辨識題意的細微差異是這型題的坑。
