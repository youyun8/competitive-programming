## 2. 策略分類（核心章節，每類含：核心想法、適用特徵、證明要點、模板碼、例題）

### 2.1 排序貪心（Sort & Sweep）
- 核心：先按某鍵值排序，再線性掃描做決策
- 2.1.1 單鍵排序
  - 部分背包（Fractional Knapsack）★
  - 分發餅乾（LC 455）★
  - 雷達安裝 / 區間選點（POJ 1328）★★
- 2.1.2 自訂比較函數（pairwise 交換論證推導 comparator）
  - 拼最大數（LC 179 Largest Number）★★
  - 國王遊戲 / 耍雜技的牛（洛谷 P1080 / P1842）★★★
  - 注意：comparator 必須滿足嚴格弱序（strict weak ordering）
- 2.1.3 雙指針配對
  - 救生艇（LC 881）★★
  - 田忌賽馬（POJ 2287）★★★

### 2.2 區間問題全家桶（Interval Scheduling Family）
- 核心：按左端點或右端點排序的抉擇
- 2.2.1 不重疊區間最多選幾個（活動選擇）— 按右端點排 ★★
  - LC 435 無重疊區間 / UVa 10382
- 2.2.2 區間完全覆蓋最少段數 — 按左端點排 + 貪心延伸 ★★★
  - LC 1024 視頻拼接、NOIP 2005 校門外的樹變形
- 2.2.3 最少資源數（區間分組 / 教室問題）— 掃描線或小根堆 ★★★
  - LC 253 會議室 II、洛谷 P2887
- 2.2.4 區間點覆蓋（最少箭射氣球 LC 452）★★
- 2.2.5 帶權區間 → 貪心失效 → 轉 DP 的邊界討論 ★★★

### 2.3 交換論證排程（Scheduling by Exchange Argument）
- 核心：假設最優解相鄰兩項可交換，推導排序鍵
- 2.3.1 最小化加權完成時間（Smith's Rule：按 p/w 排序）★★★
- 2.3.2 雙機流水線 Johnson's Rule ★★★★
- 2.3.3 截止期限＋懲罰：Lawler / 按 deadline 排 + 反悔堆 ★★★★
- 2.3.4 AtCoder DP Contest X - Tower（排序後 DP 的前處理交換論證）★★★★

### 2.4 反悔貪心（Regret Greedy / 可撤銷貪心）
- 核心：先貪心接受，之後用堆撤銷最差決策
- 2.4.1 反悔堆模板
  - 課程表 III（LC 630）★★★
  - 建築維修（洛谷 P4053）★★★★
  - 最少加油次數（LC 871）★★★
- 2.4.2 反悔配對／反悔交換
  - 股票買賣 II 反悔流（CF 865D Buy Low Sell High）★★★★
  - Work Scheduling（USACO 2009）★★★
- 2.4.3 與費用流的關係：反悔貪心 = 模擬增廣路（專家視角）★★★★★

### 2.5 堆積貪心（Priority-Queue Greedy）
- 核心：每步取全域最小/最大，用 heap 維護
- 2.5.1 Huffman 編碼與合併果子（洛谷 P1090）★★
- 2.5.2 k 叉 Huffman（補虛點技巧，洛谷 P2168 荷馬史詩）★★★★
- 2.5.3 對頂堆 / 動態中位數 ★★★
- 2.5.4 多路歸併：丑數、Super Ugly Number、序列合併 ★★★

### 2.6 字典序貪心與單調堆疊構造
- 核心：要字典序最小 → 每一位放能放的最小字元 + 可行性檢查
- 2.6.1 移掉 K 位數字（LC 402）＝單調棧貪心 ★★
- 2.6.2 去除重複字母（LC 316 / 1081）★★★
- 2.6.3 拼接最大數（LC 321，跨兩序列合併）★★★★
- 2.6.4 字典序最小拓撲序（優先佇列 + Kahn）★★★

### 2.7 數學貪心與調整法
- 核心：不等式、極端原理、中位數/平均數性質
- 2.7.1 排序不等式（Rearrangement Inequality）應用 ★★★
- 2.7.2 倉庫選址：中位數最小化絕對值和（LC 462）★★
- 2.7.3 糖果傳遞 / 環形均分紙牌（中位數進階，洛谷 P2512）★★★★
- 2.7.4 硬幣問題：canonical coin system 何時貪心正確 ★★★
- 2.7.5 木棍切割 / 乘積最大拆分（均值不等式）★★

### 2.8 圖上貪心
- 核心：經典圖演算法本質上是貪心 + 正確性由 cut/exchange 性質保證
- 2.8.1 MST：Kruskal / Prim 與 Cut Property、Cycle Property ★★★
- 2.8.2 Dijkstra 的貪心正確性（非負權）★★★
- 2.8.3 次小生成樹、瓶頸生成樹（貪心性質延伸）★★★★
- 2.8.4 Boruvka 與其在 CF 題的應用（XOR-MST）★★★★★

### 2.9 貪心 × 資料結構
- 核心：貪心決策需要快速查詢/更新 → DSU、BIT、線段樹
- 2.9.1 截止期限任務調度 + 並查集找最晚空位（Job Sequencing）★★★★
- 2.9.2 貪心 + BIT/線段樹：區間染色、種樹（洛谷 P1250）★★★★
- 2.9.3 貪心 + multiset：CF 常見「配對消除」題型 ★★★

### 2.10 貪心 × 二分 / 參數化
- 核心：「最大化最小值」→ 二分答案 + 貪心檢查可行性
- 2.10.1 二分答案 + 貪心 check 模板 ★★★
  - 跳石頭（NOIP 2015 / 洛谷 P2678）、Aggressive Cows（POJ 2456）
- 2.10.2 中位數貪心 + 二分：CF 1486D ★★★★
- 2.10.3 Fractional programming（01 分數規劃）與貪心 check ★★★★★

### 2.11 專家專題（★★★★★）

## 3. 反面教材：經典「假貪心」陷阱
- 0/1 背包不能貪心：反例
- 非 canonical 硬幣系統：{1,3,4} 湊 6
- 帶權區間排程：權重破壞交換論證
- 最長路徑不能 Dijkstra
- 每個陷阱附「為什麼失敗」與「正確解法指向」

## 網站設計規格
- **多頁架構**：每個子主題一個獨立 `.html` 頁面（首頁 + 16 個內容頁 = 17 頁），零外部依賴（CSP 友善、可離線開啟）
- 左側固定側邊欄導覽（手機收合為抽屜），依目前頁面自動高亮；頁尾提供「上一節 / 下一節」線性導讀
- **顯示設定面板**（⚙️ 設定）：主題（淺色／深色／跟隨系統）、內文字體大小（小/中/大/特大）、版面寬度（窄/預設/寬/全螢幕），設定存於 `localStorage`，跨頁面／重新整理皆保留
- **練習進度追蹤 + 跨裝置雲端同步**：每題可標記尚未練習/需複習/已通過；本機模式用 `localStorage`，設定好 Firebase（Google 登入 + Firestore）後可跨裝置同步，未設定時自動優雅降級為純本機模式（見 `assets/progress.js`、`assets/firebase-config.js`、README「跨裝置進度同步」章節）
- 元件：難度星級徽章、策略標籤 chip、摺疊式「提示 → 思路 → 代碼」三段揭示
- 程式碼區塊：C++ 為主、關鍵題附 Python，語法高亮（自寫輕量 highlighter）
- 題目總表：原生 JS 篩選（難度 / 標籤 / 關鍵字搜尋）
- 響應式：手機單欄（抽屜側欄）、桌面雙欄
- **內容與版面分離**：教學內容存於 `content/*.html`（每個子主題一份），`build.py` 套上共用版面模板（側欄、設定面板、上一頁/下一頁、頁尾）產生最終靜態頁面；共用樣式/行為在 `assets/site.css`、`assets/site.js`、`assets/problems-data.js`
