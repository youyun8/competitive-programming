2.3 堆、對頂堆與可刪堆

💡 核心想法

只需要「最值」而不需要全序時，堆用 \(O(\log n)\) 的 push/pop 換掉排序。競賽三形態：裸堆（priority_queue）；對頂堆——大根堆存較小一半、小根堆存較大一半，堆頂夾著中位數／第 k 小；可刪堆（懶惰刪除）——再開一個「待刪堆」，兩堆頂相同就一起彈，把不支援的 erase 攤銷掉。multiset 是三者的萬能替身（多送前驅後繼），常數大一點。

🔎 適用特徵
  • 流式資料只問最值／中位數／第 k 大；決策「每次取最...的」（貪心搭檔，見貪心主題 2.4/2.5）。
  • 需要刪任意元素但捨不得平衡樹 → 懶刪除。
  • 需要名次（第 k 小隨 k 亂跳）、區間查詢 → 升級 權值 BIT／平衡樹。
模板（C++）：對頂堆 + 懶刪除 
// 對頂堆：lo（大根）存較小半、hi（小根）存較大半，|lo| = |hi| 或 |hi|+1...
priority_queue<int> lo;
priority_queue<int, vector<int>, greater<int>> hi;
void insert(int x) {
    if (lo.empty() || x <= lo.top()) lo.push(x); else hi.push(x);
    if (lo.size() > hi.size() + 1) { hi.push(lo.top()); lo.pop(); }
    if (hi.size() > lo.size())     { lo.push(hi.top()); hi.pop(); }
}
int median() { return lo.top(); }

// 懶刪除：erase 只記帳，top 時對消
priority_queue<int> q, del;
void erase(int x) { del.push(x); }
int top() {
    while (!del.empty() && q.top() == del.top()) q.pop(), del.pop();
    return q.top();
}
★洛谷 P3378 堆模板堆

插入、查最小、刪最小。

思路

priority_queue 直接過。值得做一次手寫二叉堆（up/down 兩函式）——不是為了比 STL 快，是為了之後看懂可併堆與堆式線段樹。

★★★LC 295 數據流的中位數堆對頂堆

流式插入，隨時報中位數。

思路

對頂堆模板：插入後兩行 rebalance 維持大小不變量。與貪心主題 2.5 共用——中位數貪心（P1889 等）的引擎就是它。

★★★洛谷 P1801 黑匣子對頂堆

插入序列給定，第 \(i\) 次查詢問「目前第 \(i\) 小」。

思路

對頂堆的推廣：小半堆的大小固定為「目前查詢序號」，每次查詢後閾值 +1、rebalance 一次。看懂「對頂堆 = 維護一條可移動的分界線」，中位數只是分界線在中間的特例。BIT 上倍增（2.5）是另一解，對照。

★★★★LC 480 滑動窗口中位數對頂堆懶刪除

長 k 窗口的中位數序列。

思路

對頂堆 + 懶刪除的合體：出窗元素只記到待刪堆，大小不變量改用「有效大小」計算（各堆維護一個「已死但未彈」計數）。這題把兩個技巧的邊界細節全部逼出來，寫過一次終生免疫。multiset + 中位數指標是更短的替代解。

★★★LC 2102 序列順序查詢對頂堆

插入評分景點，第 \(i\) 次查詢問「第 \(i\) 好」。

思路

P1801 的變種（查詢閾值同樣單調 +1）。閾值單調 → 對頂堆；若閾值亂跳 → 就得上權值 BIT/平衡樹。「查詢參數是否單調」決定武器等級，是很好的選型練習。

⏫ 進階去向

「兩個堆合併」→ 左偏樹/可併堆（貪心主題 APIO 烟花表演有實戰）；名次隨機查 → 權值 BIT；帶區間結構 → 平衡樹。堆的本職是「配角結構」——搭貪心、搭 Dijkstra、搭模擬，單獨成題反而少。
