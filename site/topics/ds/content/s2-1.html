2.1 前綴和、差分與 ST 表（靜態的力量）

💡 核心想法

沒有修改（或修改全部發生在查詢之前），就不需要「結構」——預處理換查詢。三件套：前綴和把「區間求和」變 \(O(1)\) 相減；差分把「區間加」變兩個端點打點（修改與查詢的對偶：前綴和的逆運算）；ST 表把「可重複貢獻」的區間 min/max/gcd 變 \(O(1)\)（兩個重疊子區間蓋滿整段，重疊不影響答案）。

🔎 適用特徵
  • 「先做完所有修改、再回答所有查詢」——差分收工後做一次前綴和。
  • 完全靜態的區間和/min/gcd——別開線段樹，會被同行嘲笑。
  • 二維小規模（\(n\times m \le 10^7\)）的子矩陣統計——二維前綴和 + 容斥。
模板（C++）：三件套 
// 一維前綴和：S[i] = S[i-1] + a[i]；sum(l,r) = S[r] - S[l-1]
// 差分：  d[l] += v, d[r+1] -= v；最後 a = d 的前綴和
// 二維前綴和：S[i][j] = S[i-1][j] + S[i][j-1] - S[i-1][j-1] + a[i][j]
// 二維差分：d[x1][y1]+=v; d[x1][y2+1]-=v; d[x2+1][y1]-=v; d[x2+1][y2+1]+=v;

// ST 表：f[j][i] = [i, i + 2^j) 的 max
for (int i = 1; i <= n; i++) {
    f[0][i] = a[i];
}
for (int j = 1; (1 << j) <= n; j++) {
    for (int i = 1; i + (1 << j) - 1 <= n; i++) {
        f[j][i] = max(f[j-1][i], f[j-1][i + (1 << (j-1))]);
    }
}
int query(int l, int r) {              // 兩段重疊覆蓋
    int k = __lg(r - l + 1);
    return max(f[k][l], f[k][r - (1 << k) + 1]);
}
★★LC 304 二維區域和檢索前綴和

多次查子矩陣和（靜態）。一維版 LC 303 ★。

思路

二維前綴和 + 容斥：\(\mathrm{sum}=S_{x_2y_2}-S_{x_1-1,y_2}-S_{x_2,y_1-1}+S_{x_1-1,y_1-1}\)。把「加一減二補一」畫成圖記住，之後二維差分就是它的鏡像。

★洛谷 P2367 語文成績差分

大量區間加，最後問全體最小值。

思路

差分模板：所有修改打完端點再一次前綴和。體會「修改全在查詢前」= 免結構的訊號。LC 1109 航班預訂統計 ★★ 是同一題換皮。

★★洛谷 P3397 地毯二維差分

二維平面上鋪矩形地毯，最後問每格層數。

思路

二維差分四個角打點，收工兩次前綴和（先行後列）。自己推一遍四角的正負號，比背模板牢固十倍。

★★洛谷 P1719 最大加權矩形前綴和Kadane

找權值和最大的子矩形（\(n\le 120\)）。

思路

枚舉上下界 \(O(n^2)\)，把中間各列壓成一維陣列，套最大子段和（Kadane，見 DP 主題 2.1）\(O(n)\)。「降維打擊：枚舉一維、壓縮一維、DP 一維」是二維統計的常青套路。

★★洛谷 P3865 ST 表模板ST表RMQ

靜態區間最大值，\(q\le 2\times 10^6\)。

思路

查詢量卡掉 \(O(q\log n)\) 的線段樹——ST 表 \(O(1)\) 查詢的主場（配快讀）。理解「可重複貢獻」：max 重疊算兩次沒關係，和不行，所以 ST 表做不了區間和。此表也是 字串主題 SA 的 height RMQ、樹上歐拉序 LCA 的引擎。

⏫ 進階去向

一旦「修改與查詢交錯」，前綴和失效：可差分資訊 → BIT；區間修改 → 線段樹。而「差分」的思想不死：樹上差分（2.11.2）、掃描線（2.8）都是它的高維化身。
