2.9 平衡樹（FHQ Treap／Splay）

💡 核心想法

維護動態有序集合：插入、刪除、名次、第 k 小、前驅後繼全部 \(O(\log n)\)。競賽首選 FHQ Treap（無旋 Treap）：靠隨機優先級維持期望平衡，只用兩個原語——split（把樹按值或按大小劈成兩棵）與 merge（把兩棵值域有序的樹縫回去），所有操作都是這兩個動作的組裝，好寫、好背、支援可持久化。

🔎 適用特徵
  • 插入/刪除與名次查詢交錯且值域無法離散化（能離散化 → BIT 倍增更短）。
  • 維護序列本身（按下標 split）：區間翻轉、區間搬移——線段樹做不到「改變元素順序」。
  • 只要有序集合＋前驅後繼 → 先想 multiset／pb_ds tree 能不能混過去。
模板（C++）：FHQ Treap 核心 
int ch[N][2], val[N], pri[N], sz[N], tot;
int node(int value) {
    int id = ++tot;
    val[id] = value;
    pri[id] = rand();
    sz[id] = 1;
    return id;
}
void pushup(int treap_node) {
    sz[treap_node] = sz[ch[treap_node][0]] + sz[ch[treap_node][1]] + 1;
}
void split(int treap_node, int value, int& low_root, int& high_root) {
    // 按值：low_root 全 ≤ value，high_root 全 > value
    if (!treap_node) {
        low_root = high_root = 0;
        return;
    }
    if (val[treap_node] <= value) {
        low_root = treap_node;
        split(ch[treap_node][1], value, ch[treap_node][1], high_root);
    } else {
        high_root = treap_node;
        split(ch[treap_node][0], value, low_root, ch[treap_node][0]);
    }
    pushup(treap_node);
}
int merge(int low_root, int high_root) {      // 前提：low_root 的值全 ≤ high_root
    if (!low_root || !high_root) {
        return low_root | high_root;
    }
    if (pri[low_root] < pri[high_root]) {
        ch[low_root][1] = merge(ch[low_root][1], high_root);
        pushup(low_root);
        return low_root;
    } else {
        ch[high_root][0] = merge(low_root, ch[high_root][0]);
        pushup(high_root);
        return high_root;
    }
}

//                root = merge(merge(low_root, node(value)), high_root);
// erase(value):  split 出恰等於 value 的樹，merge 掉一個節點
// 文藝平衡樹：split 改按 sz（前 k 個），節點加翻轉懶標記 + pushdown
★★★★洛谷 P3369 普通平衡樹平衡樹FHQ

插入、刪除、查排名、查第 k 小、前驅、後繼。

思路

六操作全部由 split/merge 組裝（前驅 = split(v−1) 後左樹最右）。先畫圖理解「split 沿途把樹撕成兩條拉鍊」，再背碼。對照解：值域離散化後權值線段樹/BIT 也能過此題——體會「平衡樹是線上無值域限制版」。

★★★★洛谷 P3391 文藝平衡樹平衡樹區間翻轉

多次翻轉序列的區間，輸出最終序列。

思路

平衡樹維護序列：中序遍歷 = 序列順序，split 按子樹大小切出 \([l,r]\)，對根打「翻轉懶標記」（交換左右兒子，pushdown 下傳）。「按值」與「按大小」是同一棵 FHQ 的兩種人格，切換的只有 split 的判據。

★★★★洛谷 P1486 [NOI2004] 郁悶的出納員平衡樹整體偏移

員工工資整體加減、低於下限自動離職、查第 k 大。

思路

整體加減不真的改每個節點——維護全域偏移量 delta，插入時存 \(v-\)delta。離職 = split 出「< 下限 − delta」的整棵子樹一次砍掉（計數即離職人數）。「全域 lazy + 按值批量切除」讓平衡樹的 split 展現出線段樹學不來的靈活性。

★★★★★洛谷 P2596 [ZJOI2006] 書架平衡樹序列維護

把某本書挪到頂/底/上下移一格、查某書名次、查第 k 本。

思路

序列平衡樹的綜合題：每本書記錄自己的節點編號，「查名次」要從節點向上爬累計左子樹大小（需父指標），移動 = split 抽出單點再 merge 到新位置。FHQ 加父指標或改用 Splay 皆可——順便理解 Splay 的賣點：把「訪問的節點轉到根」天然支援這類操作。

📎 Splay 與 pb_ds 備忘

① Splay：每次訪問後將節點旋到根，均攤 \(O(\log n)\)——競賽中主要作為 LCT 的底層；單獨使用時 FHQ 幾乎總是更好寫。② pb_ds：__gnu_pbds::tree 自帶 order_of_key / find_by_order，普通平衡樹題可以 20 行過——賽場救急合法武器，但區間翻轉這種序列操作救不了。
