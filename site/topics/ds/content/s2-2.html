2.2 單調堆疊與單調佇列

💡 核心想法

維護「還有機會成為答案」的候選集合：新元素進來時，把被它支配的舊候選（又舊又差）彈掉——留下的序列自然單調。每個元素進出各一次 ⇒ 均攤 \(O(n)\)。① 單調棧回答「左/右第一個更大（小）元素」；② 單調佇列在此之上多了「隊首過期出隊」，回答滑動窗口最值。

🔎 適用特徵
  • 句型「下一個更大元素」「以 \(i\) 為最小值能延伸多遠」→ 單調棧。
  • 句型「長度 ≤ k 的窗口最值」「決策集合隨 \(i\) 滑動」→ 單調佇列。
  • 它們不是「查詢結構」而是一次掃描的伴生結構——問題必須能重述成一次（或常數次）掃描。
模板（C++）：單調棧 + 單調佇列 
// 單調棧：每個 i 的「左邊第一個更小元素」下標 L[i]
stack<int> st;
for (int i = 1; i <= n; i++) {
    while (!st.empty() && a[st.top()] >= a[i]) st.pop();
    L[i] = st.empty() ? 0 : st.top();
    st.push(i);
}
// 單調佇列：長 k 窗口最大值（遞減佇列，存下標）
deque<int> q;
for (int i = 1; i <= n; i++) {
    while (!q.empty() && q.front() <= i - k) q.pop_front(); // 隊首過期
    while (!q.empty() && a[q.back()] <= a[i]) q.pop_back(); // 隊尾被支配
    q.push_back(i);
    if (i >= k) ans[i - k + 1] = a[q.front()];
}
★★LC 739 每日溫度單調棧

對每天求「幾天後首次更暖」。

思路

「下一個更大元素」的直譯。棧裡存「還沒等到答案的下標」——遞減棧；新元素把能回答的全彈出。洛谷 P5788 是同款模板題。

★★LC 42 接雨水單調棧雙指針

柱狀圖接住多少雨水。

思路

三解名題：按列（每列水位 = min(左max, 右max)−高，前後綴 max 預處理）、單調棧（按層結算：彈棧時「凹槽」成形）、雙指針（空間 O(1)）。單調棧解最能訓練「彈出瞬間結算貢獻」的思維——之後 P4248、LC 84 都是它。

★★★LC 84 柱狀圖中最大的矩形單調棧

柱狀圖內最大矩形面積。

思路

以每根柱子為高度瓶頸：能延伸的範圍 =（左邊第一個更矮, 右邊第一個更矮）之間——單調棧一遍求出兩側邊界。「枚舉最小值 + 求支配區間」是單調棧的招牌句型，字串主題 P4248 的 Σmin 同款。

★★★★LC 85 最大矩形單調棧降維

01 矩陣中全 1 的最大矩形。

思路

逐行維護「以本行為底、每列向上連續 1 的高度」，每行套一次 LC 84——\(O(nm)\)。二維 → 逐行一維化，與 P1719 的降維同思想。

★★洛谷 P1886 滑動窗口單調佇列

長 k 窗口的最小值與最大值序列（\(n\le 10^6\)）。

思路

單調佇列模板（= LC 239）。兩個 while 的順序無所謂，但「隊首判過期用下標、隊尾判支配用值」不能混。\(10^6\) 規模用陣列模擬 deque 更穩。

★★★洛谷 P1725 琪露諾單調佇列DP優化

每步從 \([i-r, i-l]\) 跳到 \(i\)，最大化收集值。

思路

\(f_i=\max_{j\in[i-r,i-l]}f_j + a_i\)：決策窗口隨 \(i\) 平移 → 單調佇列維護窗口 max。與 DP 主題 2.12.2 共用——那邊講「什麼樣的轉移可用單調佇列」，這邊看清楚佇列本身在做什麼。

⚠ 高頻坑

① 「≥ 還是 >」彈棧決定相等元素歸誰——求「嚴格更大」與「第一個不小於」的邊界不同，先想清楚再寫。② 單調佇列存下標不存值（過期判斷需要）。③ 窗口未滿 k 時就輸出答案。
