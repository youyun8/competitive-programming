## 0. 導讀與使用方式
- 資料結構題的本質：把「暴力重算」換成「維護增量」——操作序列 ×（修改型態 × 查詢型態）決定選型
- 一張「操作 → 結構」速查表：靜態區間查詢 → 前綴和/ST 表；單點改+區間查 → BIT；區間改 → 線段樹+懶標記；插入刪除+名次 → 平衡樹；奇怪的區間統計 → 分塊/莫隊；歷史版本 → 可持久化
- 難度分級表（資結版：★ ≈ 前綴和 … ★★★★★ ≈ Segment Tree Beats、LCT）
- 標籤系統說明：策略標籤（前綴和 / 單調棧 / 堆 / DSU / BIT / 線段樹 / 掃描線 / 平衡樹 / 莫隊 / 樹剖 / 可持久化 …）＋ 難度星級
- 與其他主題的關係：貪心 × 資料結構見貪心指南 2.9，DP 的資料結構優化見 DP 指南 2.12，01-Trie 見字串主題 2.5

## 1. 理論基礎：維護增量的思考框架
- 1.1 操作分析法：先列出（修改操作，查詢操作）配對，再問「查詢能不能拆成可合併的區間資訊」
- 1.2 可合併性（結合律）：sum/min/max/gcd 可合併；「區間眾數」不可合併 → 只能分塊/莫隊
- 1.3 均攤與勢能分析：單調棧每元素進出各一次、DSU 路徑壓縮、勢能法看懂 Segment Tree Beats
- 1.4 離線 vs 線上：離線 = 已知全部詢問 → 排序重排（掃描線、莫隊、CDQ）；強制線上 → 可持久化
- 1.5 複雜度預算：n, q ≤ 10⁵~10⁶ → O((n+q) log n)；允許 O(n√n) 的訊號；常數因子（線段樹 vs BIT 4~8 倍）
- 1.6 實作工具箱：離散化三行、1-indexed 慣例、`vector` vs 靜態陣列、快讀

## 2. 策略分類（每章含：核心想法、適用特徵、模板碼、例題卡片）

### 2.1 前綴和、差分與 ST 表（靜態的力量）
- 核心：不修改就不需要「結構」——預處理換查詢
- 2.1.1 一維/二維前綴和；子矩陣和容斥
- 2.1.2 差分：區間加 → 端點打標記；二維差分
- 2.1.3 ST 表（Sparse Table）：可重複貢獻問題（min/max/gcd）O(n log n) 預處理 O(1) 查詢
- 例題：
  - LC 303 區域和檢索 ★ / LC 304 二維區域和 ★★
  - 洛谷 P2367 語文成績（差分模板）★
  - LC 1109 航班預訂統計 ★★
  - 洛谷 P3397 地毯（二維差分）★★
  - 洛谷 P1719 最大加權矩形（2D 前綴和 + 枚舉上下界）★★
  - 洛谷 P3865 ST 表模板 ★★

### 2.2 單調堆疊與單調佇列
- 核心：維護「有用候選人」——被支配的元素永遠出局，每元素進出一次 = 均攤 O(n)
- 2.2.1 單調棧：下一個更大元素、左右第一個更小 → 以 i 為最值的最大延伸區間
- 2.2.2 單調佇列：滑動窗口最值；隊首過期、隊尾被支配
- 2.2.3 銜接：單調佇列優化 DP（DP 指南 2.12.2）、貪心的單調棧構造（貪心指南 2.6）
- 例題：
  - LC 739 每日溫度 ★★
  - 洛谷 P5788 單調棧模板 ★★
  - LC 42 接雨水 ★★
  - LC 84 柱狀圖中最大的矩形 ★★★
  - 洛谷 P1886 滑動窗口（單調佇列模板）★★
  - LC 85 最大矩形（84 的逐行疊加）★★★★

### 2.3 堆、對頂堆與可刪堆
- 核心：只要最值不要全序；std::priority_queue / multiset 的取捨
- 2.3.1 堆基礎與 heapify；競賽裡幾乎只用 push/pop/top
- 2.3.2 對頂堆：動態中位數（兩堆平衡不變量）
- 2.3.3 懶惰刪除（可刪堆）：刪除堆 + 主堆，top 對消
- 2.3.4 multiset 當「可刪堆＋前驅後繼」用；銜接反悔貪心（貪心指南 2.4）
- 例題：
  - 洛谷 P3378 堆模板 ★
  - LC 295 數據流的中位數（對頂堆，與貪心主題共用）★★★
  - 洛谷 P1801 黑匣子（第 k 小的對頂堆推廣）★★★
  - LC 480 滑動窗口中位數（對頂堆 + 懶刪除）★★★★
  - LC 2102 序列順序查詢（名次查詢的 multiset/BIT 對照）★★★

### 2.4 並查集（DSU）
- 核心：維護「等價類」——連通性、分組、傳遞關係；路徑壓縮 + 按秩合併 ≈ O(α(n))
- 2.4.1 基礎：find/union、連通塊計數
- 2.4.2 離線倒序：把「刪邊」變「加邊」
- 2.4.3 種類並查集（拆點）：敵人的敵人是朋友
- 2.4.4 帶權並查集：維護到根的相對關係（距離/奇偶/餘數）
- 2.4.5 可撤銷 DSU：只按秩不壓縮 + 操作棧；用途 → 線段樹分治（2.13）
- 例題：
  - 洛谷 P3367 並查集模板 ★
  - 洛谷 P1551 親戚 ★
  - 洛谷 P1892 團伙（種類/拆點）★★
  - 洛谷 P1197 星球大戰（離線倒序加邊）★★★
  - 洛谷 P2024 食物鏈（帶權 mod 3 / 拆三倍點）★★★★
  - 洛谷 P1955 程序自動分析（離散化 + 等式優先）★★★

### 2.5 樹狀數組（BIT / Fenwick Tree）
- 核心：lowbit 切前綴；碼量 1/5 的「前綴可差分」線段樹
- 2.5.1 單點加 + 前綴和查詢；為什麼要可差分（min 不行、和/計數行）
- 2.5.2 差分 BIT：區間加 + 單點查；兩個 BIT：區間加 + 區間和（推導）
- 2.5.3 權值 BIT：逆序對、動態名次
- 2.5.4 BIT 上倍增：O(log n) 求第 k 小（不用二分套查詢）
- 例題：
  - 洛谷 P3374 BIT 模板一（單點加區間和）★★
  - 洛谷 P3368 BIT 模板二（區間加單點查）★★
  - 洛谷 P1908 逆序對 ★★
  - LC 315 計算右側小於當前元素的個數（權值 BIT + 離散化）★★★
  - 洛谷 P5094 MooFest 加強版（雙 BIT：計數 + 求和）★★★★
  - BIT 上倍增求第 k 小（配 P1801 對照）★★★★

### 2.6 線段樹基礎（build / 懶標記 / 經典維護）
- 核心：線段二分結構，任意區間拆成 O(log n) 個節點；懶標記 = 「欠著的修改」
- 2.6.1 build/point update/range query；遞迴框架與 4n 空間
- 2.6.2 懶標記：區間加 + 區間和；下傳（pushdown）與上收（pushup）的責任劃分
- 2.6.3 多重懶標記的複合：先乘後加（仿射變換標記的複合順序）
- 2.6.4 「維護什麼才能合併」：最大子段和四元組（GSS 系列）——資訊設計思維
- 例題：
  - 洛谷 P3372 線段樹模板一（區間加區間和）★★★
  - 洛谷 P3373 線段樹模板二（區間加 + 區間乘）★★★★
  - 洛谷 P4513 小白逛公園（最大子段和四元組合併）★★★★
  - 洛谷 P1531 I Hate It（單點改區間 max 入門）★★
  - LC 307 區域和檢索—可修改 ★★★

### 2.7 線段樹進階（權值 / 動態開點 / 合併分裂 / 二分 / 李超）
- 核心：把「陣列下標」換成「值域」，把「先建滿」換成「用到才開」
- 2.7.1 權值線段樹：動態全域第 k 小、名次；與權值 BIT 的分工
- 2.7.2 動態開點：值域 10⁹ 不離散化；空間 O(q log V) 的估算
- 2.7.3 線段樹上二分：全域/區間第 k 小、第一個 ≥ x 的位置——在樹上走而非二分套查詢
- 2.7.4 線段樹合併：啟發式的反面——O(重疊節點數) 均攤；樹上問題收尾利器
- 2.7.5 李超線段樹：插入直線/線段、查單點最值；斜率優化的線上替代
- 例題：
  - 洛谷 P1908 逆序對（權值線段樹版對照 2.5）★★
  - 洛谷 P4556 [Vani有約會] 雨天的尾巴（樹上差分 + 線段樹合併）★★★★★
  - 洛谷 P4097 [HEOI2013] Segment（李超樹模板）★★★★
  - CF 1093G?（提及）；洛谷 P5490 → 移 2.8
  - 洛谷 P3960 列隊（NOIP2017，動態開點/BIT 找第 k 個空位）★★★★★

### 2.8 掃描線、二維數點與離線分治（CDQ / 整體二分）
- 核心：把一個維度變成「時間」，離線排序後用一維結構收割
- 2.8.1 掃描線求矩形面積並：y 離散化 + 覆蓋次數線段樹（不下傳的技巧）
- 2.8.2 二維數點：詢問差分成 4 個前綴 → 按 x 排序 + 權值 BIT
- 2.8.3 CDQ 分治：三維偏序 = 分治（第一維）+ 歸併（第二維）+ BIT（第三維）；「用分治換一層資料結構」
- 2.8.4 整體二分：把所有詢問一起二分答案；與主席樹（2.12）互為替代
- 例題：
  - 洛谷 P5490 掃描線模板（矩形面積並）★★★★
  - LC 218 天際線問題（掃描線 + multiset）★★★
  - 洛谷 P2163 [SHOI2007] 園丁的煩惱（二維數點）★★★
  - 洛谷 P3810 三維偏序（CDQ 模板）★★★★
  - 洛谷 P1527 [國家集訓隊] 矩陣乘法（整體二分：矩形第 k 小）★★★★★

### 2.9 平衡樹（FHQ Treap / Splay）
- 核心：維護「動態有序集合」的名次/前驅後繼/區間結構；FHQ = split/merge 兩個原語打天下
- 2.9.1 FHQ Treap：按值 split、按大小 split；insert/delete/rank/kth/pre/nxt 全部由 split+merge 組裝
- 2.9.2 文藝平衡樹：按下標 split + 翻轉懶標記——平衡樹維護序列
- 2.9.3 Splay 提及：LCT 的底層需要；均攤複雜度的直覺
- 2.9.4 偷懶選項：pb_ds（tree / priority_queue）與 multiset 能替代的範圍
- 例題：
  - 洛谷 P3369 普通平衡樹（六操作模板）★★★★
  - 洛谷 P3391 文藝平衡樹（區間翻轉）★★★★
  - 洛谷 P1486 [NOI2004] 郁悶的出納員（整體加減 + 第 k 大：偏移量技巧）★★★★
  - LC 1912?（略）；洛谷 P2596 [ZJOI2006] 書架（平衡樹維護序列）★★★★★

### 2.10 分塊與莫隊（根號演算法）
- 核心：O(√n) 是「不可合併資訊」的救生圈——塊內暴力、塊間打標記
- 2.10.1 序列分塊：塊長取 √n 的推導；區間加區間和的分塊版（與線段樹對照）
- 2.10.2 值域分塊：O(1) 修改 O(√V) 查詢——修改多查詢少的再平衡
- 2.10.3 普通莫隊：詢問排序（塊號、奇偶性優化）+ 雙指標增量維護；O((n+q)√n)
- 2.10.4 帶修莫隊：加時間維，塊長 n^(2/3)
- 2.10.5 回滾莫隊預告 → 2.13
- 例題：
  - 洛谷 P2801 教主的魔法（分塊 + 塊內排序二分）★★★★
  - LOJ 6277–6285 數列分塊入門 1–9（練功房，總述）★★~★★★★
  - 洛谷 P1494 [國家集訓隊] 小 Z 的襪子（莫隊模板：區間同色對機率）★★★★
  - SPOJ DQUERY 區間不同數個數（莫隊 / 離線 BIT 兩解）★★★
  - 洛谷 P1903 [國家集訓隊] 數顏色（帶修莫隊）★★★★★

### 2.11 樹上資料結構（LCA / 樹上差分 / 樹鏈剖分 / DSU on tree）
- 核心：把樹「線性化」——DFS 序、歐拉序、重鏈剖分序，然後套用序列結構
- 2.11.1 LCA：倍增法（跳表思想）；歐拉序 + ST 表 O(1) 查詢提及
- 2.11.2 樹上差分：點差分/邊差分——路徑加、最後一次 DFS 收和
- 2.11.3 DFS 序 + BIT/線段樹：子樹和、子樹加
- 2.11.4 樹鏈剖分：重鏈性質（每點到根 ≤ O(log n) 條鏈）；路徑改/查 = O(log²n)；順帶得到 O(log n) 的 LCA
- 2.11.5 DSU on tree（樹上啟發式合併）：子樹統計不帶修改的 O(n log n) 離線大殺器
- 例題：
  - 洛谷 P3379 LCA 模板（倍增）★★★
  - 洛谷 P3128 [USACO15DEC] Max Flow（樹上點差分）★★★
  - 洛谷 P3178 [HAOI2015] 樹上操作（DFS 序 + 線段樹）★★★
  - 洛谷 P3384 樹鏈剖分模板（路徑加/查 + 子樹加/查）★★★★
  - CF 600E Lomsat gelral（DSU on tree 模板）★★★★
  - 洛谷 P4556 雨天的尾巴（樹上差分 + 線段樹合併，與 2.7 共用）★★★★★

### 2.12 可持久化資料結構

- 2.12.2 主席樹（可持久化權值線段樹）：字首版本相減 = 區間資訊——靜態區間第 k 小
- 2.12.3 可持久化 01-Trie：字首相減求「區間最大 XOR」
- 2.12.4 空間估算與常見 MLE；帶修第 k 小 = BIT 套主席樹（樹套樹）提及 → 2.13

## 3. 常見錯誤與陷阱
- 線段樹空間開 2n → RE：為什麼是 4n；動態開點的節點數估算失誤 MLE
- 懶標記複合順序錯：先加後乘的污染；pushdown 忘記清標記、pushup 忘記呼叫
- BIT 從 0 開始 → lowbit(0)=0 死迴圈；BIT 維護 min 這種不可差分資訊
- 離散化陷阱：查詢值沒一起離散化、去重後下標錯位
- DSU 只壓縮不按秩在鏈狀卡例退化；帶權 DSU 壓縮時忘記更新權值
- 莫隊塊長亂取（q 與 n 不同階時塊長應為 n/√q）、add/del 順序造成區間瞬間為負
- 平衡樹 split 語意混亂（按值 vs 按大小）、FHQ 忘 pushdown 翻轉標記
- 樹剖兩次 DFS 順序寫反；DFS 序 vs 歐拉序 vs 重鏈序混用
- 卡常實務：遞迴改迭代（BIT）、zkw 提及、快讀、O2 與 cache 友善布局

## 4. 選型方法論與離線思維
- 4.1 選型決策表：修改型態 × 查詢型態 × 線上/離線 × n,q 規模 → 章節對照
- 4.2 「資訊設計」四問：維護什麼？怎麼合併？修改怎麼打標記？標記怎麼複合？（以最大子段和、P3373 為範例）
