初學者閱讀指南2.6 線段樹基礎（build／懶標記／資訊設計）

這頁要解決什麼問題？

先問：資料會被怎樣修改？每次又要查哪一段、哪個集合或哪個排名？

本頁聚焦「懶標記三函式分工、先乘後加、最大子段和四元組」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：若每次操作都重新掃過全部資料，單次通常要 O(n)；操作數也接近 n 時，總時間容易變成 O(n²)。

核心直覺

資料結構的工作，是把上一次已整理好的資訊留下來；下一次只修補受影響的部分，而不是全部重算。

閱讀程式前先找不變量

先說清楚每個欄位代表哪一段資料，以及更新前後必須永遠成立的關係。

先猜猜看：什麼時候該想到這頁？

修改與查詢交錯、需要維護集合關係、區間資訊、排名或動態最值。

最小可手算例子

  1. 陣列 [2,1,3,4] 的根存總和 10，左右孩子分別存 3、7。
  2. 查 [2,4] 時拆成葉 [2,2] 與完整節點 [3,4]，得到 1+7。
  3. 區間加 [1,2]+5 時先在 [1,2] 記 lazy，不必立刻改兩個葉。

2.6 線段樹基礎（build／懶標記／資訊設計）

💡 核心想法

把 \([1,n]\) 遞迴對半切成一棵樹，任意區間可拆成 \(O(\log n)\) 個節點——只要資訊可合併（1.2），就能單點改 \(O(\log n)\)、區間查 \(O(\log n)\)。懶標記解鎖區間修改：整段覆蓋的節點「先記帳、不下推」，之後路過時才 pushdown——正確性的關鍵是標記可複合（兩次記帳能合成一次）且標記對資訊的影響可計算（區間加 \(v\) 對和的影響 = \(v\times\)段長）。

🔎 適用特徵
  • 修改與查詢交錯、資訊可合併但不可差分（min/max/最大子段和）——BIT 管不了的都歸它。
  • 區間修改（加、乘、賦值、取 min…）+ 區間查詢。
  • 資訊不可合併（眾數、中位數）→ 別硬上，去 2.10。
模板（C++）：區間加 + 區間和（P3372） 
#define ls (node << 1)
#define rs (node << 1 | 1)
long long sum[N << 2], tag[N << 2];      // 空間 4n！
void pushup(int node) {
    sum[node] = sum[ls] + sum[rs];
}
void apply(int node, int len, long long value) {
    sum[node] += value * len;
    tag[node] += value;
}
void pushdown(int node, int len) {
    if (!tag[node]) {
        return;
    }
    apply(ls, len - len / 2, tag[node]);
    apply(rs, len / 2, tag[node]);
    tag[node] = 0;
}
void update(int node, int left, int right, int query_left, int query_right,
    long long value) {
    if (query_left <= left && right <= query_right) {
        apply(node, right - left + 1, value);
        return;
    }
    int mid = (left + right) >> 1;
    pushdown(node, right - left + 1);
    if (query_left <= mid) {
        update(ls, left, mid, query_left, query_right, value);
    }
    if (query_right > mid) {
        update(rs, mid + 1, right, query_left, query_right, value);
    }
    pushup(node);
}
long long query(int node, int left, int right, int query_left, int query_right) {
    if (query_left <= left && right <= query_right) {
        return sum[node];
    }
    int mid = (left + right) >> 1;
    pushdown(node, right - left + 1);
    long long total = 0;
    if (query_left <= mid) {
        total += query(ls, left, mid, query_left, query_right);
    }
    if (query_right > mid) {
        total += query(rs, mid + 1, right, query_left, query_right);
    }
    return total;
}
★★★洛谷 P3372 線段樹模板一線段樹懶標記

區間加、區間和。

思路

懶標記的第一課。寫的時候給自己立規矩：apply（打標記）／pushdown（下傳）／pushup（上收）三個函式分工明確，update/query 只負責遞迴骨架——之後所有線段樹變形都只改 apply 和 pushup。

★★洛谷 P1531 I Hate It線段樹

單點改、區間 max。LC 307（單點改區間和）同級。

思路

無懶標記的入門版：先把「骨架」寫熟再上標記。順便確認：max 不可差分，BIT 做不了這題（存在魔改 BIT 但複雜度退化）——這就是線段樹的生態位。

★★★★洛谷 P3373 線段樹模板二線段樹多重標記

區間加、區間乘、區間和（mod p）。

思路

多重標記複合的教科書：把標記統一成仿射變換 \(x\mapsto ax+b\)——乘 \(m\)：\((a,b)\to(ma, mb)\)；加 \(v\)：\((a,b)\to(a, b+v)\)。複合有順序（先乘後加），pushdown 一次下傳兩個。搞懂「標記是函數、複合是函數合成」，任何新標記組合都能自己推。

★★★★洛谷 P4513 小白逛公園線段樹資訊設計

單點改、查區間最大子段和。

思路

最大子段和不可直接合併——加維修複：每節點存四元組（總和、最大前綴、最大後綴、最大子段），合併時跨中線的候選 = 左的後綴 + 右的前綴。這就是 1.2 說的「設計複合資訊把不可合併變可合併」，是線段樹從模板走向創作的分水嶺。SPOJ GSS 系列是它的連續劇。

⚠ 高頻坑

① 空間 4n（遞迴切不齊時樹高多一層），開 2n 會 RE 在第 7 個點。② pushdown 忘記清 tag、apply 忘記同時改 sum 和 tag。③ 先乘後加的順序寫反。④ 區間長度奇偶：len - len/2 給左兒子。完整清單見第 3 章。

⏫ 進階去向

下標從「位置」換成「值域」→ 權值線段樹；不先建滿 → 動態開點；兩棵樹合併 → 線段樹合併；插入直線 → 李超樹——全在 2.7。掃描線的「覆蓋計數不下傳」特技在 2.8。

常見誤解與除錯順序

  1. 只背操作模板，卻說不出節點、陣列格或 parent 指標代表什麼。
  2. 混用 0-based、1-based 或閉區間、半開區間，造成 off-by-one。
  3. 只算單次操作，忘記乘上操作次數與節點數，也沒有估算記憶體。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
