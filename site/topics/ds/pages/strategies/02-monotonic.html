初學者閱讀指南2.2 單調堆疊與單調佇列

這頁要解決什麼問題？

先問：資料會被怎樣修改？每次又要查哪一段、哪個集合或哪個排名？

本頁聚焦「維護有用候選人：下一個更大元素、滑窗最值、均攤 O(n)」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：若每次操作都重新掃過全部資料，單次通常要 O(n)；操作數也接近 n 時，總時間容易變成 O(n²)。

核心直覺

資料結構的工作，是把上一次已整理好的資訊留下來；下一次只修補受影響的部分，而不是全部重算。

閱讀程式前先找不變量

先說清楚每個欄位代表哪一段資料，以及更新前後必須永遠成立的關係。

先猜猜看：什麼時候該想到這頁？

修改與查詢交錯、需要維護集合關係、區間資訊、排名或動態最值。

最小可手算例子

  1. 依序讀 [2,1,4,3]；找下一個更大值時，堆疊先放 2。
  2. 1 不可能回答 2，保留；4 到來時連續彈出 1、2，兩者答案都是 4。
  3. 每個元素只進出一次，這就是均攤 O(n) 的來源。

2.2 單調堆疊與單調佇列

💡 核心想法

維護「還有機會成為答案」的候選集合：新元素進來時，把被它支配的舊候選（又舊又差）彈掉——留下的序列自然單調。每個元素進出各一次 ⇒ 均攤 \(O(n)\)。① 單調棧回答「左/右第一個更大（小）元素」；② 單調佇列在此之上多了「隊首過期出隊」，回答滑動窗口最值。

🔎 適用特徵
  • 句型「下一個更大元素」「以 \(i\) 為最小值能延伸多遠」→ 單調棧。
  • 句型「長度 ≤ k 的窗口最值」「決策集合隨 \(i\) 滑動」→ 單調佇列。
  • 它們不是「查詢結構」而是一次掃描的伴生結構——問題必須能重述成一次（或常數次）掃描。
模板（C++）：單調棧 + 單調佇列 
// 單調棧：每個 i 的「左邊第一個更小元素」下標 left[i]
stack<int> st;
for (int i = 1; i <= n; i++) {
    while (!st.empty() && a[st.top()] >= a[i]) {
        st.pop();
    }
    left[i] = st.empty() ? 0 : st.top();
    st.push(i);
}
// 單調佇列：長 k 窗口最大值（遞減佇列，存下標）
deque<int> q;
for (int i = 1; i <= n; i++) {
    while (!q.empty() && q.front() <= i - k) {
        q.pop_front();  // 隊首過期
    }
    while (!q.empty() && a[q.back()] <= a[i]) {
        q.pop_back();  // 隊尾被支配
    }
    q.push_back(i);
    if (i >= k) {
        ans[i - k + 1] = a[q.front()];
    }
}
★★LC 739 每日溫度單調棧

對每天求「幾天後首次更暖」。

思路

「下一個更大元素」的直譯。棧裡存「還沒等到答案的下標」——遞減棧；新元素把能回答的全彈出。洛谷 P5788 是同款模板題。

★★LC 42 接雨水單調棧雙指針

柱狀圖接住多少雨水。

思路

三解名題：按列（每列水位 = min(左max, 右max)−高，前後綴 max 預處理）、單調棧（按層結算：彈棧時「凹槽」成形）、雙指針（空間 O(1)）。單調棧解最能訓練「彈出瞬間結算貢獻」的思維——之後 P4248、LC 84 都是它。

★★★LC 84 柱狀圖中最大的矩形單調棧

柱狀圖內最大矩形面積。

思路

以每根柱子為高度瓶頸：能延伸的範圍 =（左邊第一個更矮, 右邊第一個更矮）之間——單調棧一遍求出兩側邊界。「枚舉最小值 + 求支配區間」是單調棧的招牌句型，字串主題 P4248 的 Σmin 同款。

★★★★LC 85 最大矩形單調棧降維

01 矩陣中全 1 的最大矩形。

思路

逐行維護「以本行為底、每列向上連續 1 的高度」，每行套一次 LC 84——\(O(nm)\)。二維 → 逐行一維化，與 P1719 的降維同思想。

★★洛谷 P1886 滑動窗口單調佇列

長 k 窗口的最小值與最大值序列（\(n\le 10^6\)）。

思路

單調佇列模板（= LC 239）。兩個 while 的順序無所謂，但「隊首判過期用下標、隊尾判支配用值」不能混。\(10^6\) 規模用陣列模擬 deque 更穩。

★★★洛谷 P1725 琪露諾單調佇列DP優化

每步從 \([i-r, i-l]\) 跳到 \(i\)，最大化收集值。

思路

\(f_i=\max_{j\in[i-r,i-l]}f_j + a_i\)：決策窗口隨 \(i\) 平移 → 單調佇列維護窗口 max。與 DP 主題 2.12.2 共用——那邊講「什麼樣的轉移可用單調佇列」，這邊看清楚佇列本身在做什麼。

⚠ 高頻坑

① 「≥ 還是 >」彈棧決定相等元素歸誰——求「嚴格更大」與「第一個不小於」的邊界不同，先想清楚再寫。② 單調佇列存下標不存值（過期判斷需要）。③ 窗口未滿 k 時就輸出答案。

常見誤解與除錯順序

  1. 只背操作模板，卻說不出節點、陣列格或 parent 指標代表什麼。
  2. 混用 0-based、1-based 或閉區間、半開區間，造成 off-by-one。
  3. 只算單次操作，忘記乘上操作次數與節點數，也沒有估算記憶體。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
