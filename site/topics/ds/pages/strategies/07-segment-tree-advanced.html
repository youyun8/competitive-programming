2.7 線段樹進階：可導航、可稀疏、可合併的統計工具

這頁不假設你已經懂「權值化」「動態開點」「樹上二分」或「線段樹合併」。 我們會從四個可手算的小問題出發，先建立直覺，再看正式不變量與可直接使用的 C++ 範本。

這頁要解決什麼問題？

目前插入的數字是 [2, 5, 5, 8]，現在要問第 3 小。每次查詢都重新排序可以得到答案， 但若有十萬次插入與查詢，就會反覆做大量相同工作。真正需要維護的是「每個值出現幾次」。

  • 權值化省掉反覆排序：沿數值範圍的計數找排名。
  • 動態開點省掉巨大空樹：走到某段值域時才建立節點。
  • 樹上二分省掉外層 binary search：直接用子樹資訊導航。
  • 線段樹合併省掉逐元素重插：對齊相同值域的節點合併。
功能解決的痛點範例問題可搭配的功能
權值化要依數值大小統計第 k 小動態開點、樹上二分
動態開點值域太大座標達 10^9權值化、線段樹合併
樹上二分不想做 binary search + query第一個 prefix 達標位置權值化
線段樹合併多棵樹要合成一棵子樹統計、DSU on tree動態開點、權值化

讀者應從這張表理解：四者不是互斥模板。它們分別改變座標的意義、 節點的建立時機、查詢走法與資料合併方式，因此可以自由組合。

核心直覺：線段樹的外形不變，改的是它代表的世界

把線段樹想成一張可以逐層縮小搜尋範圍的地圖。權值化換掉地圖座標的意義； 動態開點只畫真正走過的道路；樹上二分利用路標直接選方向；線段樹合併則把兩張同尺度地圖疊起來。 四種升級都沒有改變「區間對半切、父節點由孩子合成」這個基本骨架。

一、權值化：位置不再代表插入順序，而代表數值

目前插入的數字是 [2, 5, 5, 8]。
現在要問：第 3 小的數字是什麼？

一般陣列下標 0, 1, 2, 3 只表示元素放在哪裡。「下標 1 比下標 3 小」 不代表那裡的數值也較小，所以不能直接拿來找排名。

權值化在處理「我們在意數值大小，而不是原本位置」的問題。做法是換掉座標軸： 座標 5 表示「數值 5」，插入 5 就是 count[5] += 1。因為 5 出現兩次， 它的葉節點計數是 2。建立值域 [1,10] 的線段樹後，每個節點儲存該數值範圍內共有幾個元素。

正式定義與不變量

權值線段樹就是把線段樹的下標／座標軸改成值域。節點區間 [value_left, value_right] 儲存這段值域內的出現次數，並維持：

\(\text{count(node)}=\text{count(left child)}+\text{count(right child)}\)

它適合頻率、排名、第 k 小、第 k 大與 median。值域小時可用普通線段樹； 值域巨大且稀疏時，可再搭配動態開點。

二、動態開點：只替真正走過的路配置節點

值域是 [1, 10^9]，
但實際上只插入 7、42、1000000000 三個數。

普通線段樹通常配置約 4 × V 個節點。當 \(V=10^9\) 時會需要數十 GB， 但實際只用了三個位置。問題在於我們預先替永遠不會走到的區間也建了節點。

動態開點像沒有訪客就不開燈的旅館：根先存在；插入一個值時，只沿包含該值的 二分路徑前進，某段區間第一次被走到才建立節點。

層插入 7 時的區間判斷動作
0[1, 1,000,000,000]7 在左半使用根，往左
1[1, 500,000,000]仍在左半第一次走到，建立節點
2[1, 250,000,000]仍在左半第一次走到，建立節點
…區間持續折半…每層至多新建一個

28[5,8]7 在 [7,8]往右
29[7,8]7 在 [7,7]建立葉節點並加一

值域雖有十億格，單次插入每次都把候選範圍折半，只走約 30 層。

插入 42 時，前段路徑會與 7 共用，分岔後才建立新節點；插入 1,000,000,000 則一路走右側。每次更新最多建立 \(O(\log V)\) 個節點， 所以 \(q\) 次操作的安全空間上界約為 \(O(q\log V)\)。實際通常更少，因為路徑會共用。

方法優點代價適合情況
座標壓縮空間通常 \(O(q)\)，常數小要先知道所有座標；距離資訊可能消失離線、操作可預讀
動態開點保留原值域，可線上加入新座標空間約 \(O(q\log V)\)，常數較大強制線上、座標不可預知、要保留區間長度
值域大不代表一定要動態開點

若所有操作可先讀入，且只比較大小或統計頻率，座標壓縮通常更省空間也更容易除錯。 動態開點的價值在於真正線上或必須保留原始區間。

三、樹上二分：不要在樹外反覆問同一類問題

維護非負陣列 [2, 1, 0, 4, 3, 0, 2, 1]，找最左邊使 prefix_sum >= 6 的位置。前綴和是 [2, 3, 3, 7, 10, 10, 12, 13]， 答案是位置 4（本節採 1-based）。

做法 A：外部 binary search + query

因為元素非負，達標狀態有單調性，可以 binary search 位置；但每試一個中點都要再查 sum(1, middle)。外層試 \(O(\log n)\) 次，每次查詢 \(O(\log n)\)， 總時間是 \(O(\log^2 n)\)。

做法 B：直接看左子樹總和

[1,4]sum = 7 ≥ 6[5,8]sum = 6
[1,2]sum = 3，跳過[3,4]target = 3
[3,3]sum = 0[4,4]答案 4
1
[1,8]

左子樹 [1,4] 的和是 7， 已足以湊到 target 6，所以最左答案在左邊。

2
[1,4]

左子樹 [1,2] 的和只有 3。 跳過這整段，令 target = 6 - 3 = 3，往 [3,4]。

3
[3,4]

左葉 [3,3] 的和是 0， 仍不足 3；往右到 [4,4]，得到答案 4。

每層用已存好的子樹資訊排除半棵樹，因此只走樹高 \(O(\log n)\)。

導航成立需要什麼？

元素非負使 prefix sum 不會下降；節點的區間和又能判斷左側是否已足夠。 更一般地，需要一個能由子樹資訊判斷，且排除半邊後仍成立的條件。

不適用反例

陣列 [5, -10, 10] 的前綴和是 [5, -5, 5]。對 target 4， 達標狀態是 true、false、true，沒有單調性。只存區間總和也會漏掉中途曾達標的位置。 若要支援負數，節點至少要改存「區間總和 + 最大前綴和」，導航條件也要跟著改。

四、線段樹合併：把兩份值域統計接成一份

左子樹收集到的數值：[2, 5, 5]
右子樹收集到的數值：[1, 5, 8]

原始資料若是一棵樹，父節點常要回答整個子樹的第 k 小、眾數或頻率。 若兩個孩子各有一棵權值線段樹，父節點便要把兩份統計合成一份。 逐元素重插會重走很多已存在的路徑。

線段樹合併改成「相同值域對相同值域」遞迴。兩邊都有節點時才深入； 某一邊沒有這段資料時，直接接上另一邊的節點，不需複製或逐元素插入。

合併不是把兩棵樹掛在新根下，而是把代表同一值域的節點疊在一起。 合併後仍是權值線段樹，因此可以直接查第 k 小；本例第 4 小是 5。

  1. A 空、B 存在：直接回傳 B，因為 A 沒有資訊。
  2. A、B 都是同一葉節點：頻率相加。
  3. A、B 都存在且不是葉：分別合併左右孩子，再重新計算目前節點。

為何總複雜度常寫成 \(O(n\log V)\)？動態開點時，每個元素只建立一條 \(O(\log V)\) 路徑，所以全部樹共有 \(O(n\log V)\) 個節點。合併遇到一邊為空就停止； 真正深入時，兩個存在節點會永久合成一個。若合併後不再保留來源版本， 全部合併工作可以用總節點數界住。這是破壞式合併的分析。

方法核心做法適合情況
逐元素重插把一份集合的值逐個插入另一棵樹集合很小、清楚比極致效率重要
線段樹合併對齊相同值域，空側直接接管很多子樹值域統計自底向上匯流
DSU on tree保留大兒子資料，逐個加入小兒子答案能用 add/remove 維護，不一定需要排名
可持久化版本共享未修改節點要保留歷史版本，而不是吃掉來源

程式碼導讀：動態權值線段樹

下列範本同時實作動態開點、第 k 小與破壞式合併。先記住四個不變量：

  • 索引 0 永遠代表空節點，空節點 count 為 0。
  • 非葉節點的 count 等於兩個孩子 count 的和；葉節點 count 是該值的頻率。
  • kth_smallest 前必須保證 1 <= k <= root.count。
  • merge_trees 修改第一棵樹；合併後不可再把第二個 root 當獨立版本。
#include <stdexcept>
#include <vector>

using std::vector;

class DynamicFrequencySegmentTree {
public:
    DynamicFrequencySegmentTree(int minimum_value, int maximum_value)
        : minimum_value_(minimum_value), maximum_value_(maximum_value) {
        nodes_.push_back(Node{});
    }

    void add_value(int& root, int value, int delta) {
        if (value < minimum_value_ || value > maximum_value_) {
            throw std::out_of_range("value is outside the domain");
        }
        add_value_impl(root, minimum_value_, maximum_value_, value, delta);
    }

    int kth_smallest(int root, int k) const {
        if (root == 0 || k < 1 || k > nodes_[root].count) {
            throw std::out_of_range("k is outside the multiset size");
        }
        return kth_smallest_impl(root, minimum_value_, maximum_value_, k);
    }

    int merge_trees(int first_root, int second_root) {
        return merge_impl(
            first_root, second_root, minimum_value_, maximum_value_);
    }

private:
    struct Node {
        int left_child = 0;
        int right_child = 0;
        int count = 0;
    };

    int create_node() {
        nodes_.push_back(Node{});
        return static_cast<int>(nodes_.size()) - 1;
    }

    int node_count(int node_index) const {
        return node_index == 0 ? 0 : nodes_[node_index].count;
    }

    void add_value_impl(
        int& node_index,
        int value_left,
        int value_right,
        int value,
        int delta) {
        if (node_index == 0) {
            node_index = create_node();
        }
        if (value_left == value_right) {
            nodes_[node_index].count += delta;
            if (nodes_[node_index].count < 0) {
                throw std::logic_error("frequency cannot become negative");
            }
            return;
        }

        const int middle = value_left + (value_right - value_left) / 2;
        if (value <= middle) {
            int child = nodes_[node_index].left_child;
            add_value_impl(child, value_left, middle, value, delta);
            nodes_[node_index].left_child = child;
        } else {
            int child = nodes_[node_index].right_child;
            add_value_impl(child, middle + 1, value_right, value, delta);
            nodes_[node_index].right_child = child;
        }
        nodes_[node_index].count =
            node_count(nodes_[node_index].left_child) +
            node_count(nodes_[node_index].right_child);
    }

    int kth_smallest_impl(
        int node_index,
        int value_left,
        int value_right,
        int k) const {
        if (value_left == value_right) {
            return value_left;
        }
        const int left_child = nodes_[node_index].left_child;
        const int left_count = node_count(left_child);
        const int middle = value_left + (value_right - value_left) / 2;
        if (left_count >= k) {
            return kth_smallest_impl(left_child, value_left, middle, k);
        }
        return kth_smallest_impl(
            nodes_[node_index].right_child,
            middle + 1,
            value_right,
            k - left_count);
    }

    int merge_impl(
        int first_node,
        int second_node,
        int value_left,
        int value_right) {
        if (first_node == 0) {
            return second_node;
        }
        if (second_node == 0) {
            return first_node;
        }
        if (value_left == value_right) {
            nodes_[first_node].count += nodes_[second_node].count;
            return first_node;
        }

        const int middle = value_left + (value_right - value_left) / 2;
        nodes_[first_node].left_child = merge_impl(
            nodes_[first_node].left_child,
            nodes_[second_node].left_child,
            value_left,
            middle);
        nodes_[first_node].right_child = merge_impl(
            nodes_[first_node].right_child,
            nodes_[second_node].right_child,
            middle + 1,
            value_right);
        nodes_[first_node].count =
            node_count(nodes_[first_node].left_child) +
            node_count(nodes_[first_node].right_child);
        return first_node;
    }

    int minimum_value_;
    int maximum_value_;
    vector<Node> nodes_;
};
為什麼孩子索引要先複製再寫回？

create_node() 可能讓 vector 擴容，使指向 vector 元素內欄位的 reference 失效。 所以更新時先把孩子索引複製到局部變數，遞迴後再寫回。這是 vector 節點池常見的隱蔽 bug。

複雜度與適用時機：如何辨識題型

操作時間額外空間原因
插入／刪除一個值\(O(\log V)\)最多 \(O(\log V)\) 新節點值域每層折半
第 k 小／prefix 導航\(O(\log V)\)\(O(\log V)\) 遞迴堆疊每層只走一個孩子
外部二分 + query\(O(\log^2 V)\)依 query 實作試 log 次，每次查 log
q 次動態更新\(O(q\log V)\)安全上界 \(O(q\log V)\)每次一條根到葉路徑

看到什麼訊號時該想到它？
  • 「排名、第 k 小、第 k 大、中位數、比 x 小的有幾個」：想到權值化。
  • 座標到 \(10^9\) 或 \(10^{18}\)，操作只有 \(10^5\)：比較座標壓縮與動態開點。
  • 「第一個 prefix 達標」「第 k 個仍存活的位置」：檢查能否樹上導航。
  • 每個原樹節點要統計整個子樹的值域分布：比較線段樹合併與 DSU on tree。
  • 要保留每次插入後的版本：應使用可持久化線段樹，不是破壞式合併。

常見誤解與除錯

  1. 把不同值的種類數當元素數。[2,5,5,8] 有 3 種值，但排名要算 4 個元素。
  2. 往右時忘記扣 left_count。跳過的左側元素都比右側小，必須從 k 扣掉。
  3. 沒有檢查 k。空樹查第 1 小或 4 個元素查第 5 小，會走到空孩子。
  4. 刪除不存在的值。count 變負數後，排名判斷會全面失真。
  5. 中點 overflow。使用 left + (right - left) / 2；到 \(10^{18}\) 時邊界用 64-bit。
  6. 負數仍照抄 prefix sum 導航。[5,-10,10] 已證明達標狀態不單調。
  7. 合併後繼續修改第二棵樹。破壞式合併會共享／接管節點，舊 root 不再是獨立版本。
  8. 節點池容量靠猜。至少用「更新次數 × 最大樹高」估算並留餘裕。

★★洛谷 P1908 逆序對權值統計

★★★★★洛谷 P4556 雨天的尾巴線段樹合併樹上差分

李超樹在這張地圖的位置

李超樹也使用區間二分成樹的外形，但節點存的是函數，不是頻率。它處理「動態加入直線／線段， 查某個 x 的最小值或最大值」，需要另外理解中點比較與兩條直線至多交一次，不能直接把 count 欄位換掉就使用。可先掌握本頁四個正交升級，再到專家專題深入。

你現在應該記得什麼
  • 權值化把座標從元素位置改成數值；count 是值域內的元素總數。
  • 動態開點只建立更新路徑；離線時要先比較通常更省的座標壓縮。
  • 樹上二分每層排除半棵樹，把常見 \(O(\log^2 n)\) 降成 \(O(\log n)\)。
  • 線段樹合併只在兩邊節點都存在時深入，總工作常由 \(O(n\log V)\) 個節點界住。
  • 最常見陷阱是重複值沒計數、往右忘記扣 k、負數破壞單調性與誤用合併後的舊 root。
