初學者閱讀指南2.8 掃描線、二維數點與離線分治（CDQ／整體二分）

這頁要解決什麼問題？

先問：資料會被怎樣修改？每次又要查哪一段、哪個集合或哪個排名？

本頁聚焦「把一個維度變成時間：矩形面積並、三維偏序、整體二分」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：若每次操作都重新掃過全部資料，單次通常要 O(n)；操作數也接近 n 時，總時間容易變成 O(n²)。

核心直覺

資料結構的工作，是把上一次已整理好的資訊留下來；下一次只修補受影響的部分，而不是全部重算。

閱讀程式前先找不變量

先說清楚每個欄位代表哪一段資料，以及更新前後必須永遠成立的關係。

先猜猜看：什麼時候該想到這頁？

修改與查詢交錯、需要維護集合關係、區間資訊、排名或動態最值。

最小可手算例子

  1. 三個點 (1,3),(2,1),(3,2) 依 x 排序後，x 維度已由掃描順序處理。
  2. 掃到每個點時，用 BIT 查目前 y 以下有幾點，再把自己的 y 加入。
  3. 離線的意思是先重排詢問，讓一個維度不再需要資料結構維護。

2.8 掃描線、二維數點與離線分治（CDQ／整體二分）

💡 核心想法

二維問題的離線總綱：把一個維度變成時間。掃描線讓 x 軸流逝、線段樹管 y 軸；二維數點把矩形詢問差分成 4 個「前綴詢問」按 x 排序處理；CDQ 分治用「分治順序」硬造出一個維度的有序性；整體二分把「答案值域」當作分治維度。四者都是同一句話的變奏：離線 = 重排的自由（1.4）。

🔎 適用特徵
  • 矩形面積並/周長並、天際線 → 掃描線。
  • 「數平面上矩形內的點」「\(l\le i\le r\) 且 \(a_i\le x\) 的個數」→ 二維數點。
  • 三維偏序、「帶時間的二維問題」→ CDQ。
  • 一大堆「第 k 小」型詢問 → 整體二分（主席樹的離線替身）。

2.8.1 掃描線：矩形面積並

每個矩形拆成左邊界（+1）右邊界（−1）兩個事件，按 x 排序；線段樹維護 y 軸「被覆蓋 ≥1 的總長」。特技：覆蓋計數標記永不下傳——節點存（本段覆蓋次數 cnt, 子樹內被蓋長度 len），cnt>0 時 len = 整段，否則 = 兒子之和。行得通是因為 +1/−1 嚴格配對、查詢只查全域。

★★★★洛谷 P5490 掃描線模板掃描線線段樹

n 個矩形的面積並。

思路

y 離散化成 \(O(n)\) 段（線段樹葉子是「段」不是「點」——經典 off-by-one 窩點）；答案 = Σ 覆蓋長 ×（相鄰事件的 x 距離）。把「標記不下傳為什麼對」講明白，才算真的會這題。

★★★LC 218 天際線問題掃描線multiset

建築輪廓的關鍵點。

思路

掃描線的輕量版：事件排序後用 multiset 維護「目前活躍高度集合」，最大值變化時輸出關鍵點。體會掃描線的本質是事件驅動，y 軸結構可按需求換（multiset/堆/線段樹）。

2.8.2 二維數點

★★★洛谷 P2163 [SHOI2007] 園丁的煩惱二維數點BIT

百萬級點與矩形詢問：每個矩形內幾個點。

思路

矩形差分成 4 個前綴詢問 \(Q(x,y)\)＝「\(x_i\le x, y_i\le y\) 的點數」；點與詢問混排按 x 掃，權值 BIT 管 y。管線背熟：差分 → 排序 → BIT，這是所有離線二維統計的母版。

2.8.3 CDQ 分治：三維偏序

★★★★洛谷 P3810 三維偏序（陌上花開）CDQBIT

對每個元素數「三維都 ≤ 它」的元素個數。

思路

第一維排序解決；CDQ 分治：算左半對右半的貢獻時，兩半各按第二維有序（歸併），左半元素按序插入 BIT（第三維），右半按序查詢。「分治 + 歸併」憑空造出一維有序，於是三維退化為一維 BIT——「用分治換一層資料結構」的口號由此而來。注意去重（完全相同的元素互相貢獻）。

2.8.4 整體二分

★★★★★洛谷 P1527 [國家集訓隊] 矩陣乘法整體二分BIT

多次詢問子矩陣第 k 小。

思路

單詢問可以二分答案 + 二維計數；整體二分把所有詢問一起二分：solve(值域[lo,hi], 詢問集 Q) 中把 ≤ mid 的元素插入二維 BIT，按計數把 Q 劈成兩半遞迴。每層總工作量 \(O((n+q)\log)\)，共 \(\log V\) 層。與主席樹（2.12）互為離線/線上替身——主席樹難以二維化，整體二分無壓力，這正是離線的紅利。

🧭 四兄弟怎麼選

幾何形狀的並/覆蓋 → 掃描線；純計數且矩形可差分 → 二維數點；「修改也在時間軸上」（帶修改的二維統計）→ CDQ 把修改當點、詢問當區間；答案本身要二分 → 整體二分。共同前提：能離線。強制線上 → 可持久化（2.12）或樹套樹（2.13.6）。

常見誤解與除錯順序

  1. 只背操作模板，卻說不出節點、陣列格或 parent 指標代表什麼。
  2. 混用 0-based、1-based 或閉區間、半開區間，造成 off-by-one。
  3. 只算單次操作，忘記乘上操作次數與節點數，也沒有估算記憶體。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
