1理論基礎：維護增量的思考框架

🧪 一道題如何被「操作分析」拆解

「\(q\) 次操作：把 \(a_l..a_r\) 都加 \(v\)；或問 \(a_l..a_r\) 的和」。第一步列配對：修改 =（區間, 加法），查詢 =（區間, 求和）。第二步問合併性：和可以由左右兩半的和相加——可合併。第三步問修改的表達：整段加 \(v\) 對「和」的影響 \(= v\times\)段長，可以先記帳不下推——標記可複合。三個答案分別對應：需要樹狀結構、線段樹可用、懶標記可用 → P3372。所有資料結構選型都是這三問的排列組合，而不是背「看到 X 用 Y」。

1.1 操作分析法

要問的問題答案決定什麼
修改是什麼型態？（單點／區間／插入刪除／換樹形）BIT ↔ 線段樹 ↔ 平衡樹 ↔ LCT 的分界
查詢的資訊可合併嗎？（結合律）能上樹狀結構，還是只能分塊/莫隊
修改對資訊的影響能整段記帳嗎？懶標記是否存在、怎麼複合
詢問能離線嗎？掃描線/莫隊/CDQ/整體二分全家桶解禁
\(n, q\) 多大、時限多少？\(\log\) 幾層、能不能 \(\sqrt{}\)、常數預算

1.2 可合併性：結構的入場券

資訊 \(f\) 可合併 ⟺ 存在 \(\oplus\)：\(f([l,r]) = f([l,m]) \oplus f([m+1,r])\)。和、min/max、gcd、矩陣乘、「最大子段和四元組」都可以；區間眾數、區間中位數不可以（左右兩半的眾數推不出整段眾數）——這條線直接把題目分成「線段樹世界」與「根號世界」（2.10）。設計複合資訊（如四元組）把不可合併變可合併，是線段樹題的核心創意點（2.6.4）。

1.3 均攤與勢能：看懂「為什麼不會超時」

  • 單調棧：每元素進棧出棧各一次 → 總量 \(O(n)\)，單次可能 \(O(n)\) 但均攤 \(O(1)\)。
  • DSU 路徑壓縮 + 按秩：\(O(\alpha(n))\) 近乎常數（結論記住即可）。
  • 勢能法：定義勢能 \(\Phi\)，昂貴操作使 \(\Phi\) 大幅下降、便宜操作最多讓它 +1。Segment Tree Beats（2.13.1）與 ODT 的複雜度都靠它——★★★★★ 之前只需會「單調棧式」的存款論證。

1.4 離線 vs 線上：最重要的一次自由選擇

💡 離線 = 重排的自由

已知全部詢問時，你可以換一個順序處理它們：按 x 掃（掃描線）、按塊排（莫隊）、按時間分治（CDQ、線段樹分治）、倒著做（把刪除變插入，P1197）。離線常常把需要的結構降一級——帶修第 k 小，線上要樹套樹，離線整體二分只要 BIT。讀完題第一件事：圈出「強制線上」有沒有出現。

1.5 複雜度預算

資料範圍（n, q 同階）預算典型武器
\(10^6\)\(O(n\log n)\) 且常數小BIT、排序、單調結構；線段樹要小心常數
\(10^5 \sim 5\times10^5\)\(O(n\log n)\)～\(O(n\log^2 n)\)線段樹、平衡樹、樹剖（\(\log^2\)）、CDQ
\(10^5\)，統計不可合併\(O(n\sqrt n)\)分塊、莫隊
\(n\le 10^5\)，時限寬\(O(n\sqrt n\log n)\) 邊緣塊內二分、帶值域的根號做法——先估常數再動手

常數經驗值：BIT 比線段樹快 4～8 倍；遞迴線段樹每秒約 \(10^7\sim10^8\) 次操作；莫隊的 \(\sqrt{}\) 常數極小、幾乎跑不滿。

1.6 實作工具箱

  • 離散化三行：sort; unique; lower_bound——查詢用到的值也要一起離散化（第 3 章高頻坑）。
  • 1-indexed 慣例：BIT 必須（lowbit(0)=0 死迴圈）；線段樹、樹上結構建議跟進，全站模板統一。
  • 靜態陣列 > vector：節點池 int ch[N][2] 比指標/vector 快且好清；N 的估算寫成註解（線段樹 4n、動態開點 q log V）。
  • 快讀：\(10^6\) 級輸入用 scanf／手寫 getchar 快讀；cin 要關同步。
  • 對拍腳手架：隨機操作序列 + 暴力陣列——資料結構是對拍收益最高的題型（第 4 章附完整範例）。
