初學者閱讀指南學習路線圖

這頁要解決什麼問題？

先問：資料會被怎樣修改？每次又要查哪一段、哪個集合或哪個排名？

本頁聚焦「四階段學習計畫，每階段附代表題與畢業檢定」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：若每次操作都重新掃過全部資料，單次通常要 O(n)；操作數也接近 n 時，總時間容易變成 O(n²)。

核心直覺

資料結構的工作，是把上一次已整理好的資訊留下來；下一次只修補受影響的部分，而不是全部重算。

閱讀程式前先找不變量

先說清楚每個欄位代表哪一段資料，以及更新前後必須永遠成立的關係。

先猜猜看：什麼時候該想到這頁？

修改與查詢交錯、需要維護集合關係、區間資訊、排名或動態最值。

6學習路線圖

🎯 路線圖的使用方式

資料結構的成長曲線是「模板 → 對拍 → 變形 → 組合」：每學一個結構，先默寫模板並通過一萬組對拍（4.4），再做 2–3 道變形題，最後在更高階段回頭把它當零件用。星級是參考；畢業檢定過不了就留在原階段。搭配全站題目總表的 ds 篩選刷題。

第一階段 ★–★★：零結構技巧與兩件輕武器（約 2 週）

目標：前綴和/差分/ST 表、單調棧/佇列、堆、DSU、BIT 的模板全部能默寫。

  • 靜態三件套：LC 303/304 → P2367 → P3397 → P3865 → P1719
  • 單調結構：LC 739 → P1886 → LC 42 → LC 84
  • 堆：P3378 → LC 295
  • DSU：P3367 → P1551 → P1892
  • BIT：P3374 → P3368 → P1908 → LC 315

✅ 畢業檢定：P1908 用歸併與 BIT 各寫一遍；說明 BIT 為什麼做不了區間 min。

第二階段 ★★★：線段樹成型與離線初體驗（約 1 個月）

目標：線段樹（含懶標記）閉眼寫對；LCA/樹上差分上手；第一次體會「離線 = 自由」。

  • 線段樹：P1531 → P3372 → P3373（先乘後加）→ LC 307
  • 樹上：P3379（倍增 LCA）→ P3128（樹上差分）→ P3178（DFS 序）
  • 離線思維：P1197（倒序 DSU）→ P1955（離散化）→ SPOJ DQUERY（兩解）
  • 堆進階：P1801 → LC 480；掃描線輕量版：LC 218

✅ 畢業檢定：不看模板寫出 P3373 並通過對拍；口頭回答 4.2 的資訊設計四問（以區間加乘為例）。

第三階段 ★★★★：重武器庫（約 2–3 月）

目標：平衡樹、主席樹、樹剖、掃描線、CDQ、莫隊、線段樹進階全部入庫；一題能報出兩種以上解法。

  • 線段樹進階：P4513（資訊設計）→ P4097（李超）→ 動態開點練習
  • 掃描線與離線分治：P5490 → P2163 → P3810（CDQ）
  • 平衡樹：P3369 → P3391 → P1486
  • 可持久化：P3919 → P3834 → P4735
  • 樹上：P3384（樹剖）→ CF 600E（DSU on tree）
  • 根號：P2801 → P1494 → LOJ 分塊入門 1–9
  • BIT 進階：P5094 → BIT 上倍增

✅ 畢業檢定：P3834（主席樹）與 P3810（CDQ）獨立完成；對任一題能先寫出 4.1 的操作規格書再動手。

第四階段 ★★★★★：組合與勢能（長期）

目標：結構的「組合」成為直覺；勢能分析看得懂、講得出。

  • 組合拳：P4556（樹上差分×線段樹合併）→ P2617（BIT×主席樹）→ P1527（整體二分）
  • 勢能系：P6242（吉司機）→ CF 896C（ODT）→ AT 歷史研究（回滾莫隊）
  • 時間軸：P5787（線段樹分治×可撤銷 DSU）
  • 動態樹：P3690（LCT）；序列平衡樹收尾：P2596
  • NOIP 壓軸重溫：P3960 列隊、P1903 帶修莫隊

✅ 畢業檢定：面對 CF 2400+ 資結題，10 分鐘內完成操作分析表＋兩套候選方案（線上/離線各一）；能向別人講清楚 Segment Tree Beats 的勢能論證。

常見誤解與除錯順序

  1. 只背操作模板，卻說不出節點、陣列格或 parent 指標代表什麼。
  2. 混用 0-based、1-based 或閉區間、半開區間，造成 off-by-one。
  3. 只算單次操作，忘記乘上操作次數與節點數，也沒有估算記憶體。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
