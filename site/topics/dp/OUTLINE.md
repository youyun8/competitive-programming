## 1. 理論基礎：DP 思考框架
- 1.1 三大前提：重疊子問題、最優子結構、無後效性（各附一個反例）
- 1.2 解題六步驟：狀態定義 → 轉移方程 → 初始化與邊界 → 計算順序 → 答案位置 → 優化
- 1.3 記憶化搜尋 vs 遞推：優缺點對照（跳過無效狀態 vs 便於優化）、爆棧與手動展開
- 1.4 複雜度 = 狀態數 × 單次轉移成本；從資料範圍反推狀態設計
  - n ≤ 20 → 狀壓 2ⁿ；n ≤ 500 → 區間 O(n³)；n ≤ 5000 → O(n²)；n ≤ 10⁶ → O(n log n) / O(n)
- 1.5 實作工具箱：滾動陣列、下標偏移、-INF 哨兵、記錄決策點還原方案、取模細節

### 2.1 線性 DP（入門三件套與 LIS 家族）
- 核心：狀態沿序列一維推進，「以 i 結尾」vs「前 i 個」兩種視角
- 2.1.1 一維遞推入門
  - 爬樓梯（LC 70）★、使用最小花費爬樓梯（LC 746）★
- 2.1.2 「選 / 不選」決策
  - 打家劫舍（LC 198）★、環形打家劫舍（LC 213，拆環為兩條鏈）★★
- 2.1.3 最大子段和（Kadane）
  - LC 53 ★、環形子陣列最大和（LC 918，正反兩用）★★
- 2.1.4 LIS 家族
  - 最長遞增子序列（LC 300）：O(n²) → 貪心+二分 O(n log n) ★★
  - 導彈攔截（洛谷 P1020）：Dilworth 定理（最少不上升覆蓋 = LIS）★★★

- 2.1.5 值域當狀態：Boredom（CF 455A）★★

## 3. 常見錯誤與除錯（假 DP 教材）
- 後效性翻車：狀態漏資訊，轉移依賴「未來」——附最短路不能沿下標遞推的反例
- 0/1 背包正序枚舉 → 不小心變完全背包（一題兩寫對照）
- 「恰好裝滿」初始化：0 與 −INF 的語意差異
- 記憶化搜尋的狀態不完整：答案依賴額外全域量卻沒進狀態
- 貪心誤用邊界：帶權區間排程 / 0/1 背包不能貪心（連回貪心指南第 3 章）
- 工程坑：爆 int、取模出負數、多測未清空、記憶化陣列初始值撞答案值
- 除錯流程：印 DP 表、小資料暴力對拍、二分定位出錯的轉移
