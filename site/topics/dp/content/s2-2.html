2.2 網格與座標 DP

目標常用轉移不可達格初值
路徑數前驅方案數相加0起點 = 1
最小代價前驅取 min 再加本格+INF起點 = 本格代價
最大收益前驅取 max 再加本格−INF起點 = 本格收益
多人同步同一步數下枚舉每人的前驅越界／衝突依題意處理所有人同在起點
⚠ 網格不等於一定能 DP

真正的判準是移動關係能否排成拓撲序。只能右／下時，i+j 每步嚴格增加，所以依賴無環；四方向自由移動時，同一格可能繞回，應改成 BFS／Dijkstra。若題目另給「步數 k」且每次 k 增加，則即使座標能回頭，狀態 (k,i,j) 仍可能無環。初始化時推薦在表外放哨兵（最小化放 +INF、最大化放 −INF），避免第一行第一列充滿分支特判。

💡 核心想法

狀態就是座標 (i, j)，轉移來自固定方向的鄰居（上方、左方）。只要移動方向不構成環（只能往右/往下），網格天然就是一張 DAG，按行列順序填表即可。

🔎 適用特徵
  • 在矩陣上從角落走到角落，統計路徑數或最優路徑值。
  • 轉移方向固定；若能四方向移動則不是 DP 而是最短路（見 第 3 章）。
★LC 62 不同路徑網格計數

m×n 網格從左上走到右下（只能右/下），路徑數？

思路

dp[i][j] = dp[i−1][j] + dp[i][j−1]，邊界第一行第一列全為 1。有障礙的 LC 63 只是把障礙格設 0。組合數 C(m+n−2, m−1) 是它的封閉解——DP 與組合計數常常互為驗算。

程式碼（C++） 
vector<long long> dp(n, 1);
for (int i = 1; i < m; i++)
    for (int j = 1; j < n; j++) dp[j] += dp[j-1];   // 滾動：dp[j] 即上方，dp[j-1] 即左方
return dp[n-1];
★LC 64 最小路徑和網格

同樣的走法，最小化路徑上數字總和。

思路

把計數的「+」換成最值的「min」：dp[i][j] = min(dp[i−1][j], dp[i][j−1]) + a[i][j]。計數與最值共用同一套狀態，只差運算子——這個對應關係全書通用。洛谷 P1216 數字三角形是同型入門題。

★★★LC 174 地下城遊戲網格倒推

騎士從左上走到右下救公主，路上有加血/扣血，血量任何時刻不得 ≤ 0，求最小初始血量。

思路

無後效性的實戰課。正著推需要同時記「目前血量」與「歷史最低點」，狀態糾纏；倒著定義就解耦了：dp[i][j] = 從 (i,j) 出發活著到終點所需的最小血量 = max(1, min(dp[i+1][j], dp[i][j+1]) − a[i][j])。正推卡住時，試著把狀態定義成「從這裡出發」而非「走到這裡」。

程式碼（C++） 
for (int i = m-1; i >= 0; i--)
    for (int j = n-1; j >= 0; j--) {
        long long need = min(dp[i+1][j], dp[i][j+1]) - a[i][j]; // 邊界外設 INF、終點外側設 1
        dp[i][j] = max(1LL, need);
    }
★★★洛谷 P1004 方格取數網格雙路徑

從左上到右下走兩趟，取過的數變 0，最大化兩趟總和。

思路

兩趟改成兩人同時走：走 k 步後兩人都在反對角線 i+j=k 上，狀態 dp[k][x1][x2]（y 由 k−x 算出）——四維壓成三維。重疊格只算一次。同型題「傳紙條」（P1006）完全同構。「多趟」問題幾乎都該轉成「多人同步」，否則第一趟的選擇對第二趟有後效性。

程式碼（C++） 
for (int k = 2; k <= 2*n; k++)
  for (int x1 = 1; x1 <= min(k-1, n); x1++)
    for (int x2 = x1; x2 <= min(k-1, n); x2++) {
      int y1 = k - x1, y2 = k - x2;
      if (y1 < 1 || y1 > n || y2 < 1 || y2 > n) continue;
      int gain = a[x1][y1] + (x1 == x2 ? 0 : a[x2][y2]);
      dp[k][x1][x2] = max({dp[k-1][x1][x2],   dp[k-1][x1-1][x2],
                           dp[k-1][x1][x2-1], dp[k-1][x1-1][x2-1]}) + gain;
    }
