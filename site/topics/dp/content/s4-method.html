寫程式前固定填六格：① \(\mathrm{dp}[\text{維度}]\) 的完整中文定義；② 這個狀態包含哪些選項範圍；③ 最後一步有哪些互斥情況；④ 不可達值與空狀態；⑤ 依賴的偏序／計算順序；⑥ 狀態數 \(\times\) 每格決策數。任何一格空白都先不要進實作。若兩條不同歷史到同一狀態後，未來合法選擇不同，立即把能區分它們的最小資訊加成新維度。

4.1 從暴力遞迴到 DP 的機械化流程

不會做的 DP 題，永遠從暴力遞迴開始——狀態設計不是靈感，是重構：

  1. 寫暴力：定義 \(\mathrm{solve}(\text{完整描述目前局面})\) 回傳答案，枚舉當下所有決策遞迴。
  2. 削參數：檢查每個參數——未來用不到的刪掉；能由其他參數導出的刪掉（交錯字串裡 \(C\) 的下標是 \(i+j\)）。
  3. 找重疊：剩下的參數組合會不會重複出現？會 → 加 memo，DP 完成。
  4. 需要時改遞推：要滾動陣列或轉移優化時，把記憶化搜尋翻成 for 迴圈。
💡 示範一：打家劫舍

暴力為 \(\mathrm{solve}(i,\text{上一家偷了嗎})\)。削參數：「上一家偷了嗎」只影響「這一家能不能偷」，可以併進轉移，比較 \(\mathrm{dp}[i-2]+a_i\) 與 \(\mathrm{dp}[i-1]\)——參數剩一個 \(i\)，重疊顯然，完工。

💡 示範二：戳氣球

暴力為 \(\mathrm{solve}(\text{剩餘氣球集合})\)，有指數個狀態。換枚舉對象：不枚舉「第一個戳誰」而枚舉「最後一個戳誰」，剩餘集合永遠是連續區間，因此參數變成 \((l,r)\)，狀態數為 \(O(n^2)\)。卡在狀態爆炸時，先懷疑「枚舉的方向」而不是硬壓縮。

4.2 狀態設計四視角

視角觸發訊號代表
前 \(i\) 個（前綴）決策從左到右、答案只問總體背包、股票、編輯距離
以 \(i\) 結尾轉移需要知道「最後一個是誰／在哪」LIS、Kadane、值域 DP
區間 \([l,r]\)操作發生在相鄰元素之間、子區間獨立石子合併、戳氣球、博弈取數
集合 \(S\)\(n \le 20\)、順序任意、每個元素用一次TSP、配對計數、集合覆蓋

再各加一組常用「附加維」：處理到第幾行＋輪廓（棋盤）、目前節點＋子樹已合併（樹）、匹配到模式串第幾位（自動機）、用了幾次特權（分層）。

4.3 降維與資訊壓縮技巧清單

  • 滾動陣列：只依賴上一排 → 兩排交替；0/1 背包型 → 原地倒序。
  • 導出量不進狀態：能由其他維算出的（例如 \(i+j\) 是常數、\(j=k-i\)）刪掉一維。
  • 相對量代替絕對量：排列計數記「相對名次」（AT DP T）；差值代替兩個絕對位置。
  • 對稱性合併：等價元素只記個數不記身分（Sushi 的 \(\mathrm{dp}[a][b][c]\)）。
  • 值域與下標互換：價值範圍小、重量範圍大時，改用 \(\mathrm{dp}[\text{價值}]=\text{最小重量}\)（經典逆轉）。
  • 模一個共同量：整除類條件只需模最小公倍數（CF 55D 的 \(2520\)）。
  • 凸性/單調性外包：整個函數當 DP 值，用堆/平衡樹維護（slope trick）。

4.4 對拍驗證器（C++）

DP 最幸福的一點：暴力遞迴天然就是正確性基準。方程一寫完就對拍，別等大測資 WA。

long long brute_force(const std::vector<int>& values, int index) {
    if (index < 0) {
        return 0;
    }
    return std::max(brute_force(values, index - 1),
                    brute_force(values, index - 2) + values[index]);
}

long long solve_dp(const std::vector<int>& values) {
    long long two_steps_back = 0;
    long long one_step_back = 0;
    for (const int value : values) {
        const long long current =
            std::max(one_step_back, two_steps_back + value);
        two_steps_back = one_step_back;
        one_step_back = current;
    }
    return one_step_back;
}

int main() {
    std::mt19937 random_engine(123456789);
    std::uniform_int_distribution<int> size_distribution(0, 8);
    std::uniform_int_distribution<int> value_distribution(-5, 20);
    for (int test = 0; test < 100000; ++test) {
        const int size = size_distribution(random_engine);
        std::vector<int> values(size);
        for (int& value : values) {
            value = value_distribution(random_engine);
        }
        const long long brute_answer =
            brute_force(values, static_cast<int>(values.size()) - 1);
        const long long dp_answer = solve_dp(values);
        if (brute_answer != dp_answer) {
            std::cout << "Counterexample found\n";
            return 0;
        }
    }
    std::cout << "100000 tests passed\n";
    return 0;
}

要點：測資小而刁（長度 \(0\) 到 \(8\)、含負數／零／重複）比大測資更會抓 bug；找到反例後把它縮到最小再除錯。
