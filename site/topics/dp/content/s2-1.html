2.1 線性 DP（入門三件套與 LIS 家族）

狀態視角典型定義答案位置代表題
前 i 個dp[i] = 前 i 個元素的完整答案dp[n]爬樓梯、打家劫舍
必須以 i 結尾dp[i] = 以 a[i] 作最後元素的答案max(dp[i])LIS、最大子段和
值域作狀態best[x] = 結尾值為 x 時的最優解值域取最優值域小或可用 BIT 優化的 LIS 變形
🧠 選狀態的判斷法

如果只知道「前 i 個的最優值」就足以接上第 i+1 個，用前綴狀態；如果未來能否接元素取決於最後值、最後位置或最後類型，就把它固定在狀態裡。例：LIS 若只存前 i 個的最長長度，無法判斷 a[i+1] 能否接上，因為長度相同的序列可能有不同尾值；改成「以 i 結尾」後，尾值就是 a[i]，轉移才無後效。空間滾動前也要先確認右式只依賴固定幾個較早狀態。

💡 核心想法

狀態沿著序列一維推進。兩種最常用的視角：「前 i 個的答案」（答案在 dp[n]）與「以 i 結尾的答案」（答案是所有 dp[i] 取 max）。分不清用哪個時，看轉移是否需要「最後一個元素是誰」——需要就用「以 i 結尾」。

🔎 適用特徵
  • 輸入是一條序列，決策按下標從左到右進行，做完第 i 個不會回頭改前面。
  • 每一步只有常數種選擇（選/不選、接/不接）。
★LC 70 爬樓梯線性

每次爬 1 或 2 階，爬到 n 階有幾種走法？

思路

dp[i] = 到第 i 階的走法數。最後一步要嘛從 i−1 跨 1 階、要嘛從 i−2 跨 2 階：dp[i] = dp[i−1] + dp[i−2]——就是斐波那契。這題示範了轉移方程的萬用推導法：枚舉最後一步。

程式碼（C++） 
int a = 1, b = 1;  // dp[0], dp[1]
for (int i = 2; i <= n; i++) {
    int c = a + b;
    a = b;
    b = c;
}
return b;  // 滾動到只剩兩個變數
★LC 198 打家劫舍線性選/不選

不能偷相鄰兩家，最大化偷竊總額。

思路

dp[i] = 只考慮前 i 家的最大收益。第 i 家不偷：dp[i−1]；偷：dp[i−2] + a[i]。取 max。「選/不選」是線性 DP 最基本的決策形式，環形版 LC 213 拆成「不偷第 1 家」與「不偷最後一家」兩條鏈各做一次。

程式碼（C++） 
int prev2 = 0, prev1 = 0;
for (int x : nums) {
    int cur = max(prev1, prev2 + x);
    prev2 = prev1;
    prev1 = cur;
}
return prev1;
★LC 53 最大子陣列和（Kadane）線性以 i 結尾

找和最大的連續子陣列。

思路

「以 i 結尾」視角的代表作：dp[i] = 以 a[i] 結尾的最大子段和 = max(dp[i−1], 0) + a[i]（前綴是負的就丟掉重開）。答案 = max 全部 dp[i]。環形版 LC 918：答案是「普通最大子段和」或「總和 − 最小子段和」二者取大。

程式碼（C++） 
int cur = 0, best = INT_MIN;
for (int x : nums) {
    cur = max(cur, 0) + x;
    best = max(best, cur);
}
return best;
★★LC 300 最長遞增子序列（LIS）線性LIS二分

求最長嚴格遞增子序列長度。

思路

O(n²)：dp[i] = 以 i 結尾的 LIS = max{dp[j]} + 1（j < i 且 a[j] < a[i]）。O(n log n)：維護陣列 g，g[k] = 「長度 k+1 的遞增子序列的最小可能結尾」——g 單調遞增，每個 a[i] 用 lower_bound 替換第一個 ≥ 它的位置。這是「DP + 貪心維護決策集合」的第一課，也是 BIT 優化轉移（2.12.8）的入門素材。

程式碼（C++） 
vector<int> g;
for (int x : nums) {
    auto it = lower_bound(g.begin(), g.end(), x);  // 非嚴格遞增改 upper_bound
    if (it == g.end())
        g.push_back(x);
    else
        *it = x;
}
return g.size();
★★★洛谷 P1020 導彈攔截LISDilworth

第一問：一套系統最多攔幾發（最長不上升子序列）；第二問：最少幾套系統攔完所有導彈。

思路

第二問是 Dilworth 定理：把序列劃分成最少的不上升鏈，等於最長嚴格上升子序列的長度。兩問都用 O(n log n) 的 g 陣列寫法。這題把「LIS 不只是求長度，還是覆蓋/劃分的對偶」講透了。

★★洛谷 P1091 合唱隊形LIS

留下一個先升後降的子序列，最少踢掉幾人？

思路

對每個 i 求「以 i 結尾的 LIS」f[i] 與「以 i 開頭的最長下降子序列」g[i]（反著做一次 LIS），答案 = n − max(f[i] + g[i] − 1)。正反各掃一遍再拼接是線性 DP 的常用組合技。

★★★LC 354 俄羅斯套娃信封LIS排序鍵設計

信封 (w, h) 嚴格雙維遞增才能嵌套，最多套幾層？

思路

二維偏序化一維：按 w 遞增、w 相同時 h 遞減排序，再對 h 做 LIS。h 遞減保證同 w 的信封不會互相接龍——排序鍵設計錯（h 也遞增）是本題最經典的 WA。

★★CF 455A Boredom線性值域當狀態

取走一個值為 v 的元素得 v 分，但所有 v−1、v+1 的元素被刪除，最大得分？

思路

對值域而非下標做 DP：cnt[v] = 值 v 的出現次數，dp[v] = max(dp[v−1], dp[v−2] + v·cnt[v])——變成值域上的打家劫舍。「對什麼做 DP」本身就是設計決策：下標、值域、時間軸都可以是階段。
