2.9 數位 DP

🔍 數位 DP 在數什麼？

把區間問題改成前綴函數 solve(x) = [0,x] 中合法數量，答案為 solve(R)-solve(L-1)。從高位到低位填數字，狀態至少含位置 pos 與描述限制所需的資訊；tight 表示目前前綴是否仍等於上界，started 表示是否已放過非前導零。只有 tight=false 的狀態能跨不同 x 的分支安全記憶化。

參數作用常見錯法
pos目前填到哪一位位數方向與 base case 不一致
tight本位上限是 digit[pos] 還是 9下一層寫成 d==limit，忘了還要且上原 tight
started區分前導零與數字中的 0把 0007 的零當真實數位計數
問題狀態前一位、餘數、出現集合、自動機節點等漏掉會影響未來合法性的歷史
⚠ 0 是否算入答案？

DFS 全程都不 started 的分支代表數字 0。它是否符合題意要在終點明確決定，不能靠前導零轉移碰運氣。含負區間也要先把正、負部分拆開，再依數位性質是否對絕對值對稱處理。

💡 核心想法

統計區間 [l, r] 內「數位滿足某性質」的數：差分成 solve(r) − solve(l−1)，然後從最高位往低位逐位填數字，狀態記「數位相關的性質」＋兩面旗：tight（前綴是否貼著上界，貼著則本位最多填到上界位）與 lead(前導零，尚未開始的數不該計入性質)。

🔎 適用特徵
  • 上界大到 10¹⁸（不能枚舉），但條件只跟數位有關：含/不含某數字、數位和、相鄰數位關係、整除性。
  • 複雜度 = 位數 × 性質狀態數 × 10，幾乎永遠跑得飛快。
// 統一模板（記憶化搜尋版）：只 cache「自由態」(!tight && !lead) 的結果
int digits[20], len;
long long memo[20][STATE];
bool vis[20][STATE];
long long dfs(int pos, int st, bool tight, bool lead) {
    if (pos == len)
        return 合法(st, lead) ? 1 : 0;
    if (!tight && !lead && vis[pos][st])
        return memo[pos][st];
    int hi = tight ? digits[pos] : 9;
    long long res = 0;
    for (int d = 0; d <= hi; d++)
        res +=
            dfs(pos + 1, 轉移(st, d, lead), tight && d == hi, lead && d == 0);
    if (!tight && !lead) {
        vis[pos][st] = true;
        memo[pos][st] = res;
    }
    return res;
}
// 答案 = solve(r) - solve(l - 1)
★★★洛谷 P2657 windy 數數位

相鄰數位之差至少為 2 的數，在 [a, b] 中有幾個？

思路

模板直出：狀態 = 上一位數字（前導零時記「無」）。lead 旗在這題是必要的：007 的「0 → 0」不能算違規，因為數還沒開始。適合當第一道自己寫的數位 DP。

★★★洛谷 P2602 數字計數數位計數

統計 [a, b] 中每個數字 0–9 各出現多少次。

思路

對每個目標數字跑一次，狀態 = 目前已出現次數（或直接把「出現次數」當返回值的一部分：返回 (數的個數, 出現總次數) 二元組）。LC 233（數字 1 的個數）是單數字版。「統計出現次數」而非「統計個數」時，讓 dfs 同時返回兩個量最不易錯。

★★★LC 902 最大為 N 的數字組合數位

只能用給定數字集合，能拼出多少個 ≤ N 的正整數？

思路

tight 旗的純粹練習：貼上界時本位只能填 ≤ 對應位的數字，且「嚴格小於」的瞬間之後全自由（自由位直接乘 |D| 的冪）。位數比 N 短的數另外加總。

★★★AtCoder DP S - Digit Sum數位

1..K（K ≤ 10^10000）中數位和被 D 整除的數有幾個？

思路

狀態 = 數位和 mod D。特別之處是 K 長達一萬位——數位 DP 的複雜度只跟位數線性相關，天生就是為這種上界準備的。記得答案要扣掉 0。

★★★★CF 55D Beautiful Numbers數位狀態壓縮

統計能被自己每個非零數位整除的數。

思路

「被每個出現的數位整除」⟺ 被它們的 lcm 整除。關鍵壓縮：1–9 任意組合的 lcm 都整除 2520，所以只需記（數 mod 2520、已出現數位的 lcm）。lcm 只有 48 種，離散化後狀態 20×2520×48。「看似要記完整資訊，實則只需 mod 某個共同倍數」是數位 DP 最漂亮的壓縮範式。
