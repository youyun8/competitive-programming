2.6 狀態機 DP

✏️ 先畫圖，再寫式子

每個節點是一種「處理完前 i 個元素後的身分」，每條邊標上本步行動與收益；next[to] = max(next[to], old[from] + gain) 就是完整模板。股票題的節點可為空手、持股、冷凍；字串限制題的節點可為已匹配的模式長度。初始時只有起始節點可達，其餘設 −INF；答案只在題目允許的終止節點中取最優。

實作檢查原因
本輪一律從 old 轉到 next避免同一天連走多條本不允許的邊
把「交易次數」定義成買入或賣出其中一種兩種定義都可，但轉移與答案必須一致
不可達狀態設 −INF不能讓「憑空持股」等非法歷史參與 max
狀態數固定時可用矩陣表示轉移步數極大可接矩陣快速冪
💡 核心想法

把「目前處於哪種模式」顯式列成有限狀態機：dp[i][state]，轉移方程就是自動機的邊。適合「同一時刻你只能處於少數幾種身分」的問題——持股/空手、健康/冷凍期、匹配到模式串第幾位。

🔎 適用特徵
  • 決策合法性取決於「目前身分」：買了才能賣、冷凍期不能買。
  • 能畫出一張節點 ≤ 10 個的狀態轉移圖。
  • 模式是「匹配進度」時，自動機可以是 KMP/AC 自動機（見 2.6.4）。
★★LC 121/122 股票 I・II狀態機

單筆交易（121）／不限次數（122）的最大利潤。

思路

兩個狀態：hold（持股）、free（空手）。hold[i] = max(hold[i−1], free[i−1] − p[i])，free[i] = max(free[i−1], hold[i−1] + p[i])。121 只是把買入時的 free[i−1] 換成 0（只准一筆）。整個股票系列共用這張兩節點狀態機，差別只在邊上的限制。

程式碼（C++） 
int hold = INT_MIN, free_ = 0;
for (int p : prices) {
    hold  = max(hold, free_ - p);
    free_ = max(free_, hold + p);
}
return free_;
★★★LC 309 股票含冷凍期狀態機

賣出後隔天不能買。

思路

狀態機加一個節點：hold / free（可買）/ cooldown（剛賣）。cooldown[i] = hold[i−1] + p[i]；free[i] = max(free[i−1], cooldown[i−1])。規則變了就改圖，不用重新發明方程——狀態機 DP 的可維護性就在這裡。含手續費的 LC 714 只是在賣出邊扣 fee。

★★★LC 188 股票 IV（至多 k 筆）狀態機wqs 預告

至多完成 k 筆交易。

思路

交易次數進狀態：dp[i][j][hold]，j ≤ k。k 很大時 k ≥ n/2 直接退化成 LC 122。更漂亮的是 wqs 二分：給每筆交易加罰款 λ，去掉次數維後二分 λ 使最優交易數恰為 k（2.12.6）——「恰好 k 個」約束的通用消去術。

★★★★洛谷 P2569 股票交易（SCOI2010）狀態機單調佇列

每天有買/賣的價格與數量上限，交易間隔至少 W 天，最大化最終獲利。

思路

dp[i][j] = 第 i 天持股 j 的最大現金。從第 i−W−1 天轉移：買入 dp[i][j] = max(dp[i−W−1][k] + (k−j)·買價)（k < j）。把式子整理成「與 k 有關的部分 + 與 j 有關的部分」，對 k 的 max 用單調佇列維護——狀態機 × 轉移優化的標準組合技（2.12.2）。

★★LC 935 騎士撥號器狀態機矩陣冪預告

騎士在電話鍵盤上跳 n−1 步，能撥出幾種號碼？

思路

把鍵盤當自動機：dp[i][v] = 第 i 步停在鍵 v 的方案數 = Σ dp[i−1][u]（u 可跳到 v）。轉移矩陣固定 → n 到 10¹⁸ 時直接上矩陣快速冪（2.12.7）。

★★★★★洛谷 P3193 GT 考試（KMP + 矩陣冪）狀態機KMP 自動機矩陣冪

長 n（≤ 10⁹）的數字串，不含長 m（≤ 20）的禁止串 X 的方案數。

思路

狀態 = 「目前匹配到 X 的第幾位」（KMP 自動機節點，共 m 個）。dp[i][j] → dp[i+1][j']：對每個追加數字，用 KMP 失配函數算出新匹配位置 j'，禁止走到 j'=m。轉移矩陣 m×m 與 i 無關 → 矩陣快速冪 O(m³ log n)。字串自動機（KMP/AC 自動機）× DP × 矩陣冪是計數硬題的黃金三件套。
