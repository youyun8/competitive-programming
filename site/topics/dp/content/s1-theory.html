1理論基礎：DP 思考框架

🧪 一個方程如何從無到有：打家劫舍

先定義 \(\mathrm{dp}[i]\) 為只看前 \(i\) 家能得到的最大金額。最優解對第 \(i\) 家只有兩種互斥情況：不偷它，答案是 \(\mathrm{dp}[i-1]\)；偷它，第 \(i-1\) 家不能偷，答案是 \(\mathrm{dp}[i-2]+a_i\)。兩類覆蓋所有合法方案且不重疊，所以 \(\mathrm{dp}[i]=\max\left(\mathrm{dp}[i-1],\mathrm{dp}[i-2]+a_i\right)\)。正確性可按 \(i\) 歸納：假設較短前綴已最優，任一前 \(i\) 家的最優方案依「是否含 \(i\)」必落入上述其中一類，取較大者即全域最優。狀態定義、分類最後決策、證明分類完整就是大多數 DP 推導的骨架。

檢查點必須說清楚的句子說不清時的警訊
狀態dp[...] 精確代表哪些選項範圍與哪種答案同一格可能代表兩種歷史
轉移用哪個「最後決策」把所有方案互斥分類漏情況或重複計數
順序右式的每個狀態為何已經算好依賴成環或原地更新污染
邊界空集合／空前綴的答案為何是該初值靠特判勉強跑過樣例
複雜度合法狀態有幾個、每格枚舉幾個決策只看迴圈層數卻漏掉轉移成本

1.1 三大前提

性質意義缺了會怎樣
重疊子問題不同決策路徑到達相同的子問題不重疊就退化成分治——記憶化毫無收益
最優子結構大問題的最優解由子問題的最優解組成例：經過中繼點的「最長簡單路徑」不能拼接子答案
無後效性到達某狀態後，未來只取決於狀態本身，與「怎麼來的」無關轉移依賴歷史 → 把缺的資訊加進狀態（見 第 3 章反例）
💡 無後效性是狀態設計的唯一準繩

「狀態設計」說穿了就是回答一個問題：要讓未來與歷史無關，我最少得記住哪些資訊？記太少 → 轉移算不出來（後效性）；記太多 → 狀態爆炸。整個第 2 章的 13 種策略，本質上是 13 種「該記什麼」的答案。

1.2 解題六步驟

  1. 狀態定義：\(\mathrm{dp}[\ldots]\) 的每一維是什麼、值是什麼（最值？方案數？可行性？）。用一句完整的中文寫下來。
  2. 轉移方程：枚舉「最後一步決策」，把狀態表成更小狀態的函數。
  3. 初始化與邊界：哪些狀態是遞迴終點；不可達狀態設 −INF／0 的語意要想清楚。
  4. 計算順序：保證算 \(\mathrm{dp}[s]\) 時它依賴的狀態都已算完（線性掃、按區間長度、拓撲序、記憶化搜尋自動處理）。
  5. 答案位置：可能是單一格、一整排取 max、或若干狀態的組合。
  6. 優化：空間滾動、轉移加速（2.12）。先寫對，再寫快。

1.3 記憶化搜尋 vs 遞推

記憶化搜尋（top-down）遞推（bottom-up）
寫法暴力遞迴 + cache，思路直翻for 迴圈按順序填表
優點不用想計算順序；自動跳過無效狀態（數位 DP、樹上 DP 最愛）方便滾動陣列、方便單調佇列/斜率等轉移優化
缺點遞迴常數大、深度 \(10^6\) 級會爆棧要自己保證順序正確、會算到用不到的狀態

建議：先用記憶化搜尋把方程想對，需要優化時再改遞推。兩者只是同一張 DAG 的兩種遍歷方向。

1.4 複雜度 \(=\) 狀態數 \(\times\) 單次轉移

這條公式同時是估時間與反推狀態設計的工具。看到資料範圍先心算：

n 的量級可接受複雜度優先聯想
\(n \le 20\)\(O(2^n n)\)狀壓 DP（2.7）
\(n \le 500\)\(O(n^3)\)區間 DP（2.4）、Floyd
\(n \le 5000\)\(O(n^2)\)雙序列、\(O(n^2)\) 線性 DP
\(n \le 10^5\text{--}10^6\)\(O(n \log n)\)／\(O(n)\)轉移必須優化：單調佇列、斜率、BIT（2.12）
\(n \le 10^{18}\)\(O(k^3 \log n)\)矩陣快速冪加速遞推（2.12.7）、數位 DP（2.9）

1.5 實作工具箱

  • 滾動陣列：\(\mathrm{dp}[i][*]\) 只依賴 \(\mathrm{dp}[i-1][*]\) 時，兩排交替或原地倒序（0/1 背包）。
  • 下標偏移：狀態含負數（差值、座標）時整體平移，別用 map 拖慢常數。
  • \(-\infty\) 哨兵：用 0x3f3f3f3f 級別的值，夠大又不會相加溢位；判斷不可達用 dp[x] < -kInfinity / 2。
  • 路徑還原：另開 from[] 記錄每格取 max 時的決策點，答案處倒著走回去。
  • 取模：每次加法/乘法後取模；相減後 (x % MOD + MOD) % MOD 防負數。
// 記憶化搜尋通用骨架
long long memo[MAXN];
bool vis[MAXN];
long long f(int s) {
    if (是邊界)
        return 邊界值;
    if (vis[s])
        return memo[s];
    vis[s] = true;  // 用 vis 而非特殊值，避免「初始值撞答案」
    long long best = -INF;
    for (每個決策 d)
        best = max(best, f(更小狀態) + 決策收益);
    return memo[s] = best;
}
