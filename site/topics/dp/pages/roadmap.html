初學者閱讀指南學習路線圖

這頁要解決什麼問題？

先問：暴力遞迴會重複計算哪些相同子問題？要記住哪些資訊，才能讓未來決策不需要回頭看完整歷史？

本頁聚焦「四階段學習計畫，每階段附代表題與畢業檢定」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：直接枚舉所有選擇常形成指數級遞迴樹；同一個剩餘問題會從不同路徑被算很多次。

核心直覺

DP 像填表：每格只保存『未來真正需要知道的摘要』，並從已算好的較小問題轉移過來。

閱讀程式前先找不變量

寫轉移前先用一句完整中文定義 dp 狀態，包含範圍、是否必選、恰好或至多等條件。

先猜猜看：什麼時候該想到這頁？

求最優值、方案數或可行性，而且問題能由較小範圍、較少物品或較短前綴組成。

6學習路線圖

🎯 路線圖的使用方式

每一階段都用「讀懂 → 關掉頁面默寫 → 口頭證明 → 變形題 → 對拍」五步完成。只 AC 題目不代表掌握：至少要能在空白編輯器寫出狀態定義、初始化、轉移順序與複雜度，並說出一個模板會失效的反例。時間只是參考；畢業檢定過不了就留在原階段補洞。

每題練習紀錄應留下的內容
模型為什麼是這類 DP，而不是貪心／最短路／搜尋
狀態一句完整定義與無後效性的理由
轉移最後決策的完整分類
邊界空狀態與不可達狀態的語意
複雜度狀態數、單次轉移與實際資料上界
復盤最小反例、第一次錯因、可遷移的變形

第一階段 ★–★★：建立狀態直覺（約 2 週）

目標：能用一句話定義狀態、寫出轉移；分得清「前 \(i\) 個」與「以 \(i\) 結尾」。

  • 線性入門：LC 70 → LC 198 → LC 53 → LC 300（先 \(O(n^2)\) 再二分版）
  • 網格：LC 62 → LC 64 → 洛谷 P1216 數字三角形
  • 背包兩件套：採藥（P1048）、疯狂的采药（P1616）——親手體會枚舉方向那一字之差
  • 計數初體驗：LC 96、LC 494

✅ 畢業檢定：獨立寫出 LC 300 的 \(O(n\log n)\) 解，並解釋 \(g\) 陣列的含義。

第二階段 ★★★：五大模板成型（約 1 個月）

目標：區間 / 雙序列 / 樹上 / 狀壓 / 數位五大骨架能默寫；記憶化搜尋與遞推自由切換。

  • 區間：AtCoder DP N → 石子合併（P1880）→ LC 312 戳氣球
  • 雙序列：LC 1143 → LC 72 → P1439（LCS 轉 LIS）
  • 樹上：P1352 舞會 → 樹的直徑 → 選課（P2014）
  • 狀壓：AtCoder DP O → TSP（P1171）→ 互不侵犯（P1896）
  • 數位：windy 數（P2657）→ 數字計數（P2602）
  • 狀態機 + 期望開胃：LC 309、綠豆蛙的歸宿（P4316）、AtCoder DP J

✅ 畢業檢定：AtCoder Educational DP Contest A–T 全部獨立 AC（U–Z 留到下一階段）。

第三階段 ★★★★：優化技術與組合技（約 1–2 月）

目標：看到 \(O(n^2)\) 轉移能判斷用哪種優化；換根、期望進階、狀壓進階不再現場發明。

  • 優化四連：AtCoder DP M（前綴和）→ 琪露諾（P1725，單調佇列）→ 玩具裝箱（P3195，斜率）→ Post Office（POJ 1160，Knuth）
  • 單調佇列實戰：股票交易（P2569）、多重背包 \(O(nW)\)（P1776 重做）
  • 換根：P3478 → CF 1187E → AtCoder DP V
  • 狀壓進階：炮兵陣地（P2704）、憤怒的小鳥（P2831）、蒙德里安（POJ 2411）
  • 期望進階：OSU!（P1365）、換教室（P1850）、收集郵票（P4550）
  • 雜項武器：分層圖（P4568）、BIT 優化計數（CF 597C）、bitset 背包

✅ 畢業檢定：拿一道沒見過的切段代價題，從零推出斜率優化全過程（式子、凸殼方向、隊首隊尾操作）。

第四階段 ★★★★★：專家專題（長期）

目標：CF 2400+ / NOI 級的 DP 有系統的切入路徑，而不是拼運氣。

  • 掃描線狀態：蒙德里安（重讀，輪廓線視角）→ 插頭 DP 模板（P5056）
  • 指數域變換：SOS DP（CF 165E）→ 高維前綴和的計數應用
  • 消去約束維：wqs 二分——LC 188 重做 → 林克卡特樹（P4383）
  • 值域掃描：連通塊插入 DP（Kangaroo，P5999）
  • 可修改 DP：動態 DP 模板（P4719）
  • 複合狀態：GT 考試（P3193，自動機×矩陣冪）→ DP 套 DP（遊園會，P4590）
  • 凸性：slope trick（CF 713C）——與貪心指南 2.11.2 對讀

✅ 畢業檢定：對一道陌生的「恰好 \(k\) 個」最優化題，能先驗證凸性、再完整實作 wqs 二分（含相同值的邊界處理）。

常見誤解與除錯順序

  1. 狀態定義少了條件，導致不同歷史被錯誤合併。
  2. 初始化把不可達狀態設成 0，讓不存在的方案參與轉移。
  3. 迴圈順序與轉移依賴相反，讀到本輪剛更新的值或尚未計算的值。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
