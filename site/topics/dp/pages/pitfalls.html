初學者閱讀指南常見錯誤與除錯（假 DP 教材）

這頁要解決什麼問題？

先問：暴力遞迴會重複計算哪些相同子問題？要記住哪些資訊，才能讓未來決策不需要回頭看完整歷史？

本頁聚焦「後效性、背包方向、恰好裝滿、狀態不完整——高頻坑與修法」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：直接枚舉所有選擇常形成指數級遞迴樹；同一個剩餘問題會從不同路徑被算很多次。

核心直覺

DP 像填表：每格只保存『未來真正需要知道的摘要』，並從已算好的較小問題轉移過來。

閱讀程式前先找不變量

寫轉移前先用一句完整中文定義 dp 狀態，包含範圍、是否必選、恰好或至多等條件。

先猜猜看：什麼時候該想到這頁？

求最優值、方案數或可行性，而且問題能由較小範圍、較少物品或較短前綴組成。

3常見錯誤與除錯（假 DP 教材）

答案症狀優先檢查最小測資方向
答案總是偏大非法狀態被當合法、0/1 背包正序、重複計數單一物品／單一方案
答案總是偏小漏轉移、完全背包倒序、答案格讀錯必須重複使用同一選項
只有邊界錯空輸入、第一列、恰好／至多語意\(n=0\)、\(n=1\)、容量 \(0\)
小測對、大測錯溢位、遞迴深度、複雜度、取模負數極值與長鏈
改迴圈順序就忽對忽錯原地更新讀到了本輪資料先用未壓縮二維版對拍
🔬 用「不變量」除錯，而不是盯最終答案

在每層迴圈結束後，寫下陣列此刻應代表什麼，例如「處理完前 \(i\) 件物品後，\(\mathrm{dp}[j]\) 是容量至多 \(j\) 的最優值」。拿最小測資逐格驗證這句話第一次失效的位置；錯誤一定發生在那一輪的初始化、轉移或更新順序。這通常比在完整程式裡單步追蹤快得多。

DP 的 bug 高度模式化。這一章按「發生頻率 × 殺傷力」排序，每個坑附最小反例與修法。

3.1 後效性翻車：狀態漏了資訊

⚠ 症狀

方程看起來很順，小測資就 WA。反例：網格圖四方向移動求最小路徑和，寫 \(\mathrm{dp}[i][j]=\min(\text{上},\text{左})+a\)——但最優路徑可能繞回頭（往上、往左走），\(\mathrm{dp}[i][j]\) 依賴 \(\mathrm{dp}[i+1][j]\)，依賴成環。修法：依賴成環就不是 DP 填表，是最短路（Dijkstra／SPFA）；或把缺的資訊（方向、已用步數）加進狀態把環拆掉。判斷口訣：畫出依賴箭頭，有環就重設狀態。

3.2 背包枚舉方向寫反

⚠ 症狀

① 0/1 背包容量正序枚舉 → 同一件物品被重複選，答案偏大（悄悄變成完全背包）。② 完全背包倒序 → 每件只選一次，答案偏小。兩份程式只差一個 for 的方向，卻是兩個不同的問題——見 2.3。③ 多測時背包陣列忘記清空是同級高頻坑。

3.3 「恰好裝滿」初始化錯誤

⚠ 症狀

求「恰好裝滿」時把 DP 全初始化成 \(0\)，不可達容量會被當成「價值 \(0\) 的合法方案」參與轉移，答案錯。修法：\(\mathrm{dp}[0]=0\)，其餘設為 \(-\infty\)；計數版則是 \(\mathrm{dp}[0]=1\)，其餘為 \(0\)。負無限要取夠大但不溢位的值（例如 0x3f3f3f3f），判不可達用 dp[x] < -kInfinity / 2，因為哨兵在轉移中可能被加減過。

3.4 記憶化搜尋的狀態不完整

⚠ 症狀

dfs 的返回值還依賴某個沒進參數的全域量（目前路徑、剩餘次數），cache 卻只按參數存 → 第二次命中時抄了錯的答案。修法：返回值的所有依賴都必須是參數（= 狀態）；「初始值撞答案值」（用 0 當「沒算過」但答案本身可為 0）用獨立的 vis 陣列解決。

3.5 該 DP 卻用了貪心（或反之）

⚠ 症狀

帶權區間排程「按結束時間能選就選」的反例是一個大權區間對三個小權區間；0/1 背包按性價比排序的反例是用 \(\{1,3,4\}\) 湊容量 \(6\)。修法：貪心需要交換論證，寫不出來就老實 DP；反方向，可以貪心的題硬上 DP 通常只是慢，不會錯——不確定時 DP 是安全側。另一個主題貪心指南第 3 章有完整的反例清單。

3.6 工程坑清單

  • 爆 int：計數取模前的乘法、路徑和上限 \(n\cdot\max A\)，先估上界再定型別。
  • 取模出負數：退背包、容斥的減法後 (x % MOD + MOD) % MOD。
  • 多測未清空：memo、vis、鄰接表——清空範圍只到「本測資用到的大小」，避免 \(O(T\cdot\mathrm{MAXN})\)。
  • 浮點期望比較：期望 DP 輸出精度用 printf("%.10f")，中途別過早四捨五入。
  • 遞迴爆棧：樹退化成鏈 + 記憶化搜尋時要手動開棧或改遞推；CF 上 \(10^6\) 深度必炸。

3.7 除錯流程（照做就好）

  1. 印出 DP 表：小輸入下把整張表印出來，跟手算對——九成的錯在第一排/第一列就看得到。
  2. 寫暴力對拍：暴力遞迴（不記憶化）+ 隨機小測資，逐一比對（模板見 4.4）。找到最小反例後手動追蹤那條轉移。
  3. 檢查三件事：初始化語意（0 還是 −INF）、枚舉順序（依賴的都算完了嗎）、答案位置（要不要對整排取 max）。

常見誤解與除錯順序

  1. 狀態定義少了條件，導致不同歷史被錯誤合併。
  2. 初始化把不可達狀態設成 0，讓不存在的方案參與轉移。
  3. 迴圈順序與轉移依賴相反，讀到本輪剛更新的值或尚未計算的值。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
