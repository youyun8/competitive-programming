初學者閱讀指南2.9 數位 DP

這頁要解決什麼問題？

先問：暴力遞迴會重複計算哪些相同子問題？要記住哪些資訊，才能讓未來決策不需要回頭看完整歷史？

本頁聚焦「tight/lead 兩面旗的統一模板，windy 數到 mod 2520 壓縮」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：直接枚舉所有選擇常形成指數級遞迴樹；同一個剩餘問題會從不同路徑被算很多次。

核心直覺

DP 像填表：每格只保存『未來真正需要知道的摘要』，並從已算好的較小問題轉移過來。

閱讀程式前先找不變量

寫轉移前先用一句完整中文定義 dp 狀態，包含範圍、是否必選、恰好或至多等條件。

先猜猜看：什麼時候該想到這頁？

求最優值、方案數或可行性，而且問題能由較小範圍、較少物品或較短前綴組成。

最小可手算例子

  1. 計算 0～25 中不含數字 2 的個數，從十位開始逐位決定。
  2. 若目前位選得比上界小，tight 變 false，後面可自由選 0～9。
  3. 若仍貼住上界，下一位最多只能選上界對應數字。

2.9 數位 DP

🔍 數位 DP 在數什麼？

把區間問題改成前綴函數 \(\mathrm{solve}(x)\)，表示 \([0,x]\) 中合法數量，答案為 \(\mathrm{solve}(R)-\mathrm{solve}(L-1)\)。從高位到低位填數字，狀態至少含位置 pos 與描述限制所需的資訊；tight 表示目前前綴是否仍等於上界，started 表示是否已放過非前導零。只有 tight == false 的狀態能跨不同 \(x\) 的分支安全記憶化。

參數作用常見錯法
pos目前填到哪一位位數方向與 base case 不一致
tight本位上限是 \(\mathrm{digit}[\mathrm{pos}]\) 還是 \(9\)下一層寫成 d == limit，忘了還要且上原 tight
started區分前導零與數字中的 0把 0007 的零當真實數位計數
問題狀態前一位、餘數、出現集合、自動機節點等漏掉會影響未來合法性的歷史
⚠ 0 是否算入答案？

DFS 全程都不 started 的分支代表數字 \(0\)。它是否符合題意要在終點明確決定，不能靠前導零轉移碰運氣。含負區間也要先把正、負部分拆開，再依數位性質是否對絕對值對稱處理。

💡 核心想法

統計區間 \([l,r]\) 內「數位滿足某性質」的數：差分成 \(\mathrm{solve}(r)-\mathrm{solve}(l-1)\)，然後從最高位往低位逐位填數字，狀態記「數位相關的性質」＋兩面旗：① tight（前綴是否貼著上界，貼著則本位最多填到上界位）與 ② lead（前導零，尚未開始的數不該計入性質）。

🔎 適用特徵
  • 上界大到 \(10^{18}\)（不能枚舉），但條件只跟數位有關：含／不含某數字、數位和、相鄰數位關係、整除性。
  • 複雜度為 \(O(\text{位數}\times\text{性質狀態數}\times10)\)，幾乎永遠跑得飛快。
// 統一模板（記憶化搜尋版）：只 cache「自由態」(!tight && !lead) 的結果
int digits[20], len;
long long memo[20][STATE];
bool vis[20][STATE];
long long dfs(int pos, int state, bool tight, bool lead) {
    if (pos == len) {
        return 合法(state, lead) ? 1 : 0;
    }
    if (!tight && !lead && vis[pos][state]) {
        return memo[pos][state];
    }
    int max_digit = tight ? digits[pos] : 9;
    long long result = 0;
    for (int digit = 0; digit <= max_digit; digit++) {
        result +=
            dfs(pos + 1, 轉移(state, digit, lead), tight && digit == max_digit,
                lead && digit == 0);
    }
    if (!tight && !lead) {
        vis[pos][state] = true;
        memo[pos][state] = result;
    }
    return result;
}
// 答案 = solve(r) - solve(l - 1)
★★★洛谷 P2657 windy 數數位

相鄰數位之差至少為 \(2\) 的數，在 \([a,b]\) 中有幾個？

思路

模板直出：狀態是上一位數字（前導零時記「無」）。lead 旗在這題是必要的：007 的前兩個零不能算違規，因為數還沒開始。適合當第一道自己寫的數位 DP。

★★★洛谷 P2602 數字計數數位計數

統計 \([a,b]\) 中每個數字 \(0\) 到 \(9\) 各出現多少次。

思路

對每個目標數字跑一次，狀態是目前已出現次數；也可直接把「出現次數」當返回值的一部分，回傳 \((\text{數的個數},\text{出現總次數})\) 二元組。LC 233（數字 \(1\) 的個數）是單數字版。「統計出現次數」而非「統計個數」時，讓 DFS 同時返回兩個量最不易錯。

★★★LC 902 最大為 N 的數字組合數位

只能用給定數字集合，能拼出多少個不超過 \(N\) 的正整數？

思路

tight 旗的純粹練習：貼上界時本位只能填不超過對應位的數字，且「嚴格小於」的瞬間之後全自由（自由位直接乘 \(|D|\) 的冪）。位數比 \(N\) 短的數另外加總。

★★★AtCoder DP S - Digit Sum數位

\(1\) 到 \(K\)（\(K\le10^{10000}\)）中數位和被 \(D\) 整除的數有幾個？

思路

狀態是數位和模 \(D\)。特別之處是 \(K\) 長達一萬位——數位 DP 的複雜度只跟位數線性相關，天生就是為這種上界準備的。記得答案要扣掉 \(0\)。

★★★★CF 55D Beautiful Numbers數位狀態壓縮

統計能被自己每個非零數位整除的數。

思路

「被每個出現的數位整除」等價於被它們的最小公倍數整除。關鍵壓縮：\(1\) 到 \(9\) 任意組合的最小公倍數都整除 \(2520\)，所以只需記 \((\text{數}\bmod2520,\text{已出現數位的最小公倍數})\)。最小公倍數只有 \(48\) 種，離散化後狀態數為 \(20\times2520\times48\)。「看似要記完整資訊，實則只需模某個共同倍數」是數位 DP 最漂亮的壓縮範式。

常見誤解與除錯順序

  1. 狀態定義少了條件，導致不同歷史被錯誤合併。
  2. 初始化把不可達狀態設成 0，讓不存在的方案參與轉移。
  3. 迴圈順序與轉移依賴相反，讀到本輪剛更新的值或尚未計算的值。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
