2.13 專家專題（★★★★★）

專題先備能力真正要壓縮的東西
輪廓線／插頭 DP狀壓、括號匹配、雜湊掃描線與已處理區域的連通介面
SOS DP子集操作、容斥對所有子集／超集的聚合查詢
連通塊插入 DP排序、組合計數具體排列壓成連通塊個數
動態 DP樹 DP、矩陣／半環、鏈剖修改後只重算受影響的轉移乘積
slope trick凸函數、堆整個分段線性函數只存斜率轉折點
DP 套 DP自動機、狀態壓縮把「內層演算法的運行狀態」當外層狀態
⚠ 這頁不是可直接套用的六份短模板

每個專題都依賴不容易從十行程式碼看出的狀態不變量。建議先能手推最小案例，再閱讀題卡；首次實作保留狀態編碼／解碼與暴力版本做對拍。若基礎模型尚未能獨立推導，先回 2.7、2.8、2.12，否則只會記住脆弱的程式外形。

💡 這一章在講什麼

五個把 DP 推到極限的方向：狀態壓到「格點級」（輪廓線/插頭）、對指數狀態整體做變換（SOS）、按值域重構問題（連通塊插入）、讓 DP 支援修改（DDP）、把另一個 DP 當作狀態（DP 套 DP）。每一個都值得單獨開一週專題。

2.13.1 輪廓線 DP 與插頭 DP

★★★★POJ 2411 蒙德里安的夢想（骨牌鋪滿）輪廓線

用 \(1\times2\) 骨牌鋪滿 \(n\times m\) 棋盤，求方案數。

思路

逐格轉移（不是逐行）：狀態是掃描線上 \(m\) 個格子的「突出」情況（輪廓線）。每格三選一：豎放（在輪廓上留一個突出）、橫放、不放（吃掉上方突出）。行寬 \(m\le20\) 時，以 \(2^m\) 個狀態逐格滾動。行輪廓 DP（互不侵犯）看的是「整行」，輪廓線把粒度細化到「格」，狀態利用率高得多。

★★★★★洛谷 P5056 插頭 DP 模板（迴路計數）插頭

統計棋盤上經過所有可用格的哈密頓迴路數。

思路

輪廓線上不只記「有沒有突出」，還記插頭的配對關係（哪兩個插頭屬於同一條路徑段）——用括號表示法（左括號／右括號／無）做三進位壓縮。轉移分「延伸、合併、新開」三類，合併左右括號時要防提前成環。用雜湊表只存出現過的狀態。競賽 DP 的技術巔峰之一，第一次寫建議對照模板逐行理解。

2.13.2 SOS DP（Sum over Subsets / 高維前綴和）

★★★★CF 165E Compatible NumbersSOS

對每個 \(a_i\) 找任一 \(a_j\)，使 \(a_i\mathbin{\&}a_j=0\)。

思路

「與 \(a_i\) 相容」等價於「是 \(\mathord{\sim}a_i\) 的子集」。SOS 模板是對每一位做一次鬆弛 f[S] |= f[S ^ (1 << b)]（若 \(S\) 含第 \(b\) 位），用 \(O(2^n n)\) 算出「每個 \(S\) 的所有子集中是否存在元素」。本質是 \(n\) 維超立方體上的前綴和——比 \(3^n\) 的子集枚舉快一個量級。計數／最值版同模板。

程式碼（C++） 
// f[S] = S 的子集中任一個存在的元素（0 表示無）
for (int b = 0; b < n; b++)
    for (int S = 0; S < (1 << n); S++)
        if ((S >> b & 1) && !f[S] && f[S ^ (1 << b)])
            f[S] = f[S ^ (1 << b)];

2.13.3 連通塊插入 DP

★★★★★洛谷 P5999 Kangaroo（CEOI 2016）連通塊插入計數

數「相鄰兩步方向必須交替（之字形）」且首尾固定的排列個數。

思路

換一個維度掃描：按值從小到大插入，狀態 \(\mathrm{dp}[i][j]\) 表示插完前 \(i\) 個值、目前形成 \(j\) 個「連通段」的方案數。插入新值時可以新開一段（\(j+1\)）、接在某段端點，或合併兩段（\(j-1\)），係數由組合計數決定。之字形約束恰好由「按值插入」自動滿足。此技巧通吃一大類「相鄰差約束的排列計數」（CF 1515E、AtCoder dwango 的同型題）。

2.13.4 動態 DP（DDP）

★★★★★洛谷 P4719 動態 DP 模板DDP樹鏈剖分

樹上最大權獨立集，帶 \(m\) 次單點改權。

思路

把樹 DP 的轉移寫成廣義矩陣乘法，例如 \((\min,+)\) 或 \((\max,+)\) 半環，只要結合律成立即可。重鏈剖分後每條鏈的轉移是矩陣連乘，用線段樹維護乘積；單點修改只動 \(O(\log^2 n)\) 個矩陣。「DP + 修改」的通解：轉移代數化 + 資料結構維護。全域平衡二叉樹可壓到 \(O(n\log n)\)。

2.13.5 凸優化視角：slope trick

★★★★★CF 713C 嚴格遞增最小代價slope trick

每次加減 \(1\) 花 \(1\) 元，把序列改成嚴格遞增的最小花費。

思路

\(\mathrm{dp}_i(x)\) 表示前 \(i\) 項且第 \(i\) 項改成 \(x\) 的最小代價。整個函數關於 \(x\) 是凸的分段線性函數，只用一個大根堆維護「左半斜率的轉折點」。取前綴最小值等於砍掉右半；加上 \(|x-a_i|\) 等於插入兩個轉折點。嚴格遞增可先做 a[i] -= i 化成不降。這是「函數整體當 DP 值」的入門，與貪心指南的 slope trick 章相互印證。

2.13.6 DP 套 DP

★★★★★洛谷 P4590 遊園會（TJOI 2018）DP 套 DP

對每個 \(k\) 統計：與給定串 \(B\) 的 LCS 恰為 \(k\)，且不含 NOI 的串 \(A\) 的個數。

思路

內層：LCS 的一整排 \(\mathrm{dp}[0\ldots m]\) 相鄰差只有 \(0/1\)，因此差分後可壓成 \(2^m\) 的位元狀態，LCS 自動機就此誕生。外層用 \(\mathrm{dp}[i][\text{自動機狀態}][\text{NOI 匹配位}]\) 計數。「把內層 DP 的整排值壓縮成外層的狀態」就是 DP 套 DP 的定義；見過一次，之後所有「統計與模式串的 DP 值恰為 \(k\) 的字串數」都是同一招。
