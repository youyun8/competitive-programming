2.4 區間 DP

💡 核心想法

dp[l][r] = 區間 [l, r] 的答案，由更短的子區間合併而來。計算順序按區間長度遞增枚舉（或記憶化搜尋）。轉移通常是枚舉分割點 k：dp[l][r] = min/max over k of dp[l][k] + dp[k+1][r] + 合併代價，複雜度 O(n³)。

🔎 適用特徵
  • 操作發生在相鄰元素之間（合併、消除、插入），使得子區間彼此獨立。
  • n ≤ 500 左右（O(n³) 的容忍範圍）。
  • 元素順序不能重排——能重排的合併問題往往是 Huffman 貪心。
★★★洛谷 P1880 石子合併（環形）區間環形

環形排列的石子堆，每次合併相鄰兩堆、代價為兩堆之和，求總代價最小/最大。

思路

直鏈版是區間 DP 開山題：dp[l][r] = min(dp[l][k] + dp[k+1][r]) + sum(l, r)。環形拆鏈：把序列複製一倍接在後面（長 2n），對每個起點取長度 n 的區間答案取最優。注意與合併果子的區別：那裡可以合併任意兩堆所以貪心（Huffman）成立，這裡只能合併相鄰兩堆所以必須 DP。

程式碼（C++） 
for (int len = 2; len <= n; len++)
  for (int l = 1; l + len - 1 <= 2*n - 1; l++) {
    int r = l + len - 1;
    for (int k = l; k < r; k++)
      dp[l][r] = min(dp[l][r], dp[l][k] + dp[k+1][r] + S(l, r));  // S = 前綴和差
  }
// 答案 = min over i of dp[i][i+n-1]
★★LC 516 最長迴文子序列區間迴文

字串中最長的迴文子序列長度。

思路

dp[l][r]：兩端相等 → dp[l+1][r−1] + 2；否則 max(dp[l+1][r], dp[l][r−1])。迴文類的轉移看兩端字元而非分割點，複雜度 O(n²)。洛谷 P1435（最少插入幾個字元變迴文）答案 = n − 本題。

★★★LC 312 戳氣球區間枚舉最後一步

戳破氣球 i 得 a[l]·a[i]·a[r] 分（l、r 為當時的鄰居），最大化總分。

思路

正著想（枚舉第一個戳誰）子問題會互相糾纏——戳掉後兩側變鄰居。逆向：枚舉區間內「最後一個」被戳的氣球 k，此時 k 的鄰居必然是 l−1 與 r+1（固定！），左右子區間完全獨立：dp[l][r] = max(dp[l][k−1] + dp[k+1][r] + a[l−1]·a[k]·a[r+1])。「枚舉最後一步」是區間 DP 解耦子問題的招牌手法，能量項鍊（P1063）同理。

程式碼（C++） 
// a[0] = a[n+1] = 1 哨兵
for (int len = 1; len <= n; len++)
  for (int l = 1; l + len - 1 <= n; l++) {
    int r = l + len - 1;
    for (int k = l; k <= r; k++)
      dp[l][r] = max(dp[l][r], dp[l][k-1] + dp[k+1][r] + a[l-1]*a[k]*a[r+1]);
  }
★★★AtCoder DP N - Slimes區間

直鏈版石子合併（n ≤ 400），標準模板題。

思路

檢核模板正確性的裸題。寫完可以順手驗證：把 min 換 max、代價換乘法，模板都不變——區間 DP 的骨架與具體代價函數是解耦的。

★★★★★LC 546 移除盒子區間狀態加維

每次移除連續同色段，得分 = 段長平方，最大化總分。

思路

dp[l][r] 不夠：r 右側可能有同色盒子「留著跟 [l,r] 裡的一起消」——這是後效性。解法是把缺的資訊加進狀態：dp[l][r][k] = 區間 [l,r] 且右側已黏了 k 個與 a[r] 同色的盒子。轉移：要嘛立刻消掉 a[r] 那段（(k+1)²），要嘛在區間內找同色的 a[m] 與之會合（dp[l][m][k+1] + dp[m+1][r−1][0]）。「狀態不夠就加維」的教科書級示範。

⏫ 優化預告

石子合併類的 O(n³) 可用四邊形不等式（Knuth 優化）降到 O(n²)——決策點單調：opt(l, r−1) ≤ opt(l, r) ≤ opt(l+1, r)。見 2.12.4。
