2.4 區間 DP

模型如何拆常見複雜度代表
枚舉分割點\([l,k]\) 與 \([k+1,r]\) 合併\(O(n^3)\)石子合併、矩陣鏈乘
處理兩端刪／配 \(l,r\)，縮成更短區間\(O(n^2)\)迴文子序列、區間博弈
枚舉最後消失者最後一步固定區間外鄰居\(O(n^3)\)戳氣球、能量項鍊
帶外部資訊\(\mathrm{dp}[l][r][k]\) 記區間外對未來的影響依附加維而定移除盒子
🔁 為什麼一定按長度遞增？

任何轉移都應只讀更短區間，因此外層枚舉 len = 1..n，內層枚舉 \(l\)，再算 r = l + len - 1。若用遞迴，區間長度就是天然下降量。判斷是否適合區間 DP 時，試著問：「固定最後一次操作後，剩下部分是否切成互不影響的連續段？」若仍需知道段外元素，表示 \(\mathrm{dp}[l][r]\) 資訊不足，需改枚舉方向或加維。

💡 核心想法

\(\mathrm{dp}[l][r]\) 是區間 \([l,r]\) 的答案，由更短的子區間合併而來。計算順序按區間長度遞增枚舉（或記憶化搜尋）。轉移通常枚舉分割點 \(k\)：

\[ \mathrm{dp}[l][r]= \operatorname*{opt}_{l\le k<r} \left(\mathrm{dp}[l][k]+\mathrm{dp}[k+1][r]+\mathrm{cost}(l,k,r)\right), \]

其中 \(\operatorname*{opt}\) 依題意取最小或最大，複雜度為 \(O(n^3)\)。

🔎 適用特徵
  • 操作發生在相鄰元素之間（合併、消除、插入），使得子區間彼此獨立。
  • \(n \le 500\) 左右（\(O(n^3)\) 的容忍範圍）。
  • 元素順序不能重排——能重排的合併問題往往是 Huffman 貪心。
★★★洛谷 P1880 石子合併（環形）區間環形

環形排列的石子堆，每次合併相鄰兩堆、代價為兩堆之和，求總代價最小/最大。

思路

直鏈版是區間 DP 開山題：\(\mathrm{dp}[l][r]=\min_{l\le k<r}\left(\mathrm{dp}[l][k]+\mathrm{dp}[k+1][r]\right)+\mathrm{sum}(l,r)\)。環形拆鏈：把序列複製一倍接在後面（長 \(2n\)），對每個起點取長度 \(n\) 的區間答案取最優。注意與合併果子的區別：那裡可以合併任意兩堆所以貪心（Huffman）成立，這裡只能合併相鄰兩堆所以必須 DP。

程式碼（C++） 
for (int len = 2; len <= n; len++) {
    for (int left = 1; left + len - 1 <= 2 * n - 1; left++) {
        int right = left + len - 1;
        for (int split_point = left; split_point < right; split_point++) {
            dp[left][right] = min(dp[left][right],
                dp[left][split_point] + dp[split_point + 1][right] +
                    S(left, right));  // S = 前綴和差
        }
    }
}
// 答案 = min over i of dp[i][i + n - 1]
★★LC 516 最長迴文子序列區間迴文

字串中最長的迴文子序列長度。

思路

若兩端相等，\(\mathrm{dp}[l][r]=\mathrm{dp}[l+1][r-1]+2\)；否則 \(\mathrm{dp}[l][r]=\max\left(\mathrm{dp}[l+1][r],\mathrm{dp}[l][r-1]\right)\)。迴文類的轉移看兩端字元而非分割點，複雜度 \(O(n^2)\)。洛谷 P1435（最少插入幾個字元變迴文）答案為 \(n-\text{本題答案}\)。

★★★LC 312 戳氣球區間枚舉最後一步

戳破氣球 \(i\) 得 \(a_l a_i a_r\) 分（\(l,r\) 為當時的鄰居），最大化總分。

思路

正著想（枚舉第一個戳誰）子問題會互相糾纏——戳掉後兩側變鄰居。逆向：枚舉區間內「最後一個」被戳的氣球 \(k\)，此時 \(k\) 的鄰居必然是 \(l-1\) 與 \(r+1\)（固定！），左右子區間完全獨立：\(\mathrm{dp}[l][r]=\max_{l\le k\le r}\left(\mathrm{dp}[l][k-1]+\mathrm{dp}[k+1][r]+a_{l-1}a_k a_{r+1}\right)\)。「枚舉最後一步」是區間 DP 解耦子問題的招牌手法，能量項鍊（P1063）同理。

程式碼（C++） 
// a[0] = a[n + 1] = 1 哨兵
for (int len = 1; len <= n; len++) {
    for (int left = 1; left + len - 1 <= n; left++) {
        int right = left + len - 1;
        for (int split_point = left; split_point <= right; split_point++) {
            dp[left][right] = max(dp[left][right],
                dp[left][split_point - 1] + dp[split_point + 1][right] +
                    a[left - 1] * a[split_point] * a[right + 1]);
        }
    }
}
★★★AtCoder DP N - Slimes區間

直鏈版石子合併（\(n \le 400\)），標準模板題。

思路

檢核模板正確性的裸題。寫完可以順手驗證：把 min 換 max、代價換乘法，模板都不變——區間 DP 的骨架與具體代價函數是解耦的。

★★★★★LC 546 移除盒子區間狀態加維

每次移除連續同色段，得分為段長平方，最大化總分。

思路

\(\mathrm{dp}[l][r]\) 不夠：\(r\) 右側可能有同色盒子「留著跟 \([l,r]\) 裡的一起消」——這是後效性。解法是把缺的資訊加進狀態：\(\mathrm{dp}[l][r][k]\) 表示區間 \([l,r]\) 且右側已黏了 \(k\) 個與 \(a_r\) 同色的盒子。轉移：要嘛立刻消掉 \(a_r\) 那段，取得 \((k+1)^2\)；要嘛在區間內找同色的 \(a_m\) 與之會合，使用 \(\mathrm{dp}[l][m][k+1]+\mathrm{dp}[m+1][r-1][0]\)。「狀態不夠就加維」的教科書級示範。

⏫ 優化預告

石子合併類的 \(O(n^3)\) 可用四邊形不等式（Knuth 優化）降到 \(O(n^2)\)——決策點單調：\(\mathrm{opt}(l,r-1)\le\mathrm{opt}(l,r)\le\mathrm{opt}(l+1,r)\)。見 2.12.4。
