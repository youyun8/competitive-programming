2.11 圖上 DP 與博弈 DP

依賴結構應用方法計算順序
DAG直接做最值／計數 DP拓撲序或記憶化 DFS
一般有向圖，但環內可視為整體SCC 縮點後 DP縮點 DAG 的拓撲序
非負權最短路Dijkstra，而非普通 DP依目前最短距離動態決定
有限步數／特權次數加入層數，把環沿新維度展開按層或最短路
兩人零和博弈勝負、差值或 minimax DP從終局逆推
🎮 博弈狀態的三種常用值

只問勝負時，存在一個必敗後繼就是必勝；所有後繼都必勝才是必敗。問分差時，若目前拿 \(x\) 分後交給對手，轉移常寫成 \(\mathrm{dp}[s]=\max_{\mathrm{next}}\left(x-\mathrm{dp}[\mathrm{next}]\right)\)，因為對手的優勢要從自己的收益中扣掉。若不是零和、資訊不完全或可能無限循環，這個簡單倒推不一定成立，需另建模型。

💡 核心想法

DP 要求依賴關係無環。圖上做 DP 的三條路：① 圖本身是 DAG，拓撲序即計算順序；② 有環，則用 SCC 縮點成 DAG，或改用最短路演算法（鬆弛可視為帶環的 DP）；③ 博弈，狀態是「局面＋輪到誰」，從終局倒推勝負／最優值。

🔎 適用特徵
  • 「按依賴關係求最長鏈／方案數」用 DAG DP；「\(k\) 次特權」用分層圖。
  • 雙人輪流、最優策略、問勝負或最大差值 → 博弈 DP。
★★AtCoder DP G - Longest PathDAG

DAG 的最長路徑長度。

思路

\(\mathrm{dp}[v]=1+\max_{u\to v}\mathrm{dp}[u]\)，表示以 \(v\) 結尾的最長路。記憶化搜尋不用手排拓撲序，天然正確。「最長遞增子序列」其實就是這題——LIS 的 DAG 觀點。方案數版把最大值換成加法。

★★★洛谷 P3387 縮點（SCC + DP）DAGSCC

有向圖選一條路徑（點可重經過），最大化路徑點權和。

思路

環上的點要拿全拿，因此用 Tarjan 縮點，每個 SCC 的權值是成員和，縮完是 DAG，再按拓撲序做 \(\mathrm{dp}[v]=w[v]+\max_{u\to v}\mathrm{dp}[u]\)。「先把環壓掉再 DP」是有環圖問題的第一反應。

★★★★洛谷 P4568 飛行路線（分層圖）分層圖最短路

最多 \(k\) 條邊可免費，求最短路（\(k \le 10\)）。

思路

\(\mathrm{dist}[v][j]\) 是到 \(v\) 且用了 \(j\) 次免費機會的最短距離。實作成 \(k+1\) 層圖（層內原邊、層間免費邊）跑 Dijkstra。最短路本質是「無法排序狀態」的 DP，Dijkstra 的貪心出隊就是它的計算順序；「有限次特權」類問題（免費／翻倍／穿牆）一律狀態加維。

★★AtCoder DP K - Stones博弈

石堆剩 \(k\) 顆，每步取集合 \(A\) 中的數量，取不了者輸。誰贏？

思路

勝負態遞推：\(\mathrm{win}[k]=\bigvee_{\substack{a\in A\\a\le k}}\neg\mathrm{win}[k-a]\)，也就是存在一個 \(a\in A\) 能把對手推入必敗態時，當前狀態必勝。這是所有公平組合博弈的 DP 原型；推廣到多堆就是 SG 函數與 Nim 的異或定理（本站點到為止，深入請開博弈論專題）。

程式碼（C++） 
for (int k = 1; k <= limit; k++) {
    for (int a : moves) {
        if (a <= k && !win[k - a]) {
            win[k] = true;
            break;
        }
    }
}
★★★AtCoder DP L - Deque博弈區間

輪流從序列兩端取數，雙方都最大化「自己 \(-\) 對方」，求最終差值。

思路

差值視角把兩個人合成一個目標：\(\mathrm{dp}[l][r]\) 是當前玩家在 \([l,r]\) 上能取得的最大淨差，轉移為 \(\mathrm{dp}[l][r]=\max\left(a_l-\mathrm{dp}[l+1][r],a_r-\mathrm{dp}[l][r-1]\right)\)。對手的最優淨差取負就是我的損失。LC 486 預測贏家同構。「相對分差」讓 minimax 塌縮成單一最大值，是博弈 DP 最重要的一步化簡。

🧭 延伸

多堆獨立博弈的合成使用 SG 函數：Grundy 值是後繼狀態集合的 \(\operatorname{mex}\)，全局 SG 值是各堆 SG 值的異或。SG 值的計算本身就是一個記憶化 DP——你已經會了，缺的只是定理。
