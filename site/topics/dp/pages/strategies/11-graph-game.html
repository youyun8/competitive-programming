2.11 圖上 DP 與博弈 DP

💡 核心想法

DP 要求依賴關係無環。圖上做 DP 的三條路：① 圖本身是 DAG → 拓撲序即計算順序；② 有環 → SCC 縮點成 DAG，或改用最短路演算法（鬆弛 = 帶環的 DP）；③ 博弈 → 狀態=「局面＋輪到誰」，從終局倒推勝負/最優值。

🔎 適用特徵
  • 「按依賴關係求最長鏈/方案數」→ DAG DP；「k 次特權」→ 分層圖。
  • 雙人輪流、最優策略、問勝負或最大差值 → 博弈 DP。
★★AtCoder DP G - Longest PathDAG

DAG 的最長路徑長度。

思路

dp[v] = 以 v 結尾的最長路 = max(dp[u]) + 1（u→v 有邊）。記憶化搜尋不用手排拓撲序，天然正確。「最長遞增子序列」其實就是這題——LIS 的 DAG 觀點。方案數版把 max 換 +。

★★★洛谷 P3387 縮點（SCC + DP）DAGSCC

有向圖選一條路徑（點可重經過），最大化路徑點權和。

思路

環上的點要拿全拿 → Tarjan 縮點，每個 SCC 的權值 = 成員和，縮完是 DAG，再按拓撲序 dp[v] = w[v] + max(dp[u])。「先把環壓掉再 DP」是有環圖問題的第一反應。

★★★★洛谷 P4568 飛行路線（分層圖）分層圖最短路

最多 k 條邊可免費，求最短路（k ≤ 10）。

思路

狀態加維：dist[v][j] = 到 v 用了 j 次免費的最短距離。實作成 k+1 層圖（層內原邊、層間免費邊）跑 Dijkstra。最短路本質是「無法排序狀態」的 DP，Dijkstra 的貪心出隊就是它的計算順序；「有限次特權」類問題（免費/翻倍/穿牆）一律狀態加維。

★★AtCoder DP K - Stones博弈

石堆剩 k 顆，每步取集合 A 中的數量，取不了者輸。誰贏？

思路

勝負態遞推：win[k] = 存在 a ∈ A 使 win[k−a] 為輸（能把對手推入必敗態就必勝）。所有公平組合博弈的 DP 原型；推廣到多堆就是 SG 函數與 Nim 的異或定理（本站點到為止，深入請開博弈論專題）。

程式碼（C++） 
for (int k = 1; k <= K; k++)
    for (int a : A)
        if (a <= k && !win[k - a]) { win[k] = true; break; }
★★★AtCoder DP L - Deque博弈區間

輪流從序列兩端取數，雙方都最大化「自己 − 對方」，求最終差值。

思路

差值視角把兩個人合成一個目標：dp[l][r] = 當前玩家在 [l,r] 上能取得的最大淨差 = max(a[l] − dp[l+1][r], a[r] − dp[l][r−1])——對手的最優淨差取負就是我的損失。LC 486 預測贏家同構。「相對分差」讓 minimax 塌縮成單一 max，是博弈 DP 最重要的一步化簡。

🧭 延伸

多堆獨立博弈的合成 → SG 函數（Grundy 值 = mex of 後繼），全局 SG = 各堆 SG 異或。SG 值的計算本身就是一個記憶化 DP——你已經會了，缺的只是定理。
