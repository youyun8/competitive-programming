初學者閱讀指南2.1 線性 DP（入門三件套與 LIS 家族）

這頁要解決什麼問題？

先問：暴力遞迴會重複計算哪些相同子問題？要記住哪些資訊，才能讓未來決策不需要回頭看完整歷史？

本頁聚焦「「前 i 個」與「以 i 結尾」兩種視角，從爬樓梯到 LIS 二維偏序」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：直接枚舉所有選擇常形成指數級遞迴樹；同一個剩餘問題會從不同路徑被算很多次。

核心直覺

DP 像填表：每格只保存『未來真正需要知道的摘要』，並從已算好的較小問題轉移過來。

閱讀程式前先找不變量

寫轉移前先用一句完整中文定義 dp 狀態，包含範圍、是否必選、恰好或至多等條件。

先猜猜看：什麼時候該想到這頁？

求最優值、方案數或可行性，而且問題能由較小範圍、較少物品或較短前綴組成。

最小可手算例子

  1. 爬 4 階：dp[0]=1，dp[1]=1。
  2. dp[2]=dp[1]+dp[0]=2，dp[3]=3，dp[4]=5。
  3. 每格代表到達該位置的方法數，來源只可能是前 1 或前 2 階。

2.1 線性 DP（入門三件套與 LIS 家族）

狀態視角典型定義答案位置代表題
前 \(i\) 個\(\mathrm{dp}[i]\) \(=\) 前 \(i\) 個元素的完整答案\(\mathrm{dp}[n]\)爬樓梯、打家劫舍
必須以 \(i\) 結尾\(\mathrm{dp}[i]\) \(=\) 以 \(a_i\) 作最後元素的答案\(\max_i \mathrm{dp}[i]\)LIS、最大子段和
值域作狀態\(\mathrm{best}[x]\) \(=\) 結尾值為 \(x\) 時的最優解值域取最優值域小或可用 BIT 優化的 LIS 變形
🧠 選狀態的判斷法

如果只知道「前 \(i\) 個的最優值」就足以接上第 \(i+1\) 個，用前綴狀態；如果未來能否接元素取決於最後值、最後位置或最後類型，就把它固定在狀態裡。例：LIS 若只存前 \(i\) 個的最長長度，無法判斷 \(a_{i+1}\) 能否接上，因為長度相同的序列可能有不同尾值；改成「以 \(i\) 結尾」後，尾值就是 \(a_i\)，轉移才無後效。空間滾動前也要先確認右式只依賴固定幾個較早狀態。

💡 核心想法

狀態沿著序列一維推進。兩種最常用的視角：「前 \(i\) 個的答案」（答案在 \(\mathrm{dp}[n]\)）與「以 \(i\) 結尾的答案」（答案是所有 \(\mathrm{dp}[i]\) 取最大值）。分不清用哪個時，看轉移是否需要「最後一個元素是誰」——需要就用「以 \(i\) 結尾」。

🔎 適用特徵
  • 輸入是一條序列，決策按下標從左到右進行，做完第 i 個不會回頭改前面。
  • 每一步只有常數種選擇（選/不選、接/不接）。
★LC 70 爬樓梯線性

每次爬 \(1\) 或 \(2\) 階，爬到第 \(n\) 階有幾種走法？

思路

\(\mathrm{dp}[i]\) 為到第 \(i\) 階的走法數。最後一步要嘛從 \(i-1\) 跨 \(1\) 階、要嘛從 \(i-2\) 跨 \(2\) 階：\(\mathrm{dp}[i]=\mathrm{dp}[i-1]+\mathrm{dp}[i-2]\)——就是斐波那契。這題示範了轉移方程的萬用推導法：枚舉最後一步。

程式碼（C++） 
int two_back = 1, one_back = 1;  // dp[0], dp[1]
for (int i = 2; i <= n; i++) {
    int current = two_back + one_back;
    two_back = one_back;
    one_back = current;
}
return one_back;  // 滾動到只剩兩個變數
★LC 198 打家劫舍線性選/不選

不能偷相鄰兩家，最大化偷竊總額。

思路

\(\mathrm{dp}[i]\) 為只考慮前 \(i\) 家的最大收益。第 \(i\) 家不偷：\(\mathrm{dp}[i-1]\)；偷：\(\mathrm{dp}[i-2]+a_i\)。取最大值。「選/不選」是線性 DP 最基本的決策形式，環形版 LC 213 拆成「不偷第 \(1\) 家」與「不偷最後一家」兩條鏈各做一次。

程式碼（C++） 
int prev2 = 0, prev1 = 0;
for (int x : nums) {
    int cur = max(prev1, prev2 + x);
    prev2 = prev1;
    prev1 = cur;
}
return prev1;
★LC 53 最大子陣列和（Kadane）線性以 i 結尾

找和最大的連續子陣列。

思路

「以 \(i\) 結尾」視角的代表作：\(\mathrm{dp}[i]\) 為以 \(a_i\) 結尾的最大子段和，轉移為 \(\mathrm{dp}[i]=\max\left(\mathrm{dp}[i-1],0\right)+a_i\)（前綴是負的就丟掉重開）。答案是 \(\max_i \mathrm{dp}[i]\)。環形版 LC 918：答案是「普通最大子段和」或「總和 \(-\) 最小子段和」二者取大。

程式碼（C++） 
int cur = 0, best = INT_MIN;
for (int x : nums) {
    cur = max(cur, 0) + x;
    best = max(best, cur);
}
return best;
★★LC 300 最長遞增子序列（LIS）線性LIS二分

求最長嚴格遞增子序列長度。

思路

\(O(n^2)\)：\(\mathrm{dp}[i]=1+\max_{j最小可能結尾。\(g\) 單調遞增，每個 \(a_i\) 用 lower_bound 替換第一個大於或等於它的位置。這是「DP + 貪心維護決策集合」的第一課，也是 BIT 優化轉移（2.12.8）的入門素材。

程式碼（C++） 
vector<int> tails;
for (int x : nums) {
    auto it = lower_bound(tails.begin(), tails.end(), x);  // 非嚴格遞增改 upper_bound
    if (it == tails.end()) {
        tails.push_back(x);
    } else {
        *it = x;
    }
}
return tails.size();
★★★洛谷 P1020 導彈攔截LISDilworth

第一問：一套系統最多攔幾發（最長不上升子序列）；第二問：最少幾套系統攔完所有導彈。

思路

第二問是 Dilworth 定理：把序列劃分成最少的不上升鏈，等於最長嚴格上升子序列的長度。兩問都用 \(O(n\log n)\) 的 \(g\) 陣列寫法。這題把「LIS 不只是求長度，還是覆蓋/劃分的對偶」講透了。

★★洛谷 P1091 合唱隊形LIS

留下一個先升後降的子序列，最少踢掉幾人？

思路

對每個 \(i\) 求「以 \(i\) 結尾的 LIS」\(f_i\) 與「以 \(i\) 開頭的最長下降子序列」\(g_i\)（反著做一次 LIS），答案為 \(n-\max_i(f_i+g_i-1)\)。正反各掃一遍再拼接是線性 DP 的常用組合技。

★★★LC 354 俄羅斯套娃信封LIS排序鍵設計

信封 \((w,h)\) 嚴格雙維遞增才能嵌套，最多套幾層？

思路

二維偏序化一維：按 \(w\) 遞增、\(w\) 相同時 \(h\) 遞減排序，再對 \(h\) 做 LIS。\(h\) 遞減保證同 \(w\) 的信封不會互相接龍——排序鍵設計錯（\(h\) 也遞增）是本題最經典的 WA。

★★CF 455A Boredom線性值域當狀態

取走一個值為 \(v\) 的元素得 \(v\) 分，但所有 \(v-1\)、\(v+1\) 的元素被刪除，最大得分？

思路

對值域而非下標做 DP：\(\mathrm{cnt}[v]\) 是值 \(v\) 的出現次數，轉移為 \(\mathrm{dp}[v]=\max\left(\mathrm{dp}[v-1],\mathrm{dp}[v-2]+v\cdot\mathrm{cnt}[v]\right)\)——變成值域上的打家劫舍。「對什麼做 DP」本身就是設計決策：下標、值域、時間軸都可以是階段。

常見誤解與除錯順序

  1. 狀態定義少了條件，導致不同歷史被錯誤合併。
  2. 初始化把不可達狀態設成 0，讓不存在的方案參與轉移。
  3. 迴圈順序與轉移依賴相反，讀到本輪剛更新的值或尚未計算的值。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
