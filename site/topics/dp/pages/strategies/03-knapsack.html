2.3 背包全家桶（背包九講精華）

📚 「背包九講」到底是哪九講？

「背包九講」不是九道題，而是一套把背包 DP 系統化的經典講義。原本的九個主題依序是：0/1 背包、完全背包、多重背包、混合三種背包、二維費用、分組背包、有依賴的背包、泛化物品、背包問法的變化。本站把前七講直接落成模板與例題；第八講用「一件物品其實是一個費用—收益函數」統一各種變形；第九講則拆成可行性、計數、恰好裝滿、方案還原與退背包。所以下面的九張題卡不是「九講各一題」，而是用現代競賽題把整套知識重新編排。

原講次在回答什麼本站對應
1. 0/1 背包每件至多用一次P1048；容量倒序
2. 完全背包每種可用無限次P1616、P5020；容量正序
3. 多重背包每種至多用 mi 次P1776；二進位拆分／單調佇列
4. 混合背包不同物品的使用次數規則不同逐件判型，再套該型的更新方式
5. 二維費用同時受兩種資源限制LC 474；兩維一起倒序
6. 分組背包每組最多選一件P1757；組外、容量中、物品內
7. 有依賴背包選附件前必須選主件P1064；打包成組／一般化為樹上背包
8. 泛化物品物品不只是一對 (w,v)，而是函數 f(w)把一組選項、子樹方案看成一件「廣義物品」後合併
9. 問法的變化最值、可行、計數、恰好、輸出方案LC 494、P5020、P4141與本頁末的初始化表
💡 核心想法

「容量」是狀態、「物品」是階段：dp[j] = 容量 j 時的最優值，逐件物品更新。全家桶的分野只有一個問題：每件物品能用幾次——一次（0/1）、無限（完全）、k 次（多重）、同組互斥（分組）、有前置（依賴）。而在一維寫法裡，這個差異全部濃縮在容量的枚舉方向上。

🔎 適用特徵
  • 「在總量限制下選一個子集，最大化價值 / 統計方案 / 判斷可行」。
  • 「總量」不一定是重量：時間、金錢、下標和、甚至「差值」都可以當容量。

2.3.1 先翻譯題目：容量、物品與 dp 值各是什麼？

看到「從若干選項中挑一些，總成本不能超標」時，先不要背程式碼，先完成三個翻譯：容量 W 是哪個限制、物品 i 的費用 wi／收益 vi 是什麼、dp[j] 存的是哪一種答案。同一個轉移骨架只要換掉運算與初值，就能回答不同問法：

題目問法dp[j] 的語意合併運算初始值
容量內最大價值成本至多 j 的最大收益max(dp[j], dp[j-w]+v)全 0
恰好成本 j 的最大價值恰好用掉 j 時的最大收益同上，但不可達狀態不能參與dp[0]=0，其餘 −INF
能否恰好湊出 j布林可達性dp[j] |= dp[j-w]dp[0]=true
恰好湊出 j 的方案數不同選法數dp[j] += dp[j-w]dp[0]=1
湊出 j 的最少件數最少使用幾件min(dp[j], dp[j-w]+1)dp[0]=0，其餘 +INF

2.3.2 一張表選對背包類型

限制核心迴圈為什麼時間
每件 0/1 次物品外層，j 從 W 倒序讀到的 dp[j-w] 仍是上一階段O(nW)
每種無限次物品外層，j 從 w 正序允許讀本階段剛更新的值，等於再拿一次O(nW)
每種 m 次拆成 O(log m) 個 0/1 物品1,2,4,… 可表示 0..m 的任何件數O(WΣlog m)
每組最多一件組 → j 倒序 → 組內選項同一組的候選都從進組前的狀態轉移O(W×總物品數)
兩個容量兩個容量維都依 0/1／完全規則枚舉費用是向量，使用次數規則不變O(nW1W2)
🧠 為什麼一個 for 的方向能決定「可用幾次」？

先保留二維狀態 F[i][j]：它明確表示「只看前 i 種物品」。壓成一維後，陣列同時扮演上一排與這一排。0/1 背包必須讀上一排，所以由大到小更新，尚未碰過的 dp[j-w] 才代表 F[i-1][j-w]；完全背包需要讀這一排的 F[i][j-w]，所以由小到大，讓同一物品能沿容量方向反覆傳遞。方向不是口訣，而是在控制右式讀到哪一個階段。

手算一次：容量 5、物品 (2,3) 與 (3,4)

0/1 背包初始 [0,0,0,0,0,0]。放入 (2,3) 後變成 [0,0,3,3,3,3]；再放入 (3,4) 後變成 [0,0,3,4,4,7]，容量 5 的 7 來自兩件各用一次。若第一件正序更新，處理到容量 4 時會讀到本輪剛寫好的 dp[2]=3，得到 6，等於把 (2,3) 拿了兩次——演算法已悄悄變成完全背包。

★★洛谷 P1048 採藥（0/1 背包）背包

T 分鐘內採藥，每株耗時 t_i、價值 v_i，各最多採一次，最大化總價值。

思路

二維：dp[i][j] = 前 i 株、耗時 j 的最大價值 = max(dp[i−1][j], dp[i−1][j−t] + v)。壓一維後必須倒序枚舉 j：保證 dp[j−t] 還是「上一排」的值，否則同一株會被用兩次。

程式碼（C++） 
vector<int> dp(T + 1, 0);
for (int i = 0; i < n; i++)
    for (int j = T; j >= t[i]; j--)          // 0/1：倒序
        dp[j] = max(dp[j], dp[j - t[i]] + v[i]);
return dp[T];                                 // 「不超過 T」：初始化全 0
★★洛谷 P1616 疯狂的采药（完全背包）背包

同上，但每株可採無限次。

思路

只改一個字：正序枚舉 j。此時 dp[j−t] 已是「本排」的值，物品天然可重複選。理解這一字之差，等於理解一維背包的全部語意。

程式碼（C++） 
for (int i = 0; i < n; i++)
    for (long long j = t[i]; j <= T; j++)     // 完全：正序
        dp[j] = max(dp[j], dp[j - t[i]] + v[i]);
🔀 第四講：混合背包其實沒有新方程

若有些物品只能拿一次、有些可無限拿、有些有數量上限，就依物品類型分別做一次 0/1、完全或多重更新。關鍵不在另背模板，而是每處理完一種物品後，dp 都必須代表「只使用目前已處理物品」的完整答案；這個階段不變量讓三種更新可以安全混用。

★★★洛谷 P1776 寶物篩選（多重背包）背包二進位拆分

每種物品有 m_i 件，容量 4×10⁴、Σm 很大。

思路

二進位拆分：把 m 件拆成 1, 2, 4, …, 2^k, 剩餘 這些「打包件」，任意 0..m 都能由它們組合出來，然後當 0/1 背包做——O(ΣlogM · W)。極限資料要用單調佇列優化到 O(nW)（2.12.2）。

程式碼（C++） 
for (每種物品 w, v, m) {
    for (int k = 1; m > 0; k <<= 1) {
        int c = min(k, m); m -= c;
        items.push_back({w * c, v * c});      // 打包成 0/1 物品
    }
}
// 之後照 0/1 背包倒序做
★★★洛谷 P1757 通天之分組背包背包分組

物品分組，每組最多選一件。

思路

枚舉順序是關鍵：組在外、容量居中（倒序）、組內物品在最內——這樣每個容量對每組只做一次決策，天然保證「每組至多一件」。樹上背包（2.8.3）本質就是「每個子樹是一組」的分組背包。

程式碼（C++） 
for (auto& grp : groups)
    for (int j = W; j >= 0; j--)              // 容量倒序在中層
        for (auto& [w, v] : grp)
            if (j >= w) dp[j] = max(dp[j], dp[j - w] + v);
★★★LC 474 一和零（二維費用）背包二維費用

每個字串消耗若干個 0 與 1，兩種資源都有上限，最多選幾個字串？

思路

費用變成向量 → 狀態多一維：dp[j][k]，兩維都倒序枚舉。洛谷 P1855（金錢+時間）同型。「兩個上限」是二維費用的直接訊號。

★★★洛谷 P1064 金明的預算方案（依賴背包）背包依賴

附件必須跟著主件買，每個主件至多兩個附件。

思路

附件數少 → 把「主件 / 主+附1 / 主+附2 / 主+附1+附2」四種打包當作同組互斥物品，退化成分組背包。依賴關係一般化（樹狀）就是樹上背包。

🧩 第八講：什麼是「泛化物品」？

普通物品用一對 (費用, 價值) 描述；泛化物品則用函數 f[c] 描述「分配 c 單位容量給它，最多得到多少價值」。一組互斥選項可以先算成一個 f；一棵子樹也能先算成一個 f。合併兩個泛化物品 A、B，就是枚舉容量切分：new[j] = max( A[k] + B[j-k] )。因此分組背包、依賴背包與樹上背包並非零散技巧，而是同一個「函數合併」觀點的不同外觀。

★★LC 494 目標和（計數背包）背包計數

給每個數填 +/−，使總和為 target 的方案數。

思路

設正號集合和為 s，則 s = (sum + target) / 2 —— 轉成「選子集湊出 s 的方案數」：dp[j] += dp[j−a]，倒序枚舉。計數版把 max 換成 +=，初始 dp[0]=1。同型：LC 416 分割等和子集（可行性版，用 bool 或 bitset）。

程式碼（C++） 
int s = accumulate(a.begin(), a.end(), 0) + target;
if (s < 0 || s % 2) return 0;
s /= 2;
vector<int> dp(s + 1, 0); dp[0] = 1;
for (int x : a)
    for (int j = s; j >= x; j--) dp[j] += dp[j - x];
return dp[s];
★★★洛谷 P5020 貨幣系統（NOIP 2018）背包可行性

找最小的等價貨幣系統：能表出的金額集合完全相同。

思路

結論：答案 = 原系統中不能被其他面額湊出來的面額個數。排序後跑可行性完全背包，若 a_i 已能被更小面額表出則它是冗餘的。「判斷能否湊出」= 完全背包的 bool 版，是很多構造題的子件。

★★★★洛谷 P4141 消失之物（退背包）背包撤銷

對每個 i 回答：不用第 i 件物品時，湊出每個體積的方案數。

思路

先算全體的計數背包 f。退背包：因為加入物品的運算（卷積）可交換，「刪掉物品 i」可以逆運算：g[j] = f[j] − g[j − w_i]（正序枚舉 j）。避免對每個 i 重算 O(nW)。撤銷思想在「強制不選某物」類題目通用（另一路是前後綴背包合併）。

程式碼（C++） 
// f 為含全部物品的計數背包
for (int i = 0; i < n; i++) {
    g = f;                                    // g = 去掉物品 i 的背包
    for (int j = w[i]; j <= W; j++) g[j] = (g[j] - g[j - w[i]] + MOD) % MOD;
    // g[1..W] 即答案
}
⚠ 「恰好裝滿」的初始化

求「不超過容量」：dp 全初始化 0。求「恰好裝滿」：dp[0] = 0、其餘 −INF（計數則 dp[0]=1、其餘 0）——−INF 的語意是「此容量不可達」，讓非法狀態永遠不會勝出。詳見第 3 章。

2.3.3 第九講：問法一變，答案位置也會變

  • 至多 W：若狀態定義是「容量上限」，通常直接讀 dp[W]；若定義成「恰好使用 j」，則取 max(dp[0..W])。
  • 恰好 W：只讀 dp[W]，且必須用不可達哨兵初始化。
  • 方案數：先定義「不同」是物品集合不同、排列不同，還是數值組合不同。完全背包中物品在外層算組合數；容量在外層通常會算排列數。
  • 輸出方案：保留二維表後從 F[n][W] 倒推；若 F[i][j] == F[i-1][j] 可不選 i，否則選 i 並令 j -= w[i]。一維壓縮節省了空間，也會抹掉這段歷史。
  • 第 k 優／字典序最小方案：dp 值需保存多個候選或加上次要比較鍵，不能只存一個最大值。
⚠ 寫完背包的五項自檢
  1. dp[j] 是「容量至多 j」還是「恰好 j」？
  2. 同一物品可用幾次，迴圈方向是否對應？
  3. 計數是在數組合還是排列，物品與容量誰在外層？
  4. w=0、負價值、target 超界等邊界是否另行處理？
  5. nW 是否能過；若 W 太大，能否改成價值作容量、meet-in-the-middle 或 bitset？
