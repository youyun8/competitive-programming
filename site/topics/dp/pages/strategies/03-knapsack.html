2.3 背包九講：九個主題逐一拆解

📚 先統一符號

共有 n 件物品，背包容量為 W；物品 i 的費用為 wi、價值為 vi。除非小節另有說明，費用皆為正整數，目標是在容量限制內最大化總價值。以下九節各自給出狀態、算式和 C++ 實作，可以獨立閱讀。

講次主題一眼辨認核心技巧
10/1 背包每件至多選一次容量倒序
2完全背包每種可選無限次容量正序
3多重背包每種至多選指定件數二進位拆分
4混合三種背包每種物品的次數規則不同逐件判型
5二維費用背包同時受兩種資源限制兩維都依規則枚舉
6分組背包每組最多選一件整組從舊狀態轉移
7有依賴的背包選子項前必須選父項樹上合併
8泛化物品一個物品有多種費用—收益方案max-plus 合併
9背包問法的變化最值、可行、計數、恰好、方案改語意、初值與運算

第一講：0/1 背包

限制：每件物品只有「不選」或「選一次」兩種決策。令 Fi(c) 表示只考慮前 i 件物品、容量上限為 c 時的最大價值。

Fi(c) = max(Fi−1(c), Fi−1(c − wi) + vi)　（c ≥ wi） F0(c) = 0

壓成一維後，dp[c - item.weight] 必須仍是「還沒使用本件物品」的舊值，所以容量由大到小更新。正序會讀到本輪剛寫入的狀態，等於讓同一件物品重複出現。

實作（C++） 
struct Item {
    int weight;
    long long value;
};

long long SolveZeroOneKnapsack(const std::vector<Item>& items, int capacity) {
    std::vector<long long> dp(capacity + 1, 0);
    for (const Item& item : items) {
        for (int current = capacity; current >= item.weight; --current) {
            dp[current] =
                std::max(dp[current], dp[current - item.weight] + item.value);
        }
    }
    return dp[capacity];
}

複雜度：時間 O(nW)，空間 O(W)。代表題：洛谷 P1048 採藥。

第二講：完全背包

限制：每種物品可選任意非負次。直接枚舉選幾件的式子如下；把「再選一件同種物品」留在本階段，就能化成第二式。

Fi(c) = max0 ≤ k ≤ ⌊c / wi⌋(Fi−1(c − kwi) + kvi) Fi(c) = max(Fi−1(c), Fi(c − wi) + vi)

右式需要讀同一階段的 Fi，因此一維陣列要由小到大更新。這會讓較小容量剛得到的結果繼續傳到較大容量，恰好代表再拿一件。

實作（C++） 
struct Item {
    int weight;
    long long value;
};

long long SolveCompleteKnapsack(const std::vector<Item>& items, int capacity) {
    std::vector<long long> dp(capacity + 1, 0);
    for (const Item& item : items) {
        for (int current = item.weight; current <= capacity; ++current) {
            dp[current] =
                std::max(dp[current], dp[current - item.weight] + item.value);
        }
    }
    return dp[capacity];
}

複雜度：時間 O(nW)，空間 O(W)。代表題：洛谷 P1616 瘋狂的採藥。

第三講：多重背包

限制：物品 i 最多有 mi 件。最直接的轉移會再枚舉使用數量 k：

Fi(c) = max0 ≤ k ≤ min(mi, ⌊c / wi⌋) (Fi−1(c − kwi) + kvi)

直接做需 O(WΣmi)。常用優化是把數量拆成 1, 2, 4, … 和最後餘數；每一包只能選一次，而且任何 0..mi 的件數都能由這些包組成，問題便轉成 0/1 背包。

二進位拆分實作（C++） 
struct BoundedItem {
    int weight;
    long long value;
    int count;
};

long long SolveBoundedKnapsack(const std::vector<BoundedItem>& items,
                               int capacity) {
    std::vector<long long> dp(capacity + 1, 0);
    for (const BoundedItem& item : items) {
        int remaining = item.count;
        for (int bundle_size = 1; remaining > 0; bundle_size <<= 1) {
            const int take = std::min(bundle_size, remaining);
            remaining -= take;
            const int bundle_weight = item.weight * take;
            const long long bundle_value = item.value * take;
            for (int current = capacity; current >= bundle_weight; --current) {
                dp[current] = std::max(
                    dp[current], dp[current - bundle_weight] + bundle_value);
            }
        }
    }
    return dp[capacity];
}

複雜度：時間 O(WΣlog(mi + 1))，空間 O(W)。若資料要求 O(nW)，再使用單調佇列優化。代表題：洛谷 P1776 寶物篩選。

第四講：混合三種背包

限制：有些物品只能選一次、有些可選無限次、有些有數量上限。它沒有新的狀態方程；每處理一種物品，就依該物品的限制選用前三講的轉移。

limiti = 1：使用 0/1 背包轉移 limiti = ∞：使用完全背包轉移 1 < limiti < ∞：使用多重背包轉移

不變量是：處理完第 i 種物品後，dp[current] 已是只使用前 i 種物品的最佳答案。只要每種更新本身維持這個語意，三種方式就能安全混用。

實作（C++） 
enum class ItemKind {
    kZeroOne,
    kComplete,
    kBounded,
};

struct MixedItem {
    int weight;
    long long value;
    ItemKind kind;
    int count;
};

long long SolveMixedKnapsack(const std::vector<MixedItem>& items,
                             int capacity) {
    std::vector<long long> dp(capacity + 1, 0);
    const auto update_zero_one = [&dp, capacity](int weight, long long value) {
        for (int current = capacity; current >= weight; --current) {
            dp[current] = std::max(dp[current], dp[current - weight] + value);
        }
    };

    for (const MixedItem& item : items) {
        if (item.kind == ItemKind::kComplete) {
            for (int current = item.weight; current <= capacity; ++current) {
                dp[current] = std::max(dp[current],
                                       dp[current - item.weight] + item.value);
            }
            continue;
        }
        if (item.kind == ItemKind::kZeroOne) {
            update_zero_one(item.weight, item.value);
            continue;
        }
        int remaining = item.count;
        for (int bundle_size = 1; remaining > 0; bundle_size <<= 1) {
            const int take = std::min(bundle_size, remaining);
            remaining -= take;
            update_zero_one(item.weight * take, item.value * take);
        }
    }
    return dp[capacity];
}

複雜度：0/1 與完全物品各做 O(W)，數量為 m 的多重物品做 O(Wlog(m + 1))；空間 O(W)。

第五講：二維費用背包

限制：每件物品同時消耗兩種資源，例如金錢與時間。令費用為 (ai, bi)，兩種容量上限為 (A, B)。

Fi(x, y) = max(Fi−1(x, y), Fi−1(x − ai, y − bi) + vi)

若物品是 0/1 次，兩個容量維都倒序；若物品可無限次，兩維都正序。方向仍然只在控制右式讀到上一階段還是本階段。

0/1 二維費用實作（C++） 
struct TwoCostItem {
    int first_cost;
    int second_cost;
    long long value;
};

long long SolveTwoCostKnapsack(const std::vector<TwoCostItem>& items,
                               int first_capacity, int second_capacity) {
    std::vector<std::vector<long long>> dp(
        first_capacity + 1, std::vector<long long>(second_capacity + 1, 0));
    for (const TwoCostItem& item : items) {
        for (int first = first_capacity; first >= item.first_cost; --first) {
            for (int second = second_capacity; second >= item.second_cost;
                 --second) {
                dp[first][second] = std::max(
                    dp[first][second],
                    dp[first - item.first_cost][second - item.second_cost] +
                        item.value);
            }
        }
    }
    return dp[first_capacity][second_capacity];
}

複雜度：時間 O(nAB)，空間 O(AB)。代表題：LeetCode 474 Ones and Zeroes。

第六講：分組背包

限制：物品被分成若干組，每組最多選一件。令 Gg 為第 g 組；處理一整組時，所有候選都必須從進入這組以前的狀態轉移。

Fg(c) = max(Fg−1(c), maxi ∈ Gg, wi ≤ c(Fg−1(c − wi) + vi))

實作直接保留 previous，能清楚保證同一組不會連選兩件。若追求常數，也可用「組在外、容量倒序、組內物品在內」原地更新。

實作（C++） 
struct Item {
    int weight;
    long long value;
};

long long SolveGroupedKnapsack(const std::vector<std::vector<Item>>& groups,
                               int capacity) {
    std::vector<long long> dp(capacity + 1, 0);
    for (const std::vector<Item>& group : groups) {
        const std::vector<long long> previous = dp;
        for (int current = 0; current <= capacity; ++current) {
            for (const Item& item : group) {
                if (item.weight <= current) {
                    dp[current] =
                        std::max(dp[current],
                                 previous[current - item.weight] + item.value);
                }
            }
        }
    }
    return dp[capacity];
}

複雜度：時間 O(W × 總物品數)，空間 O(W)。代表題：洛谷 P1757 通天之分組背包。

第七講：有依賴的背包

限制：選一件物品前必須先選它的父物品，依賴關係形成森林。若每個主件只有少量附件，可列舉「主件 + 附件子集」後做分組背包；一般情況則是樹上背包。

令 Du(c) 表示已選節點 u，並在其子樹恰好使用 c 容量的最大價值。先只放入 u；合併孩子 v 時，枚舉有多少容量分給孩子：

Du(wu) = vu D′u(c + x) = max(D′u(c + x), Du(c) + Dv(x))

x = 0 代表整棵孩子子樹都不選；若 x > 0，Dv(x) 已保證先選孩子 v，因此依賴不會被破壞。加一個費用與價值皆為 0 的虛擬根，就能統一處理森林。

一般樹形依賴實作（C++） 
struct DependentItem {
    int weight;
    long long value;
    int parent;
};

long long SolveDependentKnapsack(const std::vector<DependentItem>& items,
                                 int capacity) {
    constexpr long long kNegativeInfinity =
        std::numeric_limits<long long>::lowest() / 4;
    const int root = static_cast<int>(items.size());
    std::vector<std::vector<int>> children(items.size() + 1);
    for (int node = 0; node < static_cast<int>(items.size()); ++node) {
        const int parent = items[node].parent;
        children[parent == -1 ? root : parent].push_back(node);
    }

    const std::function<std::vector<long long>(int)> dfs =
        [&](int node) -> std::vector<long long> {
        std::vector<long long> dp(capacity + 1, kNegativeInfinity);
        if (node == root) {
            dp[0] = 0;
        } else if (items[node].weight <= capacity) {
            dp[items[node].weight] = items[node].value;
        }

        for (const int child : children[node]) {
            const std::vector<long long> child_dp = dfs(child);
            std::vector<long long> next = dp;
            for (int used = 0; used <= capacity; ++used) {
                if (dp[used] == kNegativeInfinity) {
                    continue;
                }
                for (int child_cost = 1; child_cost + used <= capacity;
                     ++child_cost) {
                    if (child_dp[child_cost] == kNegativeInfinity) {
                        continue;
                    }
                    next[used + child_cost] =
                        std::max(next[used + child_cost],
                                 dp[used] + child_dp[child_cost]);
                }
            }
            dp = std::move(next);
        }
        return dp;
    };

    const std::vector<long long> answer = dfs(root);
    return *std::max_element(answer.begin(), answer.end());
}

複雜度：上述通用寫法最壞時間 O(nW²)、空間 O(nW)（含遞迴期間的 DP 陣列）；可依子樹大小限制枚舉範圍。附件特例代表題：洛谷 P1064 金明的預算方案。

第八講：泛化物品

觀點：普通物品只有「不選」與「付 w 得 v」；泛化物品則是一個函數 f(c)，表示分配恰好 c 容量給它可得到的最大價值。分組選項、整棵子樹的方案表，都可以先整理成一個泛化物品。

new(c) = max0 ≤ x ≤ c(dp(c − x) + f(x))

這是 max-plus 卷積：枚舉總容量如何切給「既有物品」和「新泛化物品」。不可用的費用方案設成負無限；f(0) = 0 表示可以不選。

實作（C++） 
long long SolveGeneralizedKnapsack(
    const std::vector<std::vector<long long>>& gain_by_cost, int capacity) {
    constexpr long long kNegativeInfinity =
        std::numeric_limits<long long>::lowest() / 4;
    std::vector<long long> dp(capacity + 1, kNegativeInfinity);
    dp[0] = 0;

    for (const std::vector<long long>& gain : gain_by_cost) {
        std::vector<long long> next(capacity + 1, kNegativeInfinity);
        for (int total = 0; total <= capacity; ++total) {
            for (int cost = 0;
                 cost <= total && cost < static_cast<int>(gain.size());
                 ++cost) {
                if (dp[total - cost] == kNegativeInfinity ||
                    gain[cost] == kNegativeInfinity) {
                    continue;
                }
                next[total] =
                    std::max(next[total], dp[total - cost] + gain[cost]);
            }
        }
        dp = std::move(next);
    }
    return *std::max_element(dp.begin(), dp.end());
}

複雜度：若每個泛化物品列出 0..W 的方案，時間 O(nW²)、空間 O(W)。它的價值在於統一建模；實際題目通常利用方案稀疏、單調性或特殊結構加速。

第九講：背包問法的變化

背包不只問最大價值。骨架相同，但 dp[current] 的語意、初始值、合併運算與答案位置必須一起改，不能只把 std::max 隨手換掉。

問法狀態語意轉移初始值
容量內最大價值費用至多 c 的最大值max(dp(c), dp(c−w)+v)全部為 0
恰好裝滿的最大價值費用恰為 c 的最大值同上，不可達狀態不轉移dp(0)=0，其餘為 −∞
能否恰好湊出費用 c 是否可達reachable(c) ∨ reachable(c−w)只有 0 為真
恰好湊出的方案數費用 c 的不同選法數ways(c) + ways(c−w)ways(0)=1
輸出一組方案保留前 i 件的最優值比較選與不選保留二維表後倒推
恰好裝滿：Ei(c) = max(Ei−1(c), Ei−1(c − wi) + vi) 計數：Qi(c) = Qi−1(c) + Qi−1(c − wi)

下面的 0/1 範例一次算出「容量內最大值、恰好裝滿最大值、恰好裝滿方案數」，並在可達時還原一組恰好裝滿的方案。方案數以 1,000,000,007 取模。

綜合實作（C++） 
struct Item {
    int weight;
    long long value;
};

struct KnapsackAnswers {
    long long best_at_most;
    std::optional<long long> best_exact;
    long long ways_exact;
    std::vector<int> chosen_indices;
};

KnapsackAnswers AnalyzeZeroOneKnapsack(const std::vector<Item>& items,
                                       int capacity) {
    constexpr long long kMod = 1'000'000'007;
    constexpr long long kNegativeInfinity =
        std::numeric_limits<long long>::lowest() / 4;
    const int item_count = static_cast<int>(items.size());
    std::vector<std::vector<long long>> best(
        item_count + 1,
        std::vector<long long>(capacity + 1, kNegativeInfinity));
    std::vector<long long> ways(capacity + 1, 0);
    best[0][0] = 0;
    ways[0] = 1;

    for (int index = 1; index <= item_count; ++index) {
        const Item& item = items[index - 1];
        best[index] = best[index - 1];
        for (int current = item.weight; current <= capacity; ++current) {
            if (best[index - 1][current - item.weight] != kNegativeInfinity) {
                best[index][current] = std::max(
                    best[index][current],
                    best[index - 1][current - item.weight] + item.value);
            }
        }
        for (int current = capacity; current >= item.weight; --current) {
            ways[current] =
                (ways[current] + ways[current - item.weight]) % kMod;
        }
    }

    KnapsackAnswers answer;
    answer.best_at_most =
        *std::max_element(best[item_count].begin(), best[item_count].end());
    answer.ways_exact = ways[capacity];
    if (best[item_count][capacity] == kNegativeInfinity) {
        answer.best_exact = std::nullopt;
        return answer;
    }

    answer.best_exact = best[item_count][capacity];
    int remaining = capacity;
    for (int index = item_count; index > 0; --index) {
        const Item& item = items[index - 1];
        if (remaining >= item.weight &&
            best[index - 1][remaining - item.weight] != kNegativeInfinity &&
            best[index][remaining] ==
                best[index - 1][remaining - item.weight] + item.value) {
            answer.chosen_indices.push_back(index - 1);
            remaining -= item.weight;
        }
    }
    std::reverse(answer.chosen_indices.begin(), answer.chosen_indices.end());
    return answer;
}

複雜度：為了還原方案保留二維表，時間 O(nW)、空間 O(nW)；若只問數值或計數，可壓回 O(W)。完全背包計數時還要先決定要數「組合」或「排列」：物品在外層通常數組合，容量在外層通常數排列。

⚠ 寫完背包的五項自檢
  1. 狀態是「容量至多」還是「費用恰好」？
  2. 每件物品可用幾次，容量枚舉方向是否對應？
  3. 不可達狀態是否使用正確的初始值與哨兵？
  4. 計數是在數組合還是排列，迴圈順序是否吻合？
  5. nW 是否能通過；若容量過大，能否改以價值作狀態、使用 bitset 或 meet-in-the-middle？
