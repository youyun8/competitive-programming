2.7 位元狀壓 DP

🔢 mask 的語意比位元技巧重要

先用一句話固定 mask：它究竟表示「已選集合」、「可用集合」、還是「上一列的佔用情況」？再決定轉移是加一個元素、刪一個元素或換到下一列。常用操作：判第 \(k\) 位 (mask >> k) & 1、加入 mask | (1 << k)、刪除 mask ^ (1 << k)（已知該位為 \(1\)）、枚舉子集 sub = (sub - 1) & mask。

骨架狀態數 × 轉移大約適用代表
集合＋最後元素\(2^n \times n^2\)\(n \lesssim 20\)TSP
集合配對／加一元素\(2^n \times n\)\(n \lesssim 22\)完美匹配計數
枚舉 mask 的所有 sub\(3^n\)\(n \lesssim 15\text{--}17\)集合劃分
逐列輪廓列數 × 合法 mask 對寬度小、長度大互不侵犯、炮兵陣地
⚠ 先算上界再開陣列

\(2^{20}\approx10^6\)，再乘 \(20\) 已是兩千萬次；\(2^{25}\approx3.3\times10^7\)，單一 int 陣列就約 \(128\ \mathrm{MB}\)。若狀態還乘位置、步數或多測，必須先估記憶體。稀疏狀態才考慮 map／unordered_map；稠密狀態用連續陣列通常快得多。

💡 核心想法

\(n \le 20\) 時，「哪些元素已處理」的集合本身塞進狀態：一個 \(n\) 位二進位整數。轉移就是位運算。兩大家族：集合推進型（\(\mathrm{dp}[S]\)，按 \(S\) 從小到大）與行輪廓型（\(\mathrm{dp}[i][S]\)，\(S\) 是第 \(i\) 行的放置狀態）。

🔎 適用特徵
  • \(n \le 20\)（\(2^n\) 狀態）或棋盤寬度不超過 \(20\)（每行 \(2^m\) 種輪廓）。
  • 「每個元素用一次、順序可任意」的排列／配對問題——用集合代替排列，將 \(n!\) 降成 \(2^n\)。
// 位運算工具箱
int lb = s & (-s);                    // lowbit：最低位的 1
int c = __builtin_popcount(s);        // 集合大小
for (int t = s; t; t = (t - 1) & s) {  // 枚舉 s 的非空子集
}
// 全體「枚舉每個集合的每個子集」總量是 3^n（每位三種歸屬），非 4^n
★★★AtCoder DP O - Matching狀壓配對計數

\(n\) 男 \(n\) 女相容關係給定，求完美匹配數（\(n \le 21\)）。

思路

\(\mathrm{dp}[S]\) 是前 \(\operatorname{popcount}(S)\) 個男生與女生集合 \(S\) 完成配對的方案數。關鍵洞見：第 \(i\) 個男生配誰不用記「哪些男生用過」——按順序處理，男生編號就是 \(\operatorname{popcount}(S)\)，一維集合就夠。轉移枚舉 \(S\) 的每個位，複雜度為 \(O(2^n n)\)。

程式碼（C++） 
dp[0] = 1;
for (int s = 1; s < (1 << n); s++) {
    int i = __builtin_popcount(s) - 1;  // 正在配第 i 個男生
    for (int j = 0; j < n; j++) {
        if ((s >> j & 1) && ok[i][j]) {
            dp[s] = (dp[s] + dp[s ^ (1 << j)]) % MOD;
        }
    }
}
★★★洛谷 P1171 售貨員的難題（TSP）狀壓TSP

訪問所有村莊一次並回起點的最短路程（\(n \le 20\)）。

思路

經典 TSP：\(\mathrm{dp}[S][v]\) 是已訪問集合 \(S\)、目前在 \(v\) 的最短距離，轉移枚舉下一站，複雜度為 \(O(2^n n^2)\)。「已訪問集合 + 目前位置」是狀壓最常用的狀態骨架——位置必須進狀態，因為它決定後續代價（無後效性）。

★★★洛谷 P1896 互不侵犯（SCOI2005）狀壓行輪廓

\(n\times n\) 棋盤放 \(k\) 個國王，互不攻擊（八方向相鄰），求方案數。

思路

行輪廓型模板：\(\mathrm{dp}[i][j][S]\) 表示前 \(i\) 行放 \(j\) 個、第 \(i\) 行狀態為 \(S\)。預處理所有「行內無相鄰 \(1\)」的合法行狀態（\(n=9\) 時僅數十個），行間檢查 S & T、S & (T << 1)、S & (T >> 1) 皆空。先枚舉合法行再兩兩配對，比枚舉全部 \(2^n\times2^n\) 快得多。

★★★★洛谷 P2704 炮兵陣地（NOI 2001）狀壓行輪廓

炮兵攻擊範圍上下左右各兩格，地形限制部署位置，最多放幾個？

思路

影響跨兩行，所以狀態記兩行：\(\mathrm{dp}[i][S_i][S_{i-1}]\)。合法狀態數少，實際跑得動。「影響範圍多遠，輪廓就記多深」——這個原則推到極致就是專家章的輪廓線／插頭 DP（2.13.1）。

★★★★洛谷 P2831 憤怒的小鳥（NOIP 2016）狀壓預處理覆蓋

每條過原點的拋物線可消滅其上所有豬（\(n \le 18\)），最少幾條拋物線？

思路

先預處理：每對豬決定一條拋物線，得到它能覆蓋的豬集合 \(\mathrm{cover}[i][j]\)。然後做最小集合覆蓋：\(\mathrm{dp}[S]\) 是已消滅集合 \(S\) 的最少線數，轉移時固定 \(S\) 中最小的未消滅豬——它必屬於某條線，只枚舉包含它的線，避免同一組合被枚舉多次，複雜度從 \(3^n\) 降到 \(2^n n\)。

程式碼（C++） 
for (int s = 0; s < (1 << n); s++) {
    if (dp[s] == INF) {
        continue;
    }
    int p = __builtin_ctz(~s);   // 最小的還活著的豬
    for (int j = 0; j < n; j++) {  // 枚舉「經過 p」的拋物線
        dp[s | cover[p][j]] = min(dp[s | cover[p][j]], dp[s] + 1);
    }
}
⏫ 進階去向

棋盤鋪滿類（骨牌、迴路）→ 輪廓線 DP／插頭 DP（2.13.1）；「對每個 S 求所有子集的貢獻和」→ SOS DP（2.13.2）。
