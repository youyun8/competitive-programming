2.8 樹上 DP

💡 核心想法

dp[u][狀態] 由所有子節點的答案合併而來，DFS 後序遍歷天然就是拓撲序。設計重點永遠是：「關於 u 的子樹，父親需要知道什麼？」——那就是狀態的第二維。

🔎 適用特徵
  • 輸入是樹（或森林），問題關於子樹/路徑/覆蓋/獨立集。
  • 「每個節點對所有節點的答案」→ 換根 DP（2.8.4）。
★★洛谷 P1352 沒有上司的舞會樹上獨立集

直屬上司與下屬不能同時出席，最大化快樂值總和（樹上最大權獨立集）。

思路

dp[u][0/1] = u 不來/來 時子樹的最大快樂值。dp[u][1] = w_u + Σ dp[v][0]；dp[u][0] = Σ max(dp[v][0], dp[v][1])。樹上 DP 的 Hello World，AtCoder DP P（染色計數版）同構。

程式碼（C++） 
void dfs(int u, int fa) {
    dp[u][1] = w[u];
    for (int v : g[u]) if (v != fa) {
        dfs(v, u);
        dp[u][0] += max(dp[v][0], dp[v][1]);
        dp[u][1] += dp[v][0];
    }
}
★★樹的直徑（樹形 DP 解法）樹上路徑

求樹上最長路徑（邊帶權，可為負——負權時兩次 DFS 法失效，DP 法仍正確）。

思路

f[u] = 從 u 往子樹下走的最長鏈。合併時經過 u 的最長路徑 = 最大鏈 + 次大鏈：dfs 中先用 ans = max(ans, f[u] + f[v] + w) 再更新 f[u]。「維護最大 + 次大」是樹上路徑類的通用手筋（換根 DP 也靠它）。

★★★洛谷 P2014 選課（樹上背包）樹上背包

課程有先修樹形依賴，選 m 門學分最大化。

思路

dp[u][j] = u 子樹裡選 j 門（含 u）的最大學分。合併子樹 = 分組背包：每個子樹是一組，「給這個子樹分配幾個名額」是組內選項。用 dfs 序 + 「只與已合併大小相乘」的上下界寫法，複雜度是 O(nm) 而非看起來的 O(nm²)。

程式碼（C++） 
void dfs(int u) {
    sz[u] = 1; dp[u][1] = w[u];
    for (int v : g[u]) {
        dfs(v);
        for (int j = min(sz[u], m); j >= 1; j--)          // 已合併部分：倒序
            for (int k = 1; k <= min(sz[v], m - j); k++)  // 分給子樹 v 的名額
                dp[u][j + k] = max(dp[u][j + k], dp[u][j] + dp[v][k]);
        sz[u] += sz[v];
    }
}
★★★洛谷 P3478 STA-Station（換根 DP）樹上換根

選一個根使所有節點深度總和最大。

思路

換根 DP 三步：① 一次 DFS 算出以 1 為根時每個子樹的大小 sz 與答案 f[1]；② 推「根從 u 移到相鄰的 v」的增量公式：f[v] = f[u] + (n − 2·sz[v])（v 子樹全體近 1 步、其餘遠 1 步）；③ 第二次 DFS 把答案推到所有點。O(n) 回答「每個點當根」的問題。

程式碼（C++） 
void dfs2(int u, int fa) {
    for (int v : g[u]) if (v != fa) {
        f[v] = f[u] + n - 2LL * sz[v];   // 增量換根
        dfs2(v, u);
    }
}
★★★★CF 1187E Tree Painting樹上換根

第一步染任一點，之後每步染與已染區相鄰的點、得分 = 該點所在未染連通塊大小，最大化總分。

思路

先證：定了起點後最優總分固定 = Σ sz[v]（每個子樹被「等」了 sz 次）。於是問題變成「選根最大化 Σ sz」，套換根：f[v] = f[u] + n − 2·sz[v]。換根 DP 的典型考法：先把問題化簡成「以每個點為根的某個量」。

★★★★AtCoder DP V - Subtree樹上換根計數

對每個 v 求：包含 v 的連通子圖數（mod M，M 不一定質數）。

思路

下行：f[u] = Π (f[v] + 1)。換根需要「去掉一個兒子的乘積」，但 M 非質數不能除 → 用前綴積 × 後綴積合成兄弟乘積。這是乘積型換根的標準細節，很多人栽在「用逆元」上。

★★★★洛谷 P2458 保安站崗（覆蓋三態）樹上最小支配集

每個點可放保安（有費用），每個點必須被自己/父/子的保安看住，最小費用。

思路

三態狀態設計：dp[u][0] 自己放、dp[u][1] 被某個兒子覆蓋、dp[u][2] 被父親覆蓋。難點在 dp[u][1]：所有兒子取 min(dp[v][0], dp[v][1]) 後，若沒有任何兒子取到 dp[v][0]，要補一個最小的「換成放保安」差價。覆蓋/支配類的樹 DP 幾乎都是這套三態 + 補差價。
