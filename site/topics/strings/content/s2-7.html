2.7 後綴數組（Suffix Array）

💡 核心想法

把 \(s\) 的 \(n\) 個後綴按字典序排序：\(sa[i]\)＝排名第 \(i\) 的後綴起點，\(rk[i]\)＝後綴 \(i\) 的排名（互逆）。再配上 \(height[i] = \mathrm{LCP}(sa[i-1], sa[i])\)（排名相鄰後綴的最長公共前綴），就得到子串世界的「排序 + 相似度」全景圖。任何子串都是某後綴的前綴——子串的字典序、去重、公共前綴問題全部化到這張表上。

🔁 兩個支柱定理

① LCP Lemma：\(\mathrm{LCP}(sa[i],sa[j]) = \min_{i<k\le j} height[k]\)——排名區間上的 RMQ，配 ST 表後任兩後綴 LCP \(O(1)\)。② height 均攤：\(height[rk[i]] \ge height[rk[i-1]] - 1\)（去掉首字元的後綴繼承 LCP−1），所以按 \(i=1..n\) 順序計算 height 時指標總移動 \(O(n)\)。

🔎 適用特徵
  • 問題涉及子串字典序（第 k 小子串、最小的滿足…的子串）。
  • 「出現 ≥ k 次的最長子串」「本質不同子串數」——height 上的分組／求和。
  • 離線、靜態；需要線上增量改用 SAM。
模板（C++）：倍增 + 基數排序 O(n log n) 
// 1-indexed；sa/rk/height；倍增每輪按（第二關鍵字, 第一關鍵字）基數排序
int sa[N], rk[N], old[N], cnt[N], id[N], height[N];
void build_sa(const char* s, int n, int m) {     // m = 字元集大小
    for (int i = 1; i <= n; i++) {
        cnt[rk[i] = s[i]]++;
    }
    for (int i = 1; i <= m; i++) {
        cnt[i] += cnt[i-1];
    }
    for (int i = n; i >= 1; i--) {
        sa[cnt[rk[i]]--] = i;
    }
    for (int w = 1; w < n; w <<= 1, m = /*新排名數*/ rk[sa[n]]) {
        int p = 0;                               // 第二關鍵字：空後綴最小
        for (int i = n - w + 1; i <= n; i++) {
            id[++p] = i;
        }
        for (int i = 1; i <= n; i++) {
            if (sa[i] > w) {
                id[++p] = sa[i] - w;
            }
        }
        memset(cnt, 0, sizeof(int) * (m + 1));   // 第一關鍵字基數排序
        for (int i = 1; i <= n; i++) {
            cnt[rk[i]]++;
        }
        for (int i = 1; i <= m; i++) {
            cnt[i] += cnt[i-1];
        }
        for (int i = n; i >= 1; i--) {
            sa[cnt[rk[id[i]]]--] = id[i];
        }
        memcpy(old, rk, sizeof(int) * (n + 1));
        for (int i = 1, p = 0; i <= n; i++) {      // 重標排名（相同對併排名）
            rk[sa[i]] = (old[sa[i]] == old[sa[i-1]] &&
                         old[sa[i]+w] == old[sa[i-1]+w]) ? p : ++p;
        }
        if (rk[sa[n]] == n) {
            break;               // 排名互異，提前結束
        }
    }
    for (int i = 1, k = 0; i <= n; i++) {        // height：均攤 O(n)
        if (k) {
            k--;
        }
        while (s[i + k] == s[sa[rk[i] - 1] + k]) {
            k++;
        }
        height[rk[i]] = k;
    }
}
★★★★洛谷 P3809 後綴排序模板SA

輸出 sa 陣列，\(|s|\le 10^6\)。

思路

倍增 + 基數排序，\(O(n\log n)\)。實作重點：第二關鍵字不用真的排序（上一輪 sa 平移即得）、排名相同的對必須併排名、提前結束判斷。SA-IS 可線性但競賽版模板背倍增即可。

★★★洛谷 P4051 [JSOI2007] 字符加密SA環形

把 s 的所有旋轉排序後取每個旋轉的末字元。

思路

環形 → 拆環倍長：對 \(s+s\) 做 SA，取起點在前半的後綴的相對順序即為旋轉排序（長度都 ≥ n，比較等價）。「環轉鏈 + 倍長」與區間 DP 的拆環同款，SA 的第一道應用題。

★★★★洛谷 P2408 不同子串個數SA去重計數

求本質不同子串數。

思路

按 sa 順序考慮每個後綴：它貢獻的「新」前綴數 = 長度 − 與前一名的 LCP，故答案 \(=\frac{n(n+1)}{2}-\sum height[i]\)。height 是「重複」的度量——這個視角在 SA 題裡反覆出現。SAM 一行 \(\sum(len_u - len_{link(u)})\) 也能做（2.8），對照理解兩結構怎麼各自「去重」。

★★★★洛谷 P2852 Milk Patterns（SA 正解）SAheight分組

至少出現 \(k\) 次（可重疊）的最長子串。

思路

出現 \(k\) 次的子串 = 某 \(k\) 個後綴的公共前綴 = sa 上連續 \(k\) 個後綴（排序後相似者相鄰！）。答案 \(=\max\) 窗口大小 \(k-1\) 的 \(\min(height)\) → 單調佇列滑窗。「出現次數 → sa 連續段 → height 區間 min」是 SA 最重要的思考鏈。對照 2.1 的 hash 二分解與 2.8 的 SAM 解。

★★★★★洛谷 P4248 [AHOI2013] 差異SA單調棧

求 \(\sum_{i<j} \big(|T_i|+|T_j|-2\,\mathrm{LCP}(T_i,T_j)\big)\)，\(T_i\) 為後綴。

思路

前半是常數，難點在 \(\sum \mathrm{LCP} = \sum_{i<j}\min(height[i+1..j])\)——「所有區間 min 的總和」：對每個 \(height[k]\) 用單調棧算它作為最小值支配的區間數（左右第一個更小者），乘上貢獻。SA × 單調棧的定番組合；同款算貢獻手法亦見資料結構主題的柱狀圖問題。

⏫ SA vs SAM 選型

要字典序／排名區間／height 的 RMQ 結構 → SA；要 endpos（出現位置集合）結構、線上增量 → SAM。兩者能互相模擬大部分功能，但順手程度差很多——第 4 章有完整對照表。
