2.5 Trie 與 01-Trie

💡 核心想法

Trie（字典樹）：把一堆字串按字元逐層插進樹裡，公共前綴自動共享路徑——「前綴」在 Trie 上就是「祖先」。01-Trie 是它最重要的變形：把整數當成固定 30～60 位的 01 字串，從最高位插入；於是「找與 \(x\) XOR 最大的數」變成在樹上逐位貪心走相反位，\(O(\log V)\)。

🔎 適用特徵
  • 一堆字串的前綴查詢：有多少串以 \(p\) 為前綴、自動補全、字典序第 k 小字串。
  • 看到 XOR 最值／XOR 第 k 大幾乎反射性上 01-Trie。
  • 空間 = 節點數 × 字元集大小——字元集大（如 Unicode）改用 map 子節點或雙陣列。
  • Trie 也是 AC 自動機的底座、廣義 SAM 的離線載體（2.11.2）。
模板（C++）：Trie 與 01-Trie 
// Trie：ch[u][c] 子節點，cnt[u] 經過該節點的串數（前綴計數）
int ch[N][26], cnt[N], tot = 0;
void insert(const char* s) {
    int u = 0;
    for (int i = 0; s[i]; i++) {
        int c = s[i] - 'a';
        if (!ch[u][c]) ch[u][c] = ++tot;
        u = ch[u][c]; cnt[u]++;
    }
}
// 01-Trie：查與 x 異或的最大值（B 為最高位）
int maxXor(int x) {
    int u = 0, res = 0;
    for (int b = B; b >= 0; b--) {
        int d = (x >> b) & 1;
        if (ch[u][d ^ 1]) { res |= 1 << b; u = ch[u][d ^ 1]; } // 貪心走相反位
        else u = ch[u][d];
    }
    return res;
}
★LC 208 實作 TrieTrie

實作 insert / search / startsWith。

思路

裸結構題。用「節點池 + 陣列」而非 new 指標——競賽寫法要為之後的 ACAM 打底（節點編號才能 BFS、才能開 fail 陣列）。

★★洛谷 P8306 Trie 模板（前綴計數）Trie多測

多測：每組給字串集合與查詢，問多少字串以查詢串為前綴。字元集含大小寫與數字。

思路

插入時沿路 cnt[u]++，查詢走到底回傳 cnt。兩個坑故意埋給你：① 字元集是 62 個（映射表要寫全）；② 多測不能 memset 整個節點池——只清 0..tot 用過的節點。這正是第 3 章「多測清空成本」的現場教學。

★★★LC 421 陣列中兩個數的最大異或值01-Trie

求 \(\max_{i,j} a_i \oplus a_j\)。

思路

全部插入 01-Trie，對每個 \(a_i\) 貪心查一次取 max。貪心正確性：高位優先——高位能取 1 就絕不為低位妥協（\(2^b > \sum_{k<b}2^k\)）。這是字典序貪心在二進位上的化身。

★★★洛谷 P4551 最長異或路徑01-Trie樹上前綴

樹上邊權，求 XOR 和最大的路徑。

思路

設 \(d[u]\)＝根到 \(u\) 的邊權 XOR，則路徑 \(u\to v\) 的 XOR \(= d[u]\oplus d[v]\)（LCA 以上抵消）——問題化成 LC 421。「樹上路徑 XOR = 兩點前綴 XOR」這步轉化比 Trie 本身更重要，是 XOR 題的第一反應。

★★★★★CF 888G Xor-MST01-TrieBoruvka

完全圖邊權 \(=a_u\oplus a_v\)，求 MST 權重（\(n\le 2\times 10^5\)）。

思路

在 01-Trie 上分治：某個節點的左右子樹（bit 0/1 兩群）之間只需連一條最小 XOR 邊——枚舉小子樹每個值到大子樹查最小 XOR（啟發式），加上遞迴兩側。等價於 Boruvka 每輪用 Trie 找最小出邊。與貪心主題 2.8 共用：那邊講 MST 正確性，這邊講 Trie 上的分治結構。

⏫ 進階去向

「區間內與 \(x\) 的最大 XOR」→ 可持久化 01-Trie（前綴版本相減），見資料結構主題 2.12；多字串匹配 → 給 Trie 加 fail 指標，就是下一章的 AC 自動機。
