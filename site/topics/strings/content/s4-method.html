4.1 選型決策樹

第一個問題追問選擇
單模式匹配？要確定性、要 fail 的副產品（週期）KMP
句型是「每個位置往後配多長」Z 函數
還要跟二分/資料結構混搭hash

逐個本質不同迴文統計PAM
所有子串的統計？字典序、排名、height 分組SA

💡 兩條經驗法則

① 能用輕工具就不用重工具：KMP 十行、SA 四十行、SAM 六十行且 debug 成本指數上升——比賽裡「hash 能不能混過去」永遠值得先想 30 秒。② 重工具的價值在「結構」不在「匹配」：如果你用 SAM 只為了找一個串出現在哪，你用錯了；用到 endpos／parent 樹／DAG 導航才回本。

4.2 一題三解對照：Milk Patterns（P2852）

解法複雜度思路一句話什麼時候選它
hash + 二分\(O(n\log n)\)二分長度，窗口 hash 計數最快寫完；容忍機率正確
SA + height\(O(n\log n)\)滑窗取 \(k-1\) 個 height 的 min 的最大值已有 SA 模板；還要順便答別的字典序問題
SAM + cnt\(O(n)\)\(\max\{len_u : cnt_u\ge k\}\)一行收尾；還要 endpos 相關資訊

三解殊途同歸，但「哪個先想到」暴露你的思考入口：hash 走「二分判定」、SA 走「排序後相鄰」、SAM 走「等價類」。三個入口都應該熟。

4.3 模板碼風格約定

  • 全域陣列 + 節點池：所有自動機用 int ch[N][Σ] 節點池寫法，不用指標——可 memcpy、可離線、multi-test 好清。
  • 下標制固定：KMP/SA 用 1-indexed（fail=長度）、Z/Manacher 用 0-indexed——每份模板頂端註解寫死，永不現場改。
  • reset 函式：每個結構配一個只清 \(0..tot\) 的 reset()，多測直接呼叫。
  • 模板自帶測試句：註解裡放一個手推過的小例（如 banana 的 sa/height 值），移植後 30 秒自檢。

4.4 對拍驗證器

字串暴力極好寫，對拍是字串模板的標配保險：

對拍範例（C++）：隨機小字串 + 暴力比對 
// gen：小字元集 + 短長度最容易觸發 border / 迴文邊界
string gen(mt19937& rng) {
    int n = rng() % 12 + 1, sigma = rng() % 2 + 2;   // 長度 1..12，字元 2..3 種
    string s;
    for (int i = 0; i < n; i++) {
        s += char('a' + rng() % sigma);
    }
    return s;
}
int brute_min_period(const string& s) {                // 暴力最小週期
    int n = s.size();
    for (int p = 1; p <= n; p++) {
        bool ok = true;
        for (int i = 0; i + p < n; i++) {
            if (s[i] != s[i+p]) {
                ok = false;
                break;
            }
        }
        if (ok) {
            return p;
        }
    }
    return n;
}
int main() {
    mt19937 rng(random_device{}());
    for (int t = 0; t < 100000; t++) {
        string s = gen(rng);
        int a = brute_min_period(s), b = kmp_min_period(s);   // 你的模板
        if (a != b) {
            cout << "WA " << s << " brute=" << a
                 << " yours=" << b << "\n";
            return 1;
        }
    }
    cout << "all passed\n";
}

同款骨架換掉「暴力函式」即可對拍 Z（暴力逐位比）、Manacher（暴力中心擴展）、SA（sort 後綴）、SAM 計數（枚舉所有子串塞 map）——每背下一份新模板，先讓它通過十萬組對拍再上賽場。
