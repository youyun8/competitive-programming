3常見錯誤與陷阱

症狀優先檢查最小測資方向
CF 上 WA on test 60+hash 被卡（自然溢位／單模／固定 base）Thue–Morse 串、全同字元長串
TLE 但複雜度「看起來對」多測 memset 整池、substr 拷貝、跳 fail 未剪枝大量小測資、深 fail 鏈
模板題就 WA下標制混用（長度制 vs 下標制、0/1-indexed）單字元、兩字元串
SAM/PAM 計數偏小統計順序（len 基數排序／逆序累加）aaaa（深 link 鏈）
答案爆負數int 溢位：cnt×len、n(n+1)/2、hash 乘法n = 10⁶ 的極端串

3.1 hash 被卡全家桶

⚠ 症狀與修法

① 自然溢位（mod \(2^{64}\)）：Thue–Morse 串可構造出必碰撞測資，CF 有現成 hack——改用大質數取模。② 單模數固定 base：出題人能預埋反例——base 開賽用 mt19937(chrono::…now()) 隨機。③ 碰撞率估算：單模 \(M\approx 10^9\)、比較 \(k\) 次，錯誤率約 \(k/M\)；\(k\sim 10^6\) 時已達千分之一，正式比賽雙模數是紀律不是潔癖。

3.2 「KMP 是線性」的兩種誤解

⚠ 症狀

① 失配時寫 j-- 或從頭重比 → 退化 \(O(nm)\)：必須 j = fail[j]（跳到次長 border，中間長度數學上不可能匹配）。② 在主循環裡對每個位置沿 fail 鏈走到底收集所有匹配 → 最壞平方：要匹配計數請一次打點，收尾在失配樹上做子樹和（2.2／2.6 的 fail 樹手法）。

3.3 Z 與 fail 方向搞反

⚠ 症狀

① 想沿「所有 border」枚舉卻去讀 z 陣列，或想要「從 i 往後配多長」卻跳 fail 鏈。口訣：fail 以 i 結尾、z 從 i 開始。② 另一個高頻筆誤：Z 的盒內繼承忘記 min(z[i-L], R-i) 的截斷——超出盒子的部分沒有任何保證。

3.4 SA 三件套下標災難

⚠ 症狀

① sa（排名→位置）與 rk（位置→排名）互為反函數，公式裡寫反一個就全錯。② \(height[i]\) 定義在「排名 \(i\) 與 \(i-1\)」之間，區間 LCP 是 \(\min(height[i+1..j])\)——左端點要 +1。修法：把三個陣列的定義用註解寫在模板頂端，用「banana」手推一遍對照。

3.5 SAM／PAM 的統計順序

⚠ 症狀

cnt 沿 parent 樹累加時用了錯誤順序（如按節點編號），深鏈上父節點先被加、子節點的貢獻漏掉。修法：① SAM 按 len 基數排序後由長到短。② PAM 直接按節點編號逆序（PAM 編號天然拓撲，SAM 因 clone 而不然——這正是兩者的差異點）。③ clone 節點 cnt 初始為 0 也常被忘。

3.6 多測清空成本

⚠ 症狀

Trie／ACAM 每組測資 memset(ch, 0, sizeof ch) → \(O(T\cdot\text{SIZE})\) 直接 TLE。修法：只清 \(0..tot\) 的節點（for (int i = 0; i <= tot; i++) memset(ch[i], ...)），或版本號懶清空。P8306 是現場教學題。

3.7 字元集與邊界雜項

  • 題面說小寫字母、測資混進大寫/數字：c-'a' 變負數下標，不一定立刻 RE——讀題時先確認字元集，映射寫成函式。
  • 拼接技巧（\(p+\#+t\)）的分隔符必須兩串都不出現；數字串用 char(1) 比 '#' 保險。
  • 爆 int 三兄弟：hash 乘法（先轉 unsigned long long）、\(cnt\times len\)、\(n(n+1)/2\)。
  • Manacher 新舊座標換算 off-by-one：把換算式寫進模板註解，不要每次現推。
  • 讀入 \(10^6\)+ 字串：cin >> string 記得關同步；gets 已死，scanf("%s") 注意緩衝區大小。
🔬 字串題的對拍姿勢

字串暴力特別好寫（子串枚舉、朴素匹配都十行內），所以對拍成本極低、收益極高：隨機小字元集（2～3 個字母最容易觸發 border/迴文邊界）短串 + 暴力比對，幾分鐘就能定位 90% 的模板錯。範例碼見 第 4 章。
