初學者閱讀指南理論基礎：字串的共同語言

這頁要解決什麼問題？

先問：字串比較中有哪些前綴、後綴或子串資訊被反覆重算？失配後能否利用已知結構跳過位置？

本頁聚焦「border 與週期的核心等式、三種匹配思維、複雜度反推與實作工具箱」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：從每個起點重新逐字比較，最壞會反覆檢查相同字元，讓長字串退化到 O(nm) 或 O(n²)。

核心直覺

字串演算法的核心常是記住『已經匹配了多少』或『哪些片段相同』，失敗時不必完全從零開始。

閱讀程式前先找不變量

先固定下標制與陣列定義；每個 pi、z、fail、len 或 link 值都必須能用一句話精確描述。

先猜猜看：什麼時候該想到這頁？

大量模式匹配、重複子串、border、週期、迴文、前綴統計或字典序比較。

1理論基礎：字串的共同語言

🧪 一條貫穿全指南的等式：border ⟺ 週期

設 \(|s|=n\)。稱 \(t\) 是 \(s\) 的 border，若 \(t\) 同時是 \(s\) 的真前綴與真後綴；稱 \(p\) 是 \(s\) 的週期，若對所有合法的 \(i\) 都有 \(s_i=s_{i+p}\)。核心等式：\(p\) 是 \(s\) 的週期 ⟺ \(s\) 有長度 \(n-p\) 的 border。證明是一行對齊：把 \(s\) 往右平移 \(p\) 格，重疊部分逐字相等，重疊部分恰是長 \(n-p\) 的前綴與後綴。這條等式是 KMP 的 fail 陣列、循環節判定、失配樹全部的地基——求最長 border 就是求最小週期。

1.1 基本名詞

名詞定義易混淆點
子串（substring）連續的一段 \(s_{l..r}\)與「子序列」不同——子序列可以不連續
前綴／後綴\(s_{1..i}\)／\(s_{i..n}\)；「真」= 不等於自己每個子串都是「某個前綴的後綴」——SAM 的視角
border既是真前綴又是真後綴的串border 的 border 還是 border（遞迴結構）
週期（period）\(s_i=s_{i+p}\) 對所有合法 \(i\)週期不必整除 \(n\)；整除的叫「循環節」
字典序逐字比較，前綴較小比較兩子串字典序 = hash 二分 LCP 後比下一字元

1.2 border 與週期的結構

  • fail 鏈 = 所有 border：\(s\) 的最長 border 是 \(\mathrm{fail}[n]\)，次長是 \(\mathrm{fail}[\mathrm{fail}[n]]\)，……沿鏈走完恰好枚舉所有 border。這是 2.2 一切應用的來源。
  • Weak Periodicity Lemma：若 \(p,q\) 都是週期且 \(p+q\le n\)，則 \(\gcd(p,q)\) 也是週期。直覺：兩個週期互相「傳染」，輾轉相減。
  • border 的等差結構：\(s\) 的所有 border 長度可以分成 \(O(\log n)\) 個等差數列——長度超過 \(n/2\) 的 border 兩兩差距形成等差。這在 ★★★★★ 的 border 理論題（2.11.5）才會用到，此處先陳述。

1.3 三種匹配思維

思維代表工具核心資訊特長
滑動比對KMP、Z、Manacher「已匹配的部分」的結構（border／Z-box／迴文鏡像）線性、確定性、常數小
指紋比對多項式 hash子串 → 數字，\(O(1)\) 比相等萬用膠水：能二分、能塞進 set、能配任何結構
自動機Trie、ACAM、SAM、PAM把所有狀態顯式建成圖多模式、子串全體統計、在圖上跑 DP
💡 選型的第一直覺

先問「我要的答案是關於一個串、兩個串、還是所有子串？」一個模式串 → 滑動比對就夠；答案需要「任意兩個子串比較」→ 指紋；答案是「對所有子串／所有迴文／所有模式串」的統計 → 自動機或後綴結構。完整決策樹見 第 4 章。

1.4 複雜度與資料範圍反推

資料範圍暗示
\(|s|\le 10^6\)線性家族：KMP／Z／Manacher／SAM／PAM；\(O(n\log n)\) 的 SA 也常能過
\(|s|\le 10^5\)\(O(n\log n)\) 隨便用：hash+二分、SA、SAM×資料結構
\(n \le 5000\)\(O(n^2)\) 可行：暴力枚舉中心、區間 DP、\(O(n^2)\) hash 表
\(\Sigma|p_i|\le 10^6\)（多模式）AC 自動機；注意字元集大小影響 goto 表空間
強制線上SAM（天然增量）；hash／SA 偏離線

1.5 實作工具箱

  • 下標慣例：本指南 KMP／Z 用 0-indexed 字串 + \(\mathrm{fail}[i]\) 表「長度」；SA 用 1-indexed。每份模板固定自己的慣例，不要現場換。
  • 字元集映射：`c - 'a'` 之前先確認題目字元集；出現數字或大寫時映射表要換（第 3 章高頻坑）。
  • 多測清空：Trie／ACAM 的節點陣列若整包 memset，複雜度變 \(O(T\cdot \text{SIZE})\)——要只清用過的節點，或用版本號打標記。
  • std::string 拷貝：函式傳參用 const string&；`substr` 是 \(O(len)\) 拷貝，迴圈裡呼叫直接退化平方。
  • 讀入：\(10^6\) 級字串用 scanf("%s") 或關同步的 cin；getline 注意殘留換行。

常見誤解與除錯順序

  1. 0-based 與 1-based 公式混用，尤其是長度、右端點與回退位置。
  2. 把子串和子序列混為一談，或忘記空字串、單字元與全部相同字元。
  3. 只背建表程式，無法手算一個短字串，因此失配時不知道該跳去哪裡。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
