2.4 Manacher（線性迴文半徑）

💡 核心想法

對每個中心 \(i\) 求最長迴文半徑 \(p[i]\)。關鍵觀察：若 \(i\) 落在某個已知大迴文 \([L,R]\)（中心 \(C\)）內，則 \(i\) 關於 \(C\) 的鏡像位置 \(i' = 2C-i\) 的半徑可以直接繼承（截斷到不超出 \(R\)），只有頂到 \(R\) 才往外擴張。與 Z 函數的匹配盒一模一樣的均攤論證：\(R\) 只會右移 ⇒ \(O(n)\)。

🔁 插 '#' 統一奇偶

迴文有奇長（單中心）與偶長（雙中心）兩種。把 \(s\) 變成 ^#a#b#c#$（頭尾哨兵防越界）：新串裡每個位置的迴文都是奇長，且新串半徑 \(p[i]\)−1 = 原串該迴文的長度；新串迴文的端點必是 '#'，換算回原串下標為 \((i-p[i])/2\)（0-indexed 起點）。這步換算是 Manacher 唯一的細節坑，建議推導一次寫進模板註解。

🔎 適用特徵
  • 只要最長迴文子串、迴文子串總數、「以 \(i\) 開頭／結尾的最長迴文」→ Manacher 一遍搞定。
  • 要把每個本質不同迴文當對象統計（每個迴文各出現幾次）→ 升級 PAM。
  • \(n\le 5000\) 時中心擴展暴力 \(O(n^2)\) 完全夠——別殺雞用牛刀。
模板（C++）：Manacher 
// t = "^#a#b#…#$"；p[i] = 以 i 為中心的迴文半徑（含自身）
int manacher(const string& text, vector<int>& p) {
    string padded = "^";
    for (char c : text) {
        padded += '#';
        padded += c;
    }
    padded += "#$";
    int n = padded.size();
    p.assign(n, 0);
    int center = 0, right = 0, best = 0;
    for (int i = 1; i < n - 1; i++) {
        if (i < right) {
            p[i] = min(p[2 * center - i], right - i);   // 鏡像繼承
        }
        while (padded[i + p[i] + 1] == padded[i - p[i] - 1]) {
            p[i]++;
        }
        if (i + p[i] > right) {
            center = i;
            right = i + p[i];
        }
        best = max(best, p[i]);                     // p[i] = 原串迴文長
    }
    return best;
}
★★★洛谷 P3805 Manacher 模板Manacher

求最長迴文子串長度，\(|s|\le 1.1\times 10^7\)。

思路

資料規模直接封殺 \(O(n\log n)\) 以上的做法——這就是 Manacher 存在的理由。注意 \(10^7\) 級字串的讀入與記憶體（p 陣列用 int、字串預留 2n+3）。

★★LC 5 最長迴文子串Manacher中心擴展

回傳最長迴文子串本身（\(n\le 1000\)）。

思路

\(n\le 1000\) 用 \(O(n^2)\) 中心擴展即可（每個中心向兩側擴），Manacher 當練習。先寫暴力再寫 Manacher，能清楚看到「鏡像繼承」到底省了哪些擴張。

★★LC 647 迴文子串數目Manacher計數

數 s 有多少個迴文子串（位置不同算不同）。

思路

以 \(i\) 為中心的迴文有 \(\lceil p[i]/2\rceil\) 個（新串座標），答案 \(=\sum\lceil p[i]/2\rceil\)。半徑陣列天生就是「每個中心的迴文計數」——這是 Manacher 比中心擴展多送的資訊。

★★★★洛谷 P1659 [國家集訓隊] 拉拉队排練Manacher貪心

取出 \(k\) 個最長的奇長迴文子串，求長度乘積（mod）。

思路

Manacher 得出每個中心的最長奇迴文長 → 開桶 cnt[len]。關鍵：長度 \(len\) 的迴文存在 ⇒ 長度 \(len-2\) 的也存在（同中心縮一圈），所以從最長往短做後綴和即得「長度 ≥ len 的迴文個數」，再由長到短貪心取滿 \(k\) 個、快速冪算乘積。「半徑 → 桶 → 後綴和」三連是 Manacher 計數題的標準管線。

★★★★洛谷 P4555 [國家集訓隊] 最長雙迴文串ManacherDP

求最長的「兩個迴文串拼接」形式的子串。

思路

令 \(f[i]\)＝以 \(i\) 開頭的最長迴文、\(g[i]\)＝以 \(i\) 結尾的最長迴文，答案 \(=\max_i g[i]+f[i+1]\)。從 \(p[]\) 推 \(f,g\)：每個迴文 \([l,r]\) 應更新 \(f[l]\) 與 \(g[r]\)，直接做是平方——利用「同中心縮短仍是迴文」，在 Manacher 擴張時只記邊界值，再左右各掃一遍遞推補全（\(f[i]\ge f[i-1]-2\)）。把「區間貢獻」攤成「掃描遞推」是這題最值得帶走的技巧。

⏫ 進階去向

Manacher 給你「每個中心最長半徑」，但迴文之間的包含／後綴關係它不維護。要對每個本質不同迴文計數、做迴文分解，升級到 2.9 迴文自動機。
