初學者閱讀指南2.8 後綴自動機（SAM）

這頁要解決什麼問題？

先問：字串比較中有哪些前綴、後綴或子串資訊被反覆重算？失配後能否利用已知結構跳過位置？

本頁聚焦「endpos 等價類、parent 樹、線上構造與應用地圖」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：從每個起點重新逐字比較，最壞會反覆檢查相同字元，讓長字串退化到 O(nm) 或 O(n²)。

核心直覺

字串演算法的核心常是記住『已經匹配了多少』或『哪些片段相同』，失敗時不必完全從零開始。

閱讀程式前先找不變量

先固定下標制與陣列定義；每個 pi、z、fail、len 或 link 值都必須能用一句話精確描述。

先猜猜看：什麼時候該想到這頁？

大量模式匹配、重複子串、border、週期、迴文、前綴統計或字典序比較。

最小可手算例子

  1. 依序加入 "a"、"b"、"a"；SAM 每步新增代表所有新後綴的狀態。
  2. 最後一步新增的不同子串是 "a"、"ba"、"aba" 中尚未出現者。
  3. 狀態代表 endpos 相同的一群子串，不是單一子串。

2.8 後綴自動機（SAM）

💡 核心想法

SAM 是能接受 \(s\) 所有後綴的最小 DFA——同時也是「所有子串」的壓縮索引。基石概念是 endpos 等價類：兩個子串等價 ⟺ 它們在 \(s\) 中的結束位置集合相同。每個 SAM 節點恰是一個等價類，類內子串長度連續（\([\mathit{len}_{link(u)}+1,\ \mathit{len}_u]\)），節點數 \(\le 2n-1\)、轉移數 \(\le 3n-4\)：指數多個子串被壓成線性大小的圖。

🔁 兩張圖一起看

SAM 其實是兩個結構的疊加：① 轉移圖（DAG）——沿轉移邊走出的每條路徑對應一個不同子串；② parent 樹（link 樹）——\(link(u)\) 指向「endpos 嚴格更大的最長真後綴類」，樹上越往根子串越短、出現越多；endpos 集合 = 子樹內前綴節點的集合。做題時每一問先想清楚：這問走 DAG（子串本身）還是走 parent 樹（出現位置）？

🔎 適用特徵
  • 「每個子串出現幾次」「出現位置集合」——endpos 就是為此而生。
  • 需要線上增量（逐字元加入）——SA 做不到。
  • 兩串/多串的公共子串問題（在 SAM 上走匹配、或廣義 SAM）。
模板（C++）：線上增量構造 
struct SAM {
    int ch[N*2][26], link[N*2], len[N*2], last = 1, tot = 1;
    long long cnt[N*2];                    // 前綴節點 cnt=1，之後樹上求和
    void extend(int c) {
        int p = last, u = ++tot;
        len[u] = len[p] + 1;
        cnt[u] = 1;
        last = u;
        while (p && !ch[p][c]) {
            ch[p][c] = u;
            p = link[p];
        }
        if (!p) {
            link[u] = 1;
            return;
        }
        int q = ch[p][c];
        if (len[q] == len[p] + 1) {
            link[u] = q;
            return;
        }
        int w = ++tot;                     // clone：拆出「恰好 len[p]+1」的部分
        len[w] = len[p] + 1;
        link[w] = link[q];
        memcpy(ch[w], ch[q], sizeof ch[q]);
        while (p && ch[p][c] == q) {
            ch[p][c] = w;
            p = link[p];
        }
        link[q] = link[u] = w;
    }
};
// cnt 統計：按 len 基數排序後由長到短 cnt[link[u]] += cnt[u]

2.8.1 應用地圖（背下來）

問題做法
本質不同子串數\(\sum_u \big(len_u - len_{link(u)}\big)\)，或 DAG 路徑計數
子串 \(t\) 出現次數沿轉移走 \(t\) 到節點 \(u\)，答案 = 子樹 cnt 和（預先算好）
兩串最長公共子串對 A 建 SAM，B 在上面走：能走則走並 len+1，失配沿 link 退
字典序第 k 小子串DAG 上按字典序 DFS，用「子樹路徑數」導航
最小循環移位對 \(s+s\) 建 SAM，貪心走最小字元 \(n\) 步
★★★★★洛谷 P3804 SAM 模板SAMendpos

求「出現次數 > 1 的子串」中，出現次數 × 長度的最大值。

思路

每個節點代表一段長度的等價子串，出現次數 = 子樹 cnt 和，取類內最長 \(len_u\) 最有利：答案 \(=\max_{cnt_u>1} cnt_u\cdot len_u\)。統計順序務必按 \(len\) 由長到短（基數排序），不能隨意沿 link DFS 不排序——第 3 章高頻坑。

★★★★不同子串個數（SAM 版，對照 P2408）SAM去重計數

用 SAM 重做 2.7 的本質不同子串計數。

思路

每個節點貢獻 \(len_u - len_{link(u)}\)（類內長度連續！），一行求和。與 SA 的 \(\frac{n(n+1)}{2}-\sum height\) 對照：SA 按「排序後扣重複前綴」去重、SAM 按「endpos 等價類」去重——同一件事的兩張座標系。

★★★★SPOJ LCS 最長公共子串SAM匹配

兩字串的最長公共子串。

思路

對 A 建 SAM，讓 B 在上面「盡量不斷開地走」：有轉移就走、目前匹配長 +1；沒有就沿 link 退（匹配長縮到 \(len_{link}\)）直到有轉移或歸零。全程取匹配長最大值。體會 link 的語意：退一步 = 砍掉最舊的前綴、保留最長可延伸後綴。

★★★★★CF 235C Cyclical QuestSAM環形匹配

多個查詢串，問每個查詢串的所有旋轉在 s 中出現的總次數。

思路

對 s 建 SAM；查詢串倍長後在 SAM 上滑動匹配（維持匹配長 = |q|：超過就沿轉移前縮、不足沿 link 退），每個「匹配長恰為 |q|」的位置貢獻該節點 cnt；用訪問標記防止同一節點對同一查詢重複計（旋轉可能重複）。SAM 匹配指標控制的高級練習。

★★★★★洛谷 P3975 [TJOI2015] 弦論SAM第k小

求字典序第 \(k\) 小子串；t=0 本質不同、t=1 位置不同各算一個。

思路

DAG 上導航：預處理每個節點出發的「路徑數」（t=0 每節點權 1；t=1 權 = cnt），從初始節點按 a..z 試走，\(k\) 夠減就減、不夠就進入該子節點。同一張圖換權值就換語意——把「計數導航」寫成通用函式，兩問合一。

⏫ 進階去向

多串 → 廣義 SAM（2.11.2）；「endpos 集合本身」要可查詢（區間內出現次數）→ parent 樹上線段樹合併（2.11.3）；反串 SAM 的 parent 樹 = 後綴樹（2.11.4）。

常見誤解與除錯順序

  1. 0-based 與 1-based 公式混用，尤其是長度、右端點與回退位置。
  2. 把子串和子序列混為一談，或忘記空字串、單字元與全部相同字元。
  3. 只背建表程式，無法手算一個短字串，因此失配時不知道該跳去哪裡。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
