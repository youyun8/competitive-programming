初學者閱讀指南2.3 Z 函數（擴展 KMP）

這頁要解決什麼問題？

先問：字串比較中有哪些前綴、後綴或子串資訊被反覆重算？失配後能否利用已知結構跳過位置？

本頁聚焦「Z-box 線性構造、exkmp、Z 與 fail 的方向對照」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：從每個起點重新逐字比較，最壞會反覆檢查相同字元，讓長字串退化到 O(nm) 或 O(n²)。

核心直覺

字串演算法的核心常是記住『已經匹配了多少』或『哪些片段相同』，失敗時不必完全從零開始。

閱讀程式前先找不變量

先固定下標制與陣列定義；每個 pi、z、fail、len 或 link 值都必須能用一句話精確描述。

先猜猜看：什麼時候該想到這頁？

大量模式匹配、重複子串、border、週期、迴文、前綴統計或字典序比較。

最小可手算例子

  1. 字串 "ababa" 的 Z 值是 [5,0,3,0,1]。
  2. Z[2]=3 表示從位置 2 開始的 "aba" 與整串前綴相同。
  3. Z-box 讓落在已知匹配區間內的位置先繼承資訊，再向外擴展。

2.3 Z 函數（擴展 KMP）

💡 核心想法

\(z[i]\)＝\(s\) 與其後綴 \(s_{i..n}\) 的最長公共前綴（LCP）長度。維護「最靠右的匹配盒」\([L,R]\)（\(s_{L..R}\) 是 \(s\) 的前綴的副本）：對 \(i\le R\)，位置 \(i\) 在盒內對應前綴中的位置 \(i-L+1\)，可以先免費繼承 \(z[i-L+1]\)（截斷到 \(R\)），只有頂到盒右緣才需要往外逐字擴張——每次擴張都把 \(R\) 右推，\(R\) 不減 ⇒ 線性。

🔎 適用特徵：Z 還是 fail？
  • fail 回答「以 \(i\) 結尾的前綴後綴結構」；\(z\) 回答「從 \(i\) 開始能配多長的前綴」——方向相反、資訊等價（可互相換算）。
  • 題目句型是「每個位置往後能匹配多長」「前綴 \(P\) 在哪些位置整段出現」→ Z 直觀得多。
  • 模式對文本：對 \(u = p + \# + t\) 求 z，\(z[i] = m\) 的位置就是出現位置——單模式匹配的 Z 寫法。
模板（C++）：Z 函數 
// s 為 0-indexed，z[0] = n（慣例）
void zfunc(const string& text, int* z) {
    int n = text.size();
    z[0] = n;
    for (int i = 1, left = 0, right = 0; i < n; i++) {   // [left,right] 最右匹配盒
        z[i] = (i < right) ? min(z[i - left], right - i) : 0; // 盒內先繼承
        while (i + z[i] < n && text[z[i]] == text[i + z[i]]) {
            z[i]++;
        }
        if (i + z[i] > right) {
            left = i;
            right = i + z[i];
        }
    }
}
★★★洛谷 P5410 擴展 KMP／Z 函數模板Z函數

求 \(b\) 的 z 陣列，以及 \(b\) 與 \(a\) 每個後綴的 LCP（即 exkmp）。

思路

第二問對 \(u=b+\#+a\) 跑同一套盒子邏輯即可（或寫標準 exkmp 雙串版）。模板題但下標細節多，是檢驗「盒內繼承要不要截斷」理解的試金石：min(z[i-L], R-i) 少了 min 就是經典 WA。

★★★LC 2223 構造字串的總得分和Z函數

對每個後綴 \(s_{i..n}\)，得分 = 它與 \(s\) 的 LCP；求總分。

思路

題目就是 \(\sum z[i]\) 的直譯——最裸的 Z 應用。適合當作寫完模板後的第一發驗證。

★★★★CF 432D Prefixes and SuffixesZ函數計數

對每個「既是前綴又是後綴」的長度，求它在 \(s\) 中的出現次數。

思路

「既是前綴又是後綴」的長度 \(k\) ⟺ \(z[n-k+1]=k\)（後綴整段匹配前綴）。「前綴長 \(k\) 的出現次數」= \(\#\{i: z[i]\ge k\}\) —— 對 z 值開桶做後綴和即得所有答案。Z 的可加工性示範：z 陣列不只回答匹配，還能整批統計。fail 鏈版解法同樣成立，對照練習。

★★★★CF 1968G1 Division + Score（Easy）Z函數枚舉

把 \(s\) 切成 \(k\) 段，得分為兩兩段間 LCP 之和的變體；固定 \(k\) 求最佳切法。

思路

答案由「每段都以某公共前綴開頭」主導：二分/枚舉公共前綴長 \(L\)，用 z 陣列數有多少個位置 \(i\) 滿足 \(z[i]\ge L\) 且互不重疊（貪心隔 \(L\) 取）。「z 值 ≥ L 的位置集合」+ 貪心選點是 Z 題的常見二段式。

⚠ 高頻坑

① \(z[0]\) 慣例上設為 \(n\)，但有些題把「自己跟自己」排除——先讀清楚。② Z 與 fail 混用：想沿「所有 border」枚舉時要用 fail 鏈，z 陣列沒有這個鏈結構。③ 拼接寫法 \(p+\#+t\) 的分隔符必須是兩串都不出現的字元。

常見誤解與除錯順序

  1. 0-based 與 1-based 公式混用，尤其是長度、右端點與回退位置。
  2. 把子串和子序列混為一談，或忘記空字串、單字元與全部相同字元。
  3. 只背建表程式，無法手算一個短字串，因此失配時不知道該跳去哪裡。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
