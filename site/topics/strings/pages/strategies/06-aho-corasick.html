2.6 AC 自動機（多模式匹配）

💡 核心想法

把所有模式串插進 Trie，再給每個節點加 fail 指標：指向「目前串的最長真後綴中，是某模式串前綴的那一個」——Trie 版的 KMP fail。文本在自動機上走一遍，每個位置的節點代表「以此結尾的最長模式前綴匹配」，沿 fail 鏈能枚舉所有此刻結束的模式串。多模式匹配 \(O(|t| + \Sigma|p_i|)\)。

🔁 兩個關鍵改寫

① Trie 圖（goto 表）：BFS 建 fail 時順手把不存在的轉移補成 ch[fail[u]][c]——失配跳躍被展開成 \(O(1)\) 轉移，自動機成為真正的 DFA（DP 可用）。② fail 樹：所有 fail 邊反向構成一棵樹；「節點 \(u\) 的串出現在文本中」⟺ 文本走過的某個節點在 fail 樹上位於 \(u\) 的子樹內。於是「每個模式串出現幾次」= 把文本經過的每個節點打 +1，fail 樹上做一次子樹和——從逐位置跳 fail 的 \(O(n\sqrt{n})\)／最壞平方，變成一次 DFS 的 \(O(n)\)。

🔎 適用特徵
  • 多個模式串在文本中的出現（存在性／次數／位置）。
  • 「文本不含任何違禁詞」類的計數／構造 → ACAM 上 DP。
  • 單模式就別上 ACAM——KMP 碼量五分之一。
模板（C++）：建 Trie 圖 + fail 
int ch[N][26], fail[N], tot = 0;
vector<int> ends[N];               // 節點 u 結束的模式串編號
void build() {                     // BFS：層序保證 fail[u] 先於 u 處理
    queue<int> q;
    for (int c = 0; c < 26; c++) if (ch[0][c]) q.push(ch[0][c]);
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int c = 0; c < 26; c++) {
            int& v = ch[u][c];
            if (v) { fail[v] = ch[fail[u]][c]; q.push(v); }
            else   v = ch[fail[u]][c];        // 補成 Trie 圖：O(1) 轉移
        }
    }
}
★★★洛谷 P3808 AC 自動機（簡單版）ACAM

問有多少個模式串在文本中出現過（存在性）。

思路

文本走自動機，每到一個節點沿 fail 鏈上跳、把沿路的結束標記收掉並標已訪問（訪問過就停，保證每節點只清一次）→ 均攤線性。這題允許「跳 fail + 剪枝」的寫法；下一題會逼你升級。

★★★洛谷 P3796 AC 自動機（加強版）ACAM計數

找出現次數最多的模式串（次數要精確）。

思路

規模仍小，可以「每步沿 fail 鏈全部 +1」硬數；但正確的思維是先意識到這是 \(O(n\cdot\text{深度})\) 的定時炸彈——為二次加強版鋪路。

★★★★洛谷 P5357 AC 自動機（二次加強版）ACAMfail樹

求每個模式串的出現次數，規模卡掉跳 fail 的做法。

思路

fail 樹技法的模板題：文本經過的每個節點 val[u]++，結束後在 fail 樹上自底向上累加（BFS 逆序即拓撲序，不必真建樹），每個模式串答案 = 其結束節點的子樹和。「打標記 + fail 樹子樹和」是 ACAM 計數的最終形態，此後所有變形（帶權、帶位置）都是它的裝飾。

★★★★洛谷 P4052 [JSOI2007] 文本生成器ACAMDP

數長度 \(m\) 的大寫字母串中，至少包含一個給定單詞的個數。

思路

補集轉化：總數 \(26^m\) 減去「一個都不含」。後者是自動機上 DP：\(f[i][u]\)＝走了 \(i\) 步、位於節點 \(u\)、從未經過危險節點的方案數；危險節點 = 自身是結尾或 fail 鏈上有結尾（建 fail 時 danger[u] |= danger[fail[u]] 順手傳遞）。ACAM 上 DP 的句型：狀態 =（步數, 節點），Trie 圖保證轉移 \(O(|\Sigma|)\)。

★★★★★洛谷 P2414 [NOI2011] 阿狸的打字機ACAMfail樹BIT

多次詢問：第 \(x\) 個串在第 \(y\) 個串中出現幾次。

思路

「\(x\) 在 \(y\) 中出現次數」= \(y\) 在 Trie 上的路徑節點中，落在 fail 樹上 \(x\) 結束節點子樹內的個數。離線：按 Trie DFS 走（進節點 +1、離開 −1 維護「目前路徑」），走到某個 \(y\) 的結尾時回答其所有詢問——fail 樹子樹 = DFS 序區間，用 BIT 查區間和。雙樹配合（Trie 樹上走、fail 樹上查）+ 離線 + DFS 序，NOI 級 ACAM 的集大成之作。

⚠ 高頻坑

① 忘記把不存在的轉移補成 Trie 圖，DP 時就得手寫跳 fail——要嘛補圖要嘛跳鏈，混用必錯。② fail 樹統計要按 BFS 逆序或建樹 DFS，不能沿 fail 鏈逐個上跳累加（平方）。③ 多測清空：節點池、fail、ends 只清 \(0..tot\)。
