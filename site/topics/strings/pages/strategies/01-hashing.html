2.1 字串雜湊（Polynomial Hashing）

💡 核心想法

把字串看成 \(B\) 進位大數對質數 \(M\) 取模：

\[ h(s)=\left(\sum_{i=1}^{n} s_i\cdot B^{\,n-i}\right)\bmod M. \]

預處理前綴 hash 與 \(B\) 的冪次後，任意子串的 hash 可 \(O(1)\) 算出：\(h(s_{l..r}) = h_{r} - h_{l-1}\cdot B^{\,r-l+1}\)。於是「比較兩子串是否相等」從 \(O(n)\) 變 \(O(1)\)——不等必真、相等大概率真。hash 是字串工具箱裡的萬用膠水：能塞進 set 去重、能當二分的判定、能跟任何資料結構組合。

🔎 適用特徵
  • 需要大量「子串相等？」判定，且可以容忍極小出錯機率。
  • 想把字串問題轉成「數字問題」：去重、計數、當 map 的 key。
  • 「最長公共前綴」「比較子串字典序」→ hash + 二分（見下）。
  • 正確性要求絕對嚴格（構造題、對抗測資）時，換 KMP／SA 等確定性工具。
⚠ 防卡三原則

① 別用自然溢位（mod \(2^{64}\)）：Thue–Morse 串可以構造出必然碰撞的測資，CF 上被卡是日常。② 模數用大質數（如 \(10^9+7\)、\(998244353\)），要更穩就雙模數各算一份。③ base 隨機化：比賽開始時 B = mt19937(seed)() % 233 + 131。單模數隨機 base 下，比較 \(k\) 次的碰撞率約 \(k\cdot n/M\)（生日悖論估算），\(10^6\) 次比較配 \(10^9\) 級模數已略嫌緊，雙模最安心。

模板（C++）：前綴 hash + O(1) 子串 
typedef unsigned long long ull;
const ull MOD = 1e9 + 7;
ull base, prefix_hash[N], power[N];          // base 開賽隨機化

void build(const char* text, int n) {         // text 為 1-indexed
    power[0] = 1;
    for (int i = 1; i <= n; i++) {
        prefix_hash[i]  = (prefix_hash[i-1] * base + text[i]) % MOD;
        power[i] = power[i-1] * base % MOD;
    }
}
ull get(int left, int right) {                    // text[left..right] 的 hash
    return (prefix_hash[right] + MOD * MOD -
        prefix_hash[left-1] * power[right-left+1]) % MOD;
}

2.1.1 hash + 二分：LCP 與子串字典序

兩個後綴（或任兩個子串）的最長公共前綴滿足可二分性：「前 \(k\) 字元相等」隨 \(k\) 單調。二分 + \(O(1)\) hash 比較 = \(O(\log n)\) 求 LCP；求出 LCP 後比較下一個字元即得字典序。這讓「對所有後綴排序」可以 sort + \(O(\log n)\) 比較器完成——\(O(n\log^2 n)\) 的懶人後綴數組。

★洛谷 P3370 字串雜湊模板hash

給 \(n\) 個字串，數本質不同的個數。

思路

每個串算 hash 丟進陣列，排序去重（或 set）。模板檢核題：驗證你的 hash 寫法與防碰撞配置能不能直接通過。

★LC 28 找出字串中第一個匹配項的下標hashKMP

在 haystack 中找 needle 第一次出現的位置（單模式匹配的最小例）。

思路

hash 解法：算出 needle 的 hash，滑過 haystack 的每個等長窗口 \(O(1)\) 比對。這題是三種匹配思維的練習場——暴力 \(O(nm)\)、hash \(O(n+m)\)、KMP \(O(n+m)\)（見 2.2）都寫一遍，理解各自「省掉了哪些重複比較」。

★★LC 187 重複的 DNA 序列hash滑動窗口

找出所有出現超過一次的長度 10 子串。

思路

定長窗口滑過去，每個窗口的 hash 丟進 unordered_map 計數。字元集只有 ACGT → 可以用 2 bit 編碼把 hash 做成精確的 20-bit 整數（無碰撞），體會「字元集小時 hash 可以是無損編碼」。

★★★CF 1200E Compress Wordshashborder

依序合併單詞：每次把新詞接上結果串，重疊部分（結果的後綴 = 新詞的前綴）只留一份。

思路

對每個新詞 \(w\)，要找最大的 \(k\le\min(|res|,|w|)\) 使 \(res\) 的後綴 \(k\) 等於 \(w\) 的前綴 \(k\)——枚舉 \(k\) 由大到小，hash \(O(1)\) 判定。注意這裡不能二分（「長度 \(k\) 可行」不單調），只能枚舉；總複雜度 \(O(\Sigma|w|)\)。KMP 解法：對 \(w + \# + res\text{ 尾段}\) 求 fail 也行——一題兩解對照 hash 與 border 的等價表達。

★★★★LC 1044 最長重複子串hash二分答案

找出現至少兩次的最長子串。

思路

「存在長度 \(k\) 的重複子串」對 \(k\) 單調 → 二分 \(k\)；判定時把所有長度 \(k\) 窗口的 hash 丟進 unordered_set 看是否撞到。\(O(n\log n)\)。這是「hash 把判定變 \(O(n)\)，二分把最優化變判定」的招牌組合；SA 的 height 也能做（2.7），對照兩解。

程式碼（C++） 
bool ok(int k, int n) {                 // 存在長 k 的重複子串？
    unordered_set<ull> seen;
    for (int start = 1; start + k - 1 <= n; start++) {
        if (!seen.insert(get(start, start + k - 1)).second) {
            return true;
        }
    }
    return false;
}
// 主程式：對 k 二分，ok(k) 單調
★★★★洛谷 P2852 Milk Patternshash二分答案

找至少出現 \(k\) 次（可重疊）的最長子串。

思路

LC 1044 的推廣：二分長度，hash 計數改成「某個 hash 出現 \(\ge k\) 次」。本題是三解對照組的主角——hash 二分 \(O(n\log n)\)、SA height 分組（2.7）、SAM 的 endpos 大小（2.8），第 4 章拿它做完整比較。

⏫ 進階去向

hash 的上限在於它只回答「相等嗎」。需要「所有子串的結構」（排序、本質不同計數、出現次數全表）時，升級到 SA／SAM；需要確定性線性匹配時用 KMP。
