初學者閱讀指南2.9 迴文自動機（PAM／迴文樹）

這頁要解決什麼問題？

先問：字串比較中有哪些前綴、後綴或子串資訊被反覆重算？失配後能否利用已知結構跳過位置？

本頁聚焦「本質不同迴文至多 n 個、雙根增量構造、出現次數統計」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：從每個起點重新逐字比較，最壞會反覆檢查相同字元，讓長字串退化到 O(nm) 或 O(n²)。

核心直覺

字串演算法的核心常是記住『已經匹配了多少』或『哪些片段相同』，失敗時不必完全從零開始。

閱讀程式前先找不變量

先固定下標制與陣列定義；每個 pi、z、fail、len 或 link 值都必須能用一句話精確描述。

先猜猜看：什麼時候該想到這頁？

大量模式匹配、重複子串、border、週期、迴文、前綴統計或字典序比較。

最小可手算例子

  1. 讀 "ababa" 時，本質不同迴文依序出現 a、b、aba、bab、ababa。
  2. 每加入一字元，只可能新增一個以目前位置結尾的最長新迴文。
  3. fail 指向目前迴文的最長真迴文後綴。

2.9 迴文自動機（PAM／迴文樹）

💡 核心想法

一個字串本質不同的迴文子串至多 \(n\) 個（每加一個字元至多新增一個——新增的必是「以新字元結尾的最長迴文後綴」）。PAM 給每個本質不同迴文開一個節點：ch[u][c] 表示「在迴文 \(u\) 兩側各加一個 \(c\)」，\(fail(u)\)＝\(u\) 的最長迴文真後綴。兩個根：長度 0 的偶根與長度 −1 的奇根（讓單字元迴文 \(c\) 由奇根加 \(c\) 得到，天然統一奇偶——比 Manacher 插 '#' 更優雅）。

🔁 增量構造為什麼線性？

加入 \(s_i\) 時，從 last 沿 fail 鏈找第一個節點 \(u\) 使 \(s_{i-len_u-1} = s_i\)（即 \(s_i u s_i\) 是迴文）。last 的深度每次至多 +1、每次跳 fail 至少 −1——同一套勢能論證第三次出現（KMP、Z、PAM），此時你應該能自己講完它。

🔎 適用特徵（vs Manacher）
  • Manacher 給「每個中心的最長半徑」；PAM 給「每個本質不同迴文一個節點」＋出現次數＋後綴結構。
  • 題目要「每種迴文出現幾次」「迴文的迴文後綴」「迴文分解」→ PAM。
  • 只要最長迴文／計數總數 → Manacher 就夠（碼量一半）。
模板（C++）：PAM 增量構造 
struct PAM {
    int ch[N][26], fail[N], len[N], cnt[N], tot, last;
    char s[N];
    int n;
    void init() {
        s[0] = -1;
        n = 0;
        tot = 1;
        last = 0;
        fail[0] = 1;
        len[0] = 0;
        len[1] = -1;      // 0=偶根, 1=奇根
    }
    int getfail(int u) {                             // 找 s[n-len-1]==s[n]
        while (s[n - len[u] - 1] != s[n]) {
            u = fail[u];
        }
        return u;
    }
    void extend(char c) {
        s[++n] = c;
        int p = getfail(last), u = ch[p][c - 'a'];
        if (!u) {
            u = ++tot;
            len[u] = len[p] + 2;
            fail[u] = ch[getfail(fail[p])][c - 'a']; // 先求 fail 再掛邊！
            ch[p][c - 'a'] = u;
        }
        last = u;
        cnt[u]++;                           // 出現次數打點
    }
    // 最後：按 tot..2 逆序 cnt[fail[u]] += cnt[u]（fail 樹子樹和）
};
★★★★洛谷 P5496 迴文自動機模板PAM

對每個前綴，求「以該位置結尾的迴文子串個數」（強制線上）。

思路

「以 \(i\) 結尾的迴文數」= last 在 fail 樹上的深度（fail 鏈上每個節點都是一個迴文後綴）——建構時順手 dep[u]=dep[fail[u]]+1。強制線上正好展示 PAM 的增量本性。

★★★★洛谷 P3649 [APIO2014] 回文串PAMendpos

求「出現次數 × 長度」最大的迴文子串。

思路

PAM 版的 P3804：extend 時 cnt[last]++，收尾逆序沿 fail 累加（一個迴文出現 ⇒ 它的迴文後綴也出現），答案 \(=\max cnt_u\cdot len_u\)。寫過 SAM 模板的話這題是十分鐘的事——「打點 + fail 樹子樹和」在 ACAM／SAM／PAM 三處是同一招。

★★★★★洛谷 P4287 [SHOI2011] 雙倍迴文PAMtrans指標

找最長的「\(ww^Rww^R\)」形（自身迴文且前半也是迴文）子串。

思路

雙倍迴文 ⟺ 節點 \(u\) 長度是 4 的倍數，且存在長度恰為 \(len_u/2\) 的迴文後綴。維護 trans 指標：\(trans(u)\)＝\(u\) 的長度 \(\le len_u/2\) 的最長迴文後綴（構造時仿 fail 多跳一條鏈）。檢查 \(len_{trans(u)} = len_u/2\) 即可。展示「在 PAM 上加自訂指標」的擴充手法。

★★★★★CF 906E Reverses（提及）PAMDP

把 t 切成最少段、每段可選擇翻轉後拼出 s——等價於對 \(s_1t_1s_2t_2\dots\) 交錯串做偶迴文分解。

思路

「最少迴文分解」是 PAM 的招牌難題：利用 fail 鏈上長度的等差數列結構（\(O(\log n)\) 條）加 series link，把 DP 轉移壓到均攤 \(O(n\log n)\)。與 border 的等差結構（2.11.5）同源，作為 PAM 上限的路標，第一次讀懂框架即可。

⚠ 高頻坑

① 求 fail[u] 必須在把 \(u\) 掛上 ch[p][c] 之前——先掛邊會讓 getfail 找到自己。② 奇根 len=−1 讓 s[n-(-1)-1]=s[n] 恆真，是單字元迴文的入口，別「修正」它。③ cnt 統計必須逆序（子節點先加給 fail），順序錯答案偏小。

常見誤解與除錯順序

  1. 0-based 與 1-based 公式混用，尤其是長度、右端點與回退位置。
  2. 把子串和子序列混為一談，或忘記空字串、單字元與全部相同字元。
  3. 只背建表程式，無法手算一個短字串，因此失配時不知道該跳去哪裡。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
