初學者閱讀指南2.2 KMP、失配函數與週期

這頁要解決什麼問題？

先問：字串比較中有哪些前綴、後綴或子串資訊被反覆重算？失配後能否利用已知結構跳過位置？

本頁聚焦「fail 鏈 = 所有 border、最小週期 n−fail[n]、KMP 自動機」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：從每個起點重新逐字比較，最壞會反覆檢查相同字元，讓長字串退化到 O(nm) 或 O(n²)。

核心直覺

字串演算法的核心常是記住『已經匹配了多少』或『哪些片段相同』，失敗時不必完全從零開始。

閱讀程式前先找不變量

先固定下標制與陣列定義；每個 pi、z、fail、len 或 link 值都必須能用一句話精確描述。

先猜猜看：什麼時候該想到這頁？

大量模式匹配、重複子串、border、週期、迴文、前綴統計或字典序比較。

最小可手算例子

  1. 模式 "abab" 的最長真前後綴長度依序是 0,0,1,2。
  2. 匹配到 "aba" 後失配，不必退回開頭，跳到長度 1 的 border。
  3. fail/pi 保存的是可重用的已匹配前綴。

2.2 KMP、失配函數與週期

💡 核心想法

定義 \(\mathrm{fail}[i]\)＝前綴 \(s_{1..i}\) 的最長 border 長度。匹配失敗時，已匹配部分的 border 告訴我們「模式串可以直接滑到哪」而不必回頭重比——文本指標永不後退，這就是 KMP 線性的來源。更重要的是 fail 陣列本身就是border／週期結構的完整地圖（1.2 的核心等式），一半以上的 KMP 題根本不做匹配，只讀 fail。

🔁 為什麼構造是均攤 O(n)？

構造時維護「目前匹配長度 \(j\)」：每一輪 \(j\) 至多 +1，而每次失配沿 fail 鏈跳都使 \(j\) 嚴格變小且 \(j\ge 0\)——把 \(j\) 當存款，總共存入 \(\le n\)，所以總共取出 \(\le n\)。勢能分析的入門範例，Z 函數與 Manacher 的線性論證同款。

🔎 適用特徵
  • 單模式匹配且要確定性線性（不怕被卡）。
  • 題目問循環節、週期、前綴=後綴——直接讀 fail，不用匹配。
  • 需要「匹配進度」當狀態 → KMP 自動機上 DP（連往 DP 主題 2.6.4）。
模板（C++）：fail 構造 + 匹配 
// p 為 1-indexed 模式串，長 m；fail[i] = p[1..i] 的最長 border 長
void build(const char* p, int m) {
    fail[1] = 0;
    for (int i = 2, j = 0; i <= m; i++) {
        while (j && p[i] != p[j+1]) {
            j = fail[j];
        }
        if (p[i] == p[j+1]) {
            j++;
        }
        fail[i] = j;
    }
}
void match(const char* t, int n, const char* p, int m) {
    for (int i = 1, j = 0; i <= n; i++) {
        while (j && t[i] != p[j+1]) {
            j = fail[j];
        }
        if (t[i] == p[j+1]) {
            j++;
        }
        if (j == m) {
            report(i - m + 1);
            j = fail[j];
        }  // 出現位置
    }
}

2.2.1 週期三定理（讀 fail 就有答案）

  • 最小週期 \(= n - \mathrm{fail}[n]\)。
  • \(s\) 由某循環節重複 \(\ge 2\) 次 ⟺ \((n - \mathrm{fail}[n]) \mid n\) 且 \(\mathrm{fail}[n]>0\)。
  • \(s_{1..i}\) 的所有 border：沿 \(\mathrm{fail}[i], \mathrm{fail}[\mathrm{fail}[i]],\dots\) 走到 0。
★★洛谷 P3375 KMP 模板KMP

輸出所有出現位置與整個 fail 陣列。

思路

裸模板。輸出 fail 的設計很好——強迫你確認自己的 fail 定義（長度制 vs 下標制）與題目一致。寫完後自問：while 裡為什麼是 j = fail[j] 而不是 j--？（答：跳到次長 border，中間的長度不可能匹配。）

★★LC 459 重複的子字串KMP週期

判斷 s 能否由某子串重複多次構成。

思路

週期定理二的直接應用：fail[n] > 0 && n % (n - fail[n]) == 0。另一個優雅解：\(s\) 在 \((s+s)\) 去頭去尾後仍出現 ⟺ 有循環節——兩解都值得懂。

★★★洛谷 P4391 [BOI2009] 無線傳輸KMP週期

s 是某未知串無限重複後截下的一段，求該未知串的最短長度。

思路

答案就是最小週期 \(n-\mathrm{fail}[n]\)——注意週期不需要整除 \(n\)（截斷可以停在循環節中間）。這題與 LC 459 成對：一個要整除、一個不用，把「週期 vs 循環節」的差別焊死在腦中。

★★★LC 214 最短迴文串KMP構造

在 s 前面加最少字元使整串成為迴文。

思路

等價於找 s 的最長迴文前綴。經典構造：令 \(u = s + \# + \mathrm{rev}(s)\)，則 \(\mathrm{fail}[|u|]\) 就是最長迴文前綴的長度（既是 \(s\) 的前綴、又是 \(\mathrm{rev}(s)\) 的後綴 = 反過來讀相同）。「拼接 + 分隔符 + 讀 fail」是 border 技巧的萬用句型，分隔符防止 border 跨界。

★★★CF 126B PasswordKMPborder

找最長的 t：t 是 s 的前綴、後綴，且還在中間（非頭尾處）出現過。

思路

候選只有 fail 鏈上的 border。「中間出現過」的判定：記錄 \(\mathrm{fail}[1..n-1]\) 中出現過的最大值 \(mx\)——若 \(\mathrm{fail}[n]\le mx\) 直接取，否則跳到 \(\mathrm{fail}[\mathrm{fail}[n]]\) 再試。體會「沿 fail 鏈枚舉所有 border」這個基本動作。

2.2.2 KMP 自動機

把「失配沿鏈跳」預先展開成轉移表：\(\delta(j,c)\)＝匹配長度 \(j\) 時讀入字元 \(c\) 後的新匹配長度。得到一台 \(O(m|\Sigma|)\) 的 DFA——這就是「文本裡不含／恰含模式串」計數 DP 的狀態機。

模板（C++）：KMP 自動機 
// go[j][c]：已匹配 j 個字元、讀入 c 之後匹配多長（j = 0..m）
for (int c = 0; c < 26; c++) {
    go[0][c] = (p[1] == 'a' + c);
}
for (int j = 1; j <= m; j++) {
    for (int c = 0; c < 26; c++) {
        go[j][c] = (j < m && p[j+1] == 'a' + c) ? j + 1
                 : go[fail[j]][c];               // 遞推繼承，O(1) 轉移
    }
}
★★★★★洛谷 P3193 [HNOI2008] GT 考試KMP自動機矩陣冪

數長度 \(n\le 10^9\) 的數字串中不含模式串 \(p\)（\(|p|\le 20\)）的個數。

思路

狀態 = KMP 自動機節點（匹配進度 \(0..m-1\)），轉移 = 讀一個數字。\(n\) 巨大 → 把轉移寫成 \(m\times m\) 矩陣後快速冪。與 DP 主題 2.12.7 共用的雙主題代表題：字串側貢獻自動機、DP 側貢獻矩陣加速。

⏫ 進階去向

fail 指標反向成樹＝失配樹（border 樹），「兩個前綴的最長公共 border」變成樹上 LCA → 2.11.1。多模式串的 KMP＝AC 自動機。

常見誤解與除錯順序

  1. 0-based 與 1-based 公式混用，尤其是長度、右端點與回退位置。
  2. 把子串和子序列混為一談，或忘記空字串、單字元與全部相同字元。
  3. 只背建表程式，無法手算一個短字串，因此失配時不知道該跳去哪裡。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
