4證明方法實戰模板

模板證明句型最適合的題型
交換論證找最優解第一個與貪心不同處，換成貪心選擇而不劣排序、排程、配對
領先法證明每個前綴中貪心的某指標都不落後任意解區間選取、覆蓋、掃描
Cut／Cycle Property把安全邊加入最優樹形成環，再換掉不更輕的邊MST 與生成樹變形
歸納＋不變量假設前 i 步保留最佳可延伸解，證明第 i+1 步後仍成立反悔堆、資料結構貪心
反證／最小反例取最早違反規則的最優解，局部修正得到矛盾字典序、規範化構造
✍️ 證明應明確指出「換誰」

以活動選擇為例：令 g 是右端點最早的區間，O 是任一最多區間解，o 是 O 的第一段。因 end(g)≤end(o)，用 g 替換 o 後，O 後面原本能接的每段仍能接，所以得到同樣大小且含 g 的最優解。刪去與 g 衝突的區間後，剩餘又是同型子問題。這比「g 留下最多空間，所以最優」多了交換對象、可行性與遞迴結構，才是完整證明。

⚠ 常見證明缺口
  • 只比較兩個局部成本，卻沒說交換後其他元素的成本為何不變。
  • 只證明貪心解「不錯」，沒與任一最優解建立不劣關係。
  • 比較器對每一對都有偏好，卻沒檢查關係可傳遞。
  • 歸納假設寫成「前 i 步最優」但沒有定義對哪個前綴、哪個指標最優。

4.1 交換論證五步驟模板

🧾 模板
  1. 設：設 OPT 為任一最優解，G 為貪心解，且 OPT ≠ G。
  2. 找：找出第一個分歧點 —— OPT 與 G 第一個不同的決策（或 OPT 中第一對「違反貪心排序鍵」的相鄰元素）。
  3. 換：把 OPT 在該處的選擇換成 G 的選擇（或交換那對相鄰元素），得到 OPT′。
  4. 證：證明 OPT′ 仍可行、且目標值不比 OPT 差。← 全部的工作量在這一步。
  5. 推：交換有限次後 OPT 變成 G 且始終不變差 ⇒ G 最優。∎

完整範例（活動選擇）：設 OPT 第一個活動為 j，G 第一個為 g（結束最早）。因 end(g) ≤ end(j)，把 OPT 中 j 換成 g：g 結束更早，不與 OPT 其餘活動衝突 ⇒ OPT′ 可行且數量相同。之後對子問題歸納。∎

4.2 領先論證模板

🧾 模板

定義「進度量」m(·)（如：前 i 個選擇後的結束時間 / 剩餘資源）。歸納證明：對每個 i，貪心解的 m(G, i) 不劣於任何解的 m(S, i)。步步領先 ⇒ 若 S 可行/達到 k，G 也能。適用於「貪心與對手逐步並排比較」自然的問題（活動選擇、跳躍遊戲、分發餅乾）。

4.3 對拍驗證器（競賽現場 5 分鐘保命工具）

寫不出證明時，可先用小資料窮舉對拍。以下以「選最多個互不重疊區間」示範：暴力枚舉所有子集，與右端點排序的貪心答案比較。

struct Interval {
    int left;
    int right;
};

int BruteForce(std::vector<Interval> intervals) {
    const int interval_count = static_cast<int>(intervals.size());
    int best = 0;
    for (int mask = 0; mask < (1 << interval_count); ++mask) {
        std::vector<Interval> chosen;
        for (int index = 0; index < interval_count; ++index) {
            if ((mask >> index) & 1) {
                chosen.push_back(intervals[index]);
            }
        }
        std::sort(chosen.begin(), chosen.end(),
                  [](const Interval& left, const Interval& right) {
                      return left.left < right.left;
                  });
        bool feasible = true;
        for (int index = 1; index < static_cast<int>(chosen.size()); ++index) {
            feasible &= chosen[index - 1].right <= chosen[index].left;
        }
        if (feasible) {
            best = std::max(best, static_cast<int>(chosen.size()));
        }
    }
    return best;
}

int Greedy(std::vector<Interval> intervals) {
    std::sort(intervals.begin(), intervals.end(),
              [](const Interval& left, const Interval& right) {
                  return left.right < right.right;
              });
    int answer = 0;
    int last_right = std::numeric_limits<int>::lowest();
    for (const Interval& interval : intervals) {
        if (interval.left >= last_right) {
            ++answer;
            last_right = interval.right;
        }
    }
    return answer;
}

int main() {
    std::mt19937 random_engine(123456789);
    std::uniform_int_distribution<int> size_distribution(1, 8);
    std::uniform_int_distribution<int> point_distribution(0, 10);
    for (int test = 0; test < 300; ++test) {
        std::vector<Interval> intervals;
        const int interval_count = size_distribution(random_engine);
        for (int index = 0; index < interval_count; ++index) {
            int left = point_distribution(random_engine);
            int right = point_distribution(random_engine);
            if (left > right) {
                std::swap(left, right);
            }
            intervals.push_back({left, right});
        }
        if (BruteForce(intervals) != Greedy(intervals)) {
            std::cout << "Counterexample found\n";
            return 0;
        }
    }
    std::cout << "300 tests passed\n";
    return 0;
}

技巧：值域開小（1–10）更容易撞出反例；找到反例後手動最小化它，反例會告訴你貪心錯在哪一類結構。
