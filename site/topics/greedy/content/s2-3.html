🧮 Smith's Rule：一步一步比較兩種順序

假設前面的工作都一樣，做完它們已經花了 T。現在只比較相鄰的工作 i、j；後面的工作無論交換前後都在同一時間開始，所以也不受影響。

先做 i 再做 j：i 在 T + pi 完成，j 在 T + pi + pj 完成。

cost(i, j) = wi(T + pi) + wj(T + pi + pj) = (wi + wj)T + wipi + wjpi + wjpj

先做 j 再做 i：

cost(j, i) = (wi + wj)T + wipi + wipj + wjpj

兩式都有 (wi + wj)T、wipi、wjpj；這就是「共同項」。相減後只剩：

cost(i, j) − cost(j, i) = wjpi − wipj

若先做 i 不會比較差，左邊必須 ≤ 0，因此 piwj ≤ pjwi。當權重皆為正時，兩邊除以 wiwj，便得到 pi/wi ≤ pj/wj：依 p/w 由小到大排序。

若某一對順序違反這條規則，交換它們不會增加成本；重複交換所有逆序對，就會得到排序後的最優序列。等號表示兩種順序同價。

C++ 比較器：用交叉相乘避免浮點誤差 
struct Job {
    long long processing_time;
    long long weight;
    int original_index;
};

bool SmithOrder(const Job& left, const Job& right) {
    const __int128 left_product =
        static_cast<__int128>(left.processing_time) * right.weight;
    const __int128 right_product =
        static_cast<__int128>(right.processing_time) * left.weight;
    if (left_product != right_product) {
        return left_product < right_product;
    }
    return left.original_index < right.original_index;
}

std::sort(jobs.begin(), jobs.end(), SmithOrder);

__int128 是 GNU C++ 擴充型別，兩邊都先轉型再相乘，避免 long long 乘法先溢位。最後用原輸入位置打破同價平手，只是讓輸出固定，不會改變目標值。

數值例：為什麼「工時短」不一定先做？

任務 A=(p=3,w=100)、B=(p=1,w=1)。A 先的成本為 100×3+1×4=304；B 先為 1×1+100×4=401。雖然 B 更短，A 的權重極高，Smith 比值 A=0.03、B=1，會正確地把 A 放前面。這也說明排序鍵必須從目標函數推導，而不是只憑「短作業優先」直覺。

💡 核心想法

單機/多機排程問題的統一解法：假設最優序列中相鄰兩個任務 i、j 交換，寫出兩種順序的目標值，令「i 在前不劣」解出不等式 — 這個不等式就是排序鍵。

🧾 推導模板

設 i 在 j 前、其餘不動。因為只有 i、j 的完成時刻改變，比較兩種順序下這兩項的目標貢獻即可。得到形如 f(i,j) ≤ f(j,i) 的可傳遞關係後，直接 sort。務必檢查關係可傳遞（通常能改寫成單一實數鍵值比較，如 p/w、min(a,b) 分類）。

★★★最小化加權完成時間（Smith's Rule）排程交換論證

單機 n 個任務，任務 i 耗時 p_i、權 w_i，最小化 Σ w_i × C_i（C_i 為完成時刻）。對應：洛谷 P1248 加工生產調度前置、AtCoder 各 ABC 排程題。

推導

相鄰交換 i、j：i 在前時兩者貢獻差為 w_j·p_i − w_i·p_j。令 i 在前不劣 ⇒ p_i/w_i ≤ p_j/w_j。所以按 p/w 遞增（性價比：短且重要優先）排序。特例 w 全等 → SJF 最短工時優先；p 全等 → 權大優先。

★★★★洛谷 P1248 加工生產調度（Johnson's Rule）排程交換論證

每個工件先過機器 A（耗時 a_i）再過機器 B（耗時 b_i），兩台機器各自序列處理，最小化總完工時間（makespan）。

思路

Johnson's Rule：min(a_i, b_j) ≤ min(a_j, b_i) 時 i 排 j 前。實作：把工件分兩組 — a < b 的按 a 遞增排在前，a ≥ b 的按 b 遞減排在後。直覺：A 快 B 慢的先上（讓 B 早點有活幹且不斷流），A 慢 B 快的殿後（結尾時 B 不用等）。這是三機以上 NP-hard、兩機卻有優美貪心的經典分界。

★★★★AtCoder DP Contest X - Tower排程交換論證貪心+DP

每個箱子有重量 w、承重 s、價值 v，疊塔使塔上每箱之上的總重 ≤ 其承重，最大化總價值。

提示

價值使純貪心失效（要選子集）— 但「選定子集後的最佳疊放順序」可以先用交換論證定死，再對這個順序做背包 DP。

思路

相鄰交換：i 疊在 j 下方可行性更寬 ⟺ w_i + s_i ≥ w_j + s_j。按 w+s 遞增排序（小的在上/先考慮），然後 dp[承重餘量] = 最大價值 做 0/1 背包。這是「交換論證定順序 + DP 選子集」的代表套路，競賽出現率極高（同型：CF 開會安排、洛谷 P1417 烹調方案）。
