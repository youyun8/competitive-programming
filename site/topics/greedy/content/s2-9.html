2.9 貪心 × 資料結構

🧰 把「策略」與「查詢器」分開

先在不考慮效率時寫出每步真正想找的對象，再選資料結構加速。資料結構不改變貪心規則，也不提供正確性；證明仍針對「為何應選這個對象」，複雜度才針對「如何快速找到它」。

每步需要的操作常用工具典型用途

動態維護最小／最大候選heap掃描線候選池、反悔
⚠ 更新必須維持查詢語意

例如 DSU 跳點的 find(x) 若定義為「\(\le x\) 的最大空位」，占用 \(t\) 後就必須令它指向 find(t - 1)；若只是 fa[t] = t - 1 而 find 沒正確壓縮，語意或效率都可能破壞。先把抽象操作寫成規格，再實作資料結構。

💡 核心想法

貪心策略本身簡單，但每一步的決策需要「快速找到／更新某個極值或空位」— 瓶頸在資料結構：並查集跳過已佔用位置、BIT/線段樹維護區間資訊、multiset 支援任意刪除。

★★★★截止期限任務調度（Job Sequencing）+ 並查集DSU排程

每個任務耗時一單位、截止時間為 \(d_i\)、報酬為 \(p_i\)，最大化總報酬。

思路

按報酬遞減排序，每個任務安排在「\(\le d_i\) 的最晚空閒時槽」（越晚越不擋別人）。找最晚空槽用並查集：find(d) 返回 \(\le d\) 的最晚空槽，佔用後令 fa[t] = t - 1，均攤複雜度接近 \(O(1)\)。正確性可用擬陣描述：「可行任務子集」構成 transversal matroid，按權降序能加就加必最優 — 2.11.1 的具體實例。

程式碼（C++） 
sort(jobs.begin(), jobs.end(), [](auto& a, auto& b) { return a.p > b.p; });
vector<int> fa(maxD + 1);
iota(fa.begin(), fa.end(), 0);
function<int(int)> find = [&](int x) {
    return fa[x] == x ? x : fa[x] = find(fa[x]);
};
long long ans = 0;
for (auto& j : jobs) {
    int t = find(min(j.d, maxD));
    if (t > 0) {
        ans += j.p;
        fa[t] = t - 1;
    }  // 佔用時槽 t
}
★★★★洛谷 P1250 種樹區間差分約束→貪心

區間 \([l_i,r_i]\) 內至少要種 \(t_i\) 棵樹，每個位置至多一棵，求最少總棵數。

思路

按右端點遞增排序，每個約束不足時從右端點往左補種（種得越右，越可能被後面的區間重複利用）。交換論證：把任何解中較左的樹右移到區間右端，不影響當前約束、只會幫助後續。加速：BIT 查區間和 + 並查集找「最右空位」。同型進階：洛谷 P2894、CF 志願者/餵牛系列。

從左到右喝藥水（可跳過），生命值不得為負，最多喝幾瓶？

思路
