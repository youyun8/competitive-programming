初學者閱讀指南2.4 反悔貪心（Regret Greedy）

這頁要解決什麼問題？

先問：能不能現在做一個局部選擇，而且保證不會害未來失去最佳答案？

本頁聚焦「先貪心接受，再用堆撤銷最差決策的萬用框架」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：暴力會枚舉每一步的所有選擇，形成龐大決策樹；貪心想證明其中大多數分支永遠不必探索。

核心直覺

貪心不是『看起來最好就選』，而是證明某個最佳答案可以被交換成包含目前選擇，且不會變差。

閱讀程式前先找不變量

每做一次選擇後，都要說清楚剩餘問題仍保留什麼性質，以及被丟掉的候選人為何不可能更好。

先猜猜看：什麼時候該想到這頁？

排序後逐一決策、要求字典序最小、選最多活動、成本可交換比較，或可撤銷最差選擇。

最小可手算例子

  1. 期限內暫時接受工作時長 [2,1,3]；總時間超標時反悔。
  2. 最大堆彈出目前最耗時的 3，保留 2、1。
  3. 反悔不是隨機撤銷，而是撤銷最能釋放資源的既有選擇。

2.4 反悔貪心（Regret Greedy）

🧠 heap 裡存的是「最便宜的修復動作」

反悔貪心通常維護一個前綴不變量：處理完前 \(i\) 個候選後，堆內是某個數量下總成本最小或總收益最大的可行集合。加入新選項若破壞限制，就刪掉對目標最不利者；刪除後集合回復可行，而且在相同基數下留下最大未來餘裕。證明要同時說清「為何只撤銷一個就夠」與「撤銷它不會破壞其他限制」。

問題型態處理順序堆內元素違規時撤銷
期限前最多做幾件deadline 遞增已選耗時（大根堆）耗時最大者
交易中保留最大利潤時間順序可被替換的買入成本較差舊成本
容量變動下保留最佳集合限制逐步收緊的順序已選代價或收益最拖累可行性的項目
⚠ 不是所有決策都能反悔

若撤銷某項會改變其他項的合法性、順序或收益（例如任務有先修依賴、區間互相覆蓋形成複雜衝突），單一 heap 通常無法修復。此時需要更完整的 DP／流／擬陣結構，而不是硬套「先選再 pop」。

💡 核心想法

普通貪心不敢貪（怕後悔）的地方，反悔貪心先貪了再說，同時把「撤銷這次選擇的代價」丟進堆裡；未來遇到更好的選項時，彈出堆頂完成撤銷。等價於在時間軸上動態維護「目前最優的選擇集合」。

🔎 適用特徵
  • 依時間/期限順序處理，選擇之間互相排擠（容量、期限、次數限制）。
  • 「不選白不選、選錯可退貨」— 撤銷一個已選項不影響其他已選項的可行性。
  • 直接貪心能構造出反例、DP 狀態又太大時，優先想反悔堆。
★★★LC 630 課程表 III反悔貪心堆

第 \(i\) 門課耗時 \(d_i\)、截止時間為 \(e_i\)，從時刻 0 開始連續上課，最多修幾門？

思路

按截止時間遞增處理，維護大根堆（已選課程的耗時）與目前總耗時 \(T\)。每門課先選上（\(T\gets T+d_i\)）；若 \(T>e_i\)，就反悔並彈掉堆中耗時最大的課。正確性：修課數相同時總耗時越小越好，彈最大者使 \(T\) 最小化，為未來保留最大餘裕。

程式碼（C++） 
sort(courses.begin(), courses.end(),
    [](auto& lhs, auto& rhs) { return lhs[1] < rhs[1]; });
priority_queue<int> pq;  // 已選課程的耗時，大根堆
long long total_time = 0;
for (auto& [duration, deadline] : courses) {
    total_time += duration;
    pq.push(duration);
    if (total_time > deadline) {
        total_time -= pq.top();
        pq.pop();
    }  // 反悔：踢掉最耗時的
}
return pq.size();
★★★LC 871 最低加油次數反悔貪心堆

路上有加油站，車開到哪算哪，最少加幾次油到終點？

思路

「路過先不加，把油量存入大根堆；油不夠時再反悔 — 取出堆中最大的油量視為當時加過」。次數相同時油越多越好，故每次補堆頂。這是把「在哪加油」的決策延遲到必要時刻的教科書範例。

程式碼（C++） 
priority_queue<int> pq;
long long fuel = start_fuel;
int cnt = 0, i = 0;
stations.push_back({target, 0});
for (auto& [pos, gas] : stations) {
    while (fuel < pos) {  // 到不了，反悔補油
        if (pq.empty()) {
            return -1;
        }
        fuel += pq.top();
        pq.pop();
        cnt++;
    }
    pq.push(gas);
}
return cnt;
★★★★洛谷 P4053 [JSOI2007] 建築搶修反悔貪心堆

\(n\) 棟建築，修第 \(i\) 棟耗時 \(t_i\)、必須在 \(d_i\) 前修完，一次修一棟，最多修幾棟？（LC 630 的原型競賽題，USACO 2009 Work Scheduling 為「最大化報酬」變體）

思路

同 LC 630 反悔堆。USACO Work Scheduling 變體：按期限處理、堆存報酬最小的已接工作，時間不夠且新工作報酬更高時換掉堆頂 — 反悔的對象從「耗時」變成「報酬」，體會反悔鍵值隨目標函數變化。

★★★★CF 865D Buy Low Sell High反悔貪心堆費用流思想

已知未來 \(n\) 天股價，每天限買一股或賣一股或不動，最大化最終利潤（結束時不持股要求可自動滿足）。

思路

經典反悔交換：每天把 \(p_i\) 推入小根堆；若 \(p_i>\text{堆頂}\)，便將利潤增加 \(p_i-\text{堆頂}\)、彈出堆頂，再把 \(p_i\) 推回去一次。第二次推入的 \(p_i\) 代表「反悔券」：若未來有更高價 \(p_j\) 用它成交，總效果為 \((p_i-p_x)+(p_j-p_i)=p_j-p_x\)，等價於當初直接在 \(x\) 買、\(j\) 賣 — 中間商 \(p_i\) 無縫退出。這正是模擬費用流中「殘量網路反向邊」的化身，是通往 2.11.5 的橋樑。

程式碼（C++） 
priority_queue<long long, vector<long long>, greater<>> pq;
long long profit = 0;
for (int i = 0; i < n; i++) {
    long long price;
    cin >> price;
    if (!pq.empty() && pq.top() < price) {
        profit += price - pq.top();
        pq.pop();
        pq.push(price);  // 反悔券：允許未來把這次交易升級
    }
    pq.push(price);
}

常見誤解與除錯順序

  1. 只憑直覺挑最大或最小，沒有交換論證、領先論證或反證。
  2. 排序鍵選錯，或忘記相同主鍵時的 tie-break 會影響答案。
  3. 只測隨機資料，沒有刻意構造讓局部最佳與全域最佳衝突的小反例。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
