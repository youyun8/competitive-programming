2.11 專家專題 ★★★★★

專題把哪個巨大物件壓縮了先備能力
擬陣貪心所有可行子集壓成「可增廣的獨立集系統」交換論證、線性獨立／圖形擬陣
slope trick整個凸分段線性 DP 函數壓成斜率轉折點凸性、heap、DP 轉移
排序後 DP先用交換論證消掉排列維度相鄰交換、背包／線性 DP
⚠ 專家技巧的名稱不是適用條件

擬陣必須驗證遺傳性與交換公理；slope trick 必須先證明值函數保持凸；排序後 DP 必須證明任一選定子集都存在同一鍵值的最優順序。任一前提不成立，短模板都可能產生看似合理的錯解。建議與 DP 的 2.12、2.13 對讀並保留暴力對拍。

2.11.1 擬陣（Matroid）：貪心正確性的統一理論

擬陣 \(M=(E,\mathcal I)\)：元素全集 \(E\) 與獨立集族 \(\mathcal I\)，滿足：（1）遺傳性：獨立集的子集獨立；（2）交換公理：若 \(|A|<|B|\) 且 \(A,B\in\mathcal I\)，則存在 \(x\in B\setminus A\) 使 \(A\cup\{x\}\in\mathcal I\)。

🧾 Rado–Edmonds 定理

在擬陣上「按權重降序、能加就加」的貪心必得最大權基；反之，若一個獨立系統對所有權重函數貪心都正確，它必是擬陣。— 這給了「貪心何時必然正確」的完整刻畫。

  • 圖形擬陣（無環邊集）→ Kruskal 正確性。
  • 分割擬陣 / 橫貫擬陣 → Job Sequencing（2.9）按報酬降序能排就排的正確性。
  • 擬陣交：兩個擬陣的公共獨立集最大化（如彩色生成樹 \(=\) 圖形擬陣 \(\cap\) 分割擬陣），有多項式演算法（增廣路），但已超出貪心 — 知道邊界在哪本身就是專家素養。CF 話題題：CF 1284G。

2.11.2 Slope Trick：凸函數 DP 的貪心壓縮

★★★★★CF 713C Sonya and Problem Wihtout a Legendslope trick堆

每次操作 \(\pm1\)，把序列改成嚴格遞增，最小化總操作數。

提示

嚴格遞增先做變換 \(a_i\gets a_i-i\) 化為非嚴格。定義 \(f_i(x)\) 為前 \(i\) 項中，第 \(i\) 項改為 \(\le x\) 時的最小代價；它是關於 \(x\) 的凸分段線性函數。

思路

轉移為 \(f_i(x)=\min_{y\le x}f_{i-1}(y)+|a_i-x|\)。凸函數的「前綴最小值」等於砍掉右半上升段；加上 \(|a_i-x|\) 則等於左右各增加一個斜率轉折點。用大根堆存左半段的斜率轉折點集合：每步 push \(a_i\) 兩次、若堆頂 \(>a_i\)，則答案累加 \(\text{top}-a_i\) 並彈出替換。整套操作把 \(O(nV)\) 的 DP 壓成 \(O(n\log n)\) 貪心。

程式碼（C++） 
priority_queue<long long> L;  // 凸函數左半段斜率轉折點
long long ans = 0;
for (int i = 0; i < n; i++) {
    long long x = a[i] - i;  // 嚴格遞增 → 非嚴格
    L.push(x);
    if (L.top() > x) {  // 出現下降段，付出代價抹平
        ans += L.top() - x;
        L.pop();
        L.push(x);
    }
}

進階應用：APIO 2016 烟花表演（樹上 slope trick，用可並堆合併子樹的凸函數）、ABC 217H。辨識特徵：代價為絕對值/分段線性、狀態值域大、DP 關於值域是凸的。

2.11.3 交換論證 × DP 的一般化

2.3 的「Tower」模式推廣：當問題是「選子集 + 定順序」時，先用相鄰交換論證證明存在最優解按某鍵有序，把順序維度消掉，再對排序後的序列 DP。訓練重點是推導鍵值的代數功力：把「\(i\) 在 \(j\) 前可行／更優」的條件化成 \(g(i)\le g(j)\) 的形式。同型：洛谷 P1417 烹調方案（鍵 \(b_ic_j\) 交叉比較）、CF 1203F。

2.11.4 對抗式貪心（博弈中的貪心）

輪流行動的博弈裡，「雙方都貪心拿當前最大」在可加性目標下常是最優（如 CF 談判分糖果：排序後輪流取），證明用策略交換。但存在後手結構（取走影響未來選項）時貪心失效需要博弈 DP — 分辨兩者的試金石：你的選擇會不會改變對手未來選項的「相對」價值。例：LC 1690、CF 1539E。

2.11.5 反悔貪心 ⇄ 模擬費用流

★★★★★洛谷 P6122 [NOI Online #1] 隊長的技能 / CF 802O April Fools' Problem模擬費用流反悔貪心二分

CF 802O：\(n\) 天中選 \(k\) 天準備（費用 \(a_i\)）、\(k\) 天列印（費用 \(b_j\)），準備日 \(\le\) 對應列印日，最小化總費用。

思路

費用流建模顯然但跑不動（\(n\le5\times10^5\)）。模擬費用流：逐天掃描，遇 \(b_j\) 時與「堆中最小的未配對 \(a_i\)」成交；成交後把 \(-a_i+b_j\) 類型的「反悔券」推回堆，允許未來更便宜的 \(a\) 替換掉這次的 \(a\)、或更貴的 \(b\) 被換掉 — 每一張反悔券精確對應殘量網路的一條反向邊。\(k\) 的限制用 wqs 二分（帶懲罰的拉格朗日鬆弛）處理。反悔貪心（2.4）是本技術的單反向邊特例；掌握「彈出成交 \(\Rightarrow\) 推回反悔券」的對應，即可自行推導新題的模擬費用流。
