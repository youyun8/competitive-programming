初學者閱讀指南2.6 字典序貪心與單調堆疊構造

這頁要解決什麼問題？

先問：能不能現在做一個局部選擇，而且保證不會害未來失去最佳答案？

本頁聚焦「逐位確定最小/最大字元，搭配可行性檢查與單調棧實作」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：暴力會枚舉每一步的所有選擇，形成龐大決策樹；貪心想證明其中大多數分支永遠不必探索。

核心直覺

貪心不是『看起來最好就選』，而是證明某個最佳答案可以被交換成包含目前選擇，且不會變差。

閱讀程式前先找不變量

每做一次選擇後，都要說清楚剩餘問題仍保留什麼性質，以及被丟掉的候選人為何不可能更好。

先猜猜看：什麼時候該想到這頁？

排序後逐一決策、要求字典序最小、選最多活動、成本可交換比較，或可撤銷最差選擇。

最小可手算例子

  1. 從 "1432" 刪 1 位求最小；讀到 3 時，前面的 4 比 3 大。
  2. 刪掉 4 得 "132"；若刪其他位置，第一個不同位會更大。
  3. 字典序問題優先保住更小的高位，單調堆疊負責反悔。

2.6 字典序貪心與單調堆疊構造

🔤 前綴支配是正確性的來源

兩個答案第一次不同的位置就決定字典序，所以只要能證明選較好字元後仍可完成，後綴再差也不會翻盤。標準流程是逐位嘗試候選字元（小到大或大到小），用可行性 oracle判斷剩餘資源能否補成完整答案，選第一個可行者。

oracle 類型需要維護典型問題
剩餘出現次數每字元最後位置／suffix count去重字母、最小子序列
剩餘長度還可刪幾個、還需選幾個刪 k 位數
括號可完成性目前 balance 與剩餘格數字典序最小合法括號
圖／匹配可完成性暫選前綴後跑連通或匹配檢查字典序最小生成結構
⚠ 單調棧的 pop 條件有三部分

新元素更優、舊棧頂未來還會出現、而且刪除／選取額度允許，三者缺一不可。只看大小就 pop，常會把唯一一次出現的必要字元刪掉。掃描結束後若仍有刪除額度，還要依題意從尾端刪除並處理前導零或答案長度。

💡 核心想法

字典序目標自帶貪心性質：第一位越小／大，後面全不用比。所以逐位確定「當前位能放的最好字元」，配一個 \(O(1)\) 或 \(O(\log n)\) 的可行性檢查（剩餘資源夠不夠完成約束）。實作上大量以單調堆疊出現：新字元把棧頂「還有機會再來的較差字元」彈掉。

★★LC 402 移掉 K 位數字字典序單調棧

刪 k 位使剩下的數最小。

思路

維護遞增棧：新數字比棧頂小且還有刪除額度時，彈棧頂（高位變小永遠更優 — 這就是字典序的貪心選擇性質）。掃完還有額度就從尾巴刪。

程式碼（C++） 
string st;
for (char digit : num) {
    while (!st.empty() && k && st.back() > digit) {
        st.pop_back();
        k--;
    }
    st.push_back(digit);
}
st.resize(st.size() - k);
st.erase(0, st.find_first_not_of('0'));  // 去前導零
return st.empty() ? "0" : st;
★★★LC 316 去除重複字母字典序單調棧可行性檢查

刪去重複字母使每個字母恰出現一次，且結果字典序最小。

思路

單調棧 + 可行性檢查：彈棧頂的條件除了「棧頂 \(>\) 新字元」，還必須「棧頂字元在後面還會出現」（否則彈了就湊不齊了）。這是字典序貪心的完整形態：貪 + 保證還能完成。用剩餘計數陣列與 in-stack 標記各一個即可。

★★★★LC 321 拼接最大數字典序單調棧歸併

從兩個陣列中共取 k 個數（各自保持相對順序），拼成最大的數。

思路

三層貪心組合：枚舉從第一個陣列取 \(i\) 個 — 各自用單調棧取出「保留 \(i\) 個的最大子序列」（LC 402 的最大版）— 再貪心歸併兩個序列（比較整段字典序決定下一個取誰，注意相等時要往後比，故歸併最壞為 \(O(n^2)\)）。展示「字典序貪心可作為零件組裝」的層次感。

★★★字典序最小拓撲序字典序圖堆
思路

Kahn 演算法中把普通 queue 換成小根堆：每次在所有入度 0 的點中取編號最小者。正確性：第一位可以是任何目前入度 0 的點，取最小者是字典序的貪心選擇性質；取走後子問題同構。⚠ 對比陷阱：「字典序最小拓撲序」\(\ne\)「每個點盡量早出現」— HDU 4857 反向建圖才是後者，辨識題意的細微差異是這型題的坑。

常見誤解與除錯順序

  1. 只憑直覺挑最大或最小，沒有交換論證、領先論證或反證。
  2. 排序鍵選錯，或忘記相同主鍵時的 tie-break 會影響答案。
  3. 只測隨機資料，沒有刻意構造讓局部最佳與全域最佳衝突的小反例。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
