2.1 排序貪心（Sort & Sweep）

🔑 排序鍵不是從樣例猜出來的

先只看相鄰兩個元素 \(i,j\)，比較順序 \(ij\) 與 \(ji\) 對目標的貢獻；把「\(ij\) 不劣」化成單鍵 \(\operatorname{key}(i) \le \operatorname{key}(j)\)，再證明任何逆序都可交換。若關係會依第三個元素或目前累積量改變，就未必能用一次全域排序解決。

實作風險安全做法
用浮點比較比值用 a.x * b.y < b.x * a.y 交叉相乘，並用 128 位防溢位
比較器在相等時仍回 true相等必回 false；需要穩定次序就顯式加 tie-break
只按一鍵排序卻未處理同鍵檢查同鍵是否任意順序等價，否則推導第二鍵
lambda 捕捉會變動的外部狀態比較結果必須固定且滿足 strict weak ordering
排序後掃描的等號寫錯先確認區間是閉、開或半開，手測相切端點
💡 核心想法

找到正確的排序鍵，排序後從頭到尾線性掃描，每一步的決策變得顯然。80% 的貪心題第一步都是排序 — 問題只剩「按什麼排」。

🔎 適用特徵
  • 元素之間的「處理順序」影響答案，但單一元素的貢獻可獨立衡量。
  • 存在一個單調的「性價比」「緊迫度」或「結束時間」可比較。
  • 排序鍵的推導方法：對任兩個相鄰元素寫出「先 A 後 B」與「先 B 後 A」的成本，令前者 \(\le\) 後者，解出比較式 — 這就是相鄰交換論證。

2.1.1 單鍵排序

★部分背包 Fractional Knapsack排序性價比

物品可切割，容量 W，最大化總價值。

思路

按「單位重量價值 \(v/w\)」由大到小排序，依序裝滿，最後一件裝部分。與 0/1 背包的對比是理解貪心邊界的第一課：可分割使得「先拿性價比最高的」永遠不會後悔 — 交換論證中把最優解裡任何一單位重量換成更高性價比的都不變差。0/1 背包不可分割，交換會破壞容量可行性，貪心失效（見第 3 章反例）。

程式碼（C++） 
sort(items.begin(), items.end(), [](auto& a, auto& b) {
    return a.v * b.w > b.v * a.w;  // v/w 大者在前，交叉相乘避免浮點
});
double ans = 0, cap = capacity;
for (auto& [w, v] : items) {
    double take = min(cap, (double)w);
    ans += take * v / w;
    cap -= take;
    if (cap == 0) {
        break;
    }
}
★LC 455 分發餅乾排序雙指針

每個孩子有胃口 \(g_i\)，每塊餅乾大小 \(s_j\)，一塊只能給一人，最多滿足幾個孩子？

思路

兩邊都排序，用最小的能滿足的餅乾去餵胃口最小的孩子（雙指針）。證明（領先論證）：任何最優方案中，把餵給最小孩子的餅乾換成「最小的可行餅乾」，剩餘餅乾集合只會更「寬鬆」，不會變差。

★★POJ 1328 Radar Installation排序區間選點

海島要被 x 軸上的雷達（半徑 d）覆蓋，求最少雷達數。

提示

把「島」轉化成「雷達可以放置的 x 軸區間」— 幾何問題化為區間選點問題。

思路

每個島對應區間 \(\left[x-\sqrt{d^2-y^2},\,x+\sqrt{d^2-y^2}\right]\)。問題變成：最少的點刺穿所有區間。按右端點排序，維護目前雷達位置；若下一個區間左端點 \(>\) 雷達位置，則在其右端點放新雷達。這是 2.2 區間家族的「區間選點」模型。

2.1.2 自訂比較函數（用交換論證推導 comparator）

★★LC 179 最大數排序交換論證

把一組非負整數串接成最大的數。

思路

比較兩個字串 \(a,b\)：若 a + b > b + a（字串串接後字典序）則 \(a\) 排前面。正確性：相鄰交換論證 — 任意相鄰逆序對交換後結果變大，故完全排序後最大。陷阱：必須驗證這個比較關係是嚴格弱序（可證其可傳遞，因 \(a+b>b+a\) \(\Longleftrightarrow\) 把字串視為 \(k\) 進位數後 \(\frac{a}{10^{|a|}-1} > \frac{b}{10^{|b|}-1}\)，是一個實數鍵值），否則 std::sort 是 UB。

程式碼（C++） 
sort(strs.begin(), strs.end(),
    [](const string& a, const string& b) { return a + b > b + a; });
string ans = accumulate(strs.begin(), strs.end(), string());
return ans[0] == '0' ? "0" : ans;  // 全零特判
★★★洛谷 P1080 國王遊戲排序交換論證高精度

\(n\) 位大臣每人左手為 \(a\)、右手為 \(b\)。大臣獲得的金幣等於排在他前面所有人的左手乘積除以自己的右手。安排順序使獲得最多金幣的大臣拿的金幣最少。

提示

對相鄰兩位大臣 \(i,j\)，分別寫出「\(i\) 在前」與「\(j\) 在前」時兩人獎金的最大值，比較之。

思路

相鄰交換論證得排序鍵：按 \(ab\) 遞增排序。設前綴乘積為 \(S\)，\(i\) 在 \(j\) 前時兩人的較大值是 \(\max(S/b_i,\,Sa_i/b_j)\)；交換後為 \(\max(S/b_j,\,Sa_j/b_i)\)。通分比較可得 \(a_i b_i \le a_j b_j\) 時 \(i\) 在前不劣。注意本題需高精度乘除。同型題：洛谷 P1842 奶牛玩雜技（按 \(w+s\) 排序）、UVa 10026 Shoemaker's Problem。

2.1.3 雙指針配對

★★LC 881 救生艇排序雙指針

每艘船限乘 2 人、限重 limit，最少幾艘船載走所有人？

思路

排序後最重配最輕：若 \(\text{最重}+\text{最輕}\le\text{limit}\) 則同船，否則最重者單獨一船。交換論證：最重的人反正要一艘船，讓他帶走「還能帶的人中任意一個」都不會比帶最輕的更好 — 因為最輕的人與任何人都相容，留下他最保值。

程式碼（C++） 
sort(p.begin(), p.end());
int i = 0, j = p.size() - 1, boats = 0;
while (i <= j) {
    if (p[i] + p[j] <= limit) {
        i++;
    }
    j--;
    boats++;
}
★★★POJ 2287 田忌賽馬排序雙指針博弈

田忌與齊王各 \(n\) 匹馬，贏一場得 \(+200\)、輸一場得 \(-200\)、平手得 \(0\)，田忌知道齊王出馬順序（由強到弱），求最大收益。

思路

雙方馬按速度排序後貪心：若田忌目前最快 \(>\) 齊王最快，直接贏；若最慢 \(>\) 齊王最慢，贏下這場；否則用最慢的馬去消耗齊王最快的馬（打不贏就墊）。正確性用交換論證逐 case 驗證，或退而用區間 DP \(O(n^2)\) 保底 — 這題是「貪心可行但 case 多，DP 更穩」的典型例子。
