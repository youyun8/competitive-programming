2.3 交換論證排程（Scheduling by Exchange Argument）

排程目標局部比較得到的規則限制
\(\sum_i C_i\)兩任務誰短先做\(p\) 遞增（SJF）單機、全部同時可用
\(\sum_i w_iC_i\)\(p_iw_j \le p_jw_i\)\(p/w\) 遞增（Smith）\(w>0\)；無 release time
兩機 flow shop makespan比較相鄰工件在 A、B 的等待Johnson：前組 a 遞增、後組 b 遞減每件都先 A 後 B
選子集再排順序先證明選定集合存在固定最優序排序後再 DP／背包排序關係不得依「選了哪些其他項」改變
🧮 Smith's Rule：一步一步比較兩種順序

假設前面的工作都一樣，做完它們已經花了 \(T\)。現在只比較相鄰的工作 \(i,j\)；後面的工作無論交換前後都在同一時間開始，所以也不受影響。

先做 \(i\) 再做 \(j\)：\(i\) 在 \(T+p_i\) 完成，\(j\) 在 \(T+p_i+p_j\) 完成。

\[ \begin{aligned} \operatorname{cost}(i,j) &= w_i(T+p_i)+w_j(T+p_i+p_j) \\ &= (w_i+w_j)T+w_ip_i+w_jp_i+w_jp_j. \end{aligned} \]

先做 \(j\) 再做 \(i\)：

\[ \operatorname{cost}(j,i) = (w_i+w_j)T+w_ip_i+w_ip_j+w_jp_j. \]

兩式都有 \((w_i+w_j)T\)、\(w_ip_i\)、\(w_jp_j\)；這就是「共同項」。相減後只剩：

\[ \operatorname{cost}(i,j)-\operatorname{cost}(j,i) = w_jp_i-w_ip_j. \]

若先做 \(i\) 不會比較差，左邊必須 \(\le 0\)，因此 \(p_iw_j \le p_jw_i\)。當權重皆為正時，兩邊除以 \(w_iw_j\)，便得到 \(p_i/w_i \le p_j/w_j\)：依 \(p/w\) 由小到大排序。

若某一對順序違反這條規則，交換它們不會增加成本；重複交換所有逆序對，就會得到排序後的最優序列。等號表示兩種順序同價。

C++ 比較器：用交叉相乘避免浮點誤差 
struct Job {
    long long processing_time;
    long long weight;
    int original_index;
};

bool SmithOrder(const Job& left, const Job& right) {
    const __int128 left_product =
        static_cast<__int128>(left.processing_time) * right.weight;
    const __int128 right_product =
        static_cast<__int128>(right.processing_time) * left.weight;
    if (left_product != right_product) {
        return left_product < right_product;
    }
    return left.original_index < right.original_index;
}

std::sort(jobs.begin(), jobs.end(), SmithOrder);

__int128 是 GNU C++ 擴充型別，兩邊都先轉型再相乘，避免 long long 乘法先溢位。最後用原輸入位置打破同價平手，只是讓輸出固定，不會改變目標值。

數值例：為什麼「工時短」不一定先做？

任務 \(A=(p=3,w=100)\)、\(B=(p=1,w=1)\)。\(A\) 先的成本為 \(100\cdot3+1\cdot4=304\)；\(B\) 先為 \(1\cdot1+100\cdot4=401\)。雖然 \(B\) 更短，\(A\) 的權重極高，Smith 比值分別為 \(0.03\) 與 \(1\)，會正確地把 \(A\) 放前面。這也說明排序鍵必須從目標函數推導，而不是只憑「短作業優先」直覺。

💡 核心想法

單機/多機排程問題的統一解法：假設最優序列中相鄰兩個任務 i、j 交換，寫出兩種順序的目標值，令「i 在前不劣」解出不等式 — 這個不等式就是排序鍵。

🧾 推導模板

設 \(i\) 在 \(j\) 前、其餘不動。因為只有 \(i,j\) 的完成時刻改變，比較兩種順序下這兩項的目標貢獻即可。得到形如 \(f(i,j) \le f(j,i)\) 的可傳遞關係後，直接 sort。務必檢查關係可傳遞（通常能改寫成單一實數鍵值比較，如 \(p/w\)、\(\min(a,b)\) 分類）。

★★★最小化加權完成時間（Smith's Rule）排程交換論證

單機 \(n\) 個任務，任務 \(i\) 耗時 \(p_i\)、權重 \(w_i\)，最小化 \(\sum_i w_iC_i\)（\(C_i\) 為完成時刻）。對應：洛谷 P1248 加工生產調度前置、AtCoder 各 ABC 排程題。

推導

相鄰交換 \(i,j\)：\(i\) 在前時兩者貢獻差為 \(w_jp_i-w_ip_j\)。令 \(i\) 在前不劣，便有 \(p_i/w_i \le p_j/w_j\)。所以按 \(p/w\) 遞增（性價比：短且重要優先）排序。特例 \(w\) 全等 \(\Rightarrow\) SJF 最短工時優先；\(p\) 全等 \(\Rightarrow\) 權大優先。

每個箱子有重量 \(w\)、承重 \(s\)、價值 \(v\)，疊塔使塔上每箱之上的總重 \(\le\) 其承重，最大化總價值。

提示

價值使純貪心失效（要選子集）— 但「選定子集後的最佳疊放順序」可以先用交換論證定死，再對這個順序做背包 DP。

思路

相鄰交換：\(i\) 疊在 \(j\) 下方可行性更寬 \(\Longleftrightarrow w_i+s_i \ge w_j+s_j\)。按 \(w+s\) 遞增排序（小的在上／先考慮），然後用 \(\mathrm{dp}[\text{承重餘量}]=\text{最大價值}\) 做 0/1 背包。這是「交換論證定順序 + DP 選子集」的代表套路，競賽出現率極高（同型：CF 開會安排、洛谷 P1417 烹調方案）。
