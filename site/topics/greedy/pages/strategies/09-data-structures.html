初學者閱讀指南2.9 貪心 × 資料結構

這頁要解決什麼問題？

先問：能不能現在做一個局部選擇，而且保證不會害未來失去最佳答案？

本頁聚焦「並查集找空位、線段樹/BIT 加速貪心決策」。先在紙上用 3～8 個元素走一次，再對照正文的公式與程式碼。

暴力做法的代價：暴力會枚舉每一步的所有選擇，形成龐大決策樹；貪心想證明其中大多數分支永遠不必探索。

核心直覺

貪心不是『看起來最好就選』，而是證明某個最佳答案可以被交換成包含目前選擇，且不會變差。

閱讀程式前先找不變量

每做一次選擇後，都要說清楚剩餘問題仍保留什麼性質，以及被丟掉的候選人為何不可能更好。

先猜猜看：什麼時候該想到這頁？

排序後逐一決策、要求字典序最小、選最多活動、成本可交換比較，或可撤銷最差選擇。

最小可手算例子

  1. 期限 1、2、2 的工作收益為 5、4、3。
  2. 按收益由大到小，DSU 找不超過期限的最晚空位。
  3. 資料結構只加速『下一個可用位置』，貪心順序仍需獨立證明。

2.9 貪心 × 資料結構

🧰 把「策略」與「查詢器」分開

先在不考慮效率時寫出每步真正想找的對象，再選資料結構加速。資料結構不改變貪心規則，也不提供正確性；證明仍針對「為何應選這個對象」，複雜度才針對「如何快速找到它」。

每步需要的操作常用工具典型用途
找 \(\le x\) 的最晚未占位置並刪除並查集跳點截止期限任務排槽
前綴／區間計數與第 k 個BIT／線段樹判斷資源、選順位
找前驅／後繼並任意刪除set／multiset配對、分配最接近資源
區間批量占用、查下一空位並查集或線段樹離線配置
動態維護最小／最大候選heap掃描線候選池、反悔
⚠ 更新必須維持查詢語意

例如 DSU 跳點的 find(x) 若定義為「\(\le x\) 的最大空位」，占用 \(t\) 後就必須令它指向 find(t - 1)；若只是 fa[t] = t - 1 而 find 沒正確壓縮，語意或效率都可能破壞。先把抽象操作寫成規格，再實作資料結構。

💡 核心想法

貪心策略本身簡單，但每一步的決策需要「快速找到／更新某個極值或空位」— 瓶頸在資料結構：並查集跳過已佔用位置、BIT/線段樹維護區間資訊、multiset 支援任意刪除。

★★★★截止期限任務調度（Job Sequencing）+ 並查集DSU排程

每個任務耗時一單位、截止時間為 \(d_i\)、報酬為 \(p_i\)，最大化總報酬。

思路

按報酬遞減排序，每個任務安排在「\(\le d_i\) 的最晚空閒時槽」（越晚越不擋別人）。找最晚空槽用並查集：find(d) 返回 \(\le d\) 的最晚空槽，佔用後令 fa[t] = t - 1，均攤複雜度接近 \(O(1)\)。正確性可用擬陣描述：「可行任務子集」構成 transversal matroid，按權降序能加就加必最優 — 2.11.1 的具體實例。

程式碼（C++） 
sort(jobs.begin(), jobs.end(),
    [](auto& lhs, auto& rhs) { return lhs.p > rhs.p; });
vector<int> parent(max_d + 1);
iota(parent.begin(), parent.end(), 0);
function<int(int)> find = [&](int slot) {
    return parent[slot] == slot ? slot : parent[slot] = find(parent[slot]);
};
long long ans = 0;
for (auto& job : jobs) {
    int slot = find(min(job.d, max_d));
    if (slot > 0) {
        ans += job.p;
        parent[slot] = slot - 1;
    }  // 佔用時槽 slot
}
★★★★洛谷 P1250 種樹區間差分約束→貪心

區間 \([l_i,r_i]\) 內至少要種 \(t_i\) 棵樹，每個位置至多一棵，求最少總棵數。

思路

按右端點遞增排序，每個約束不足時從右端點往左補種（種得越右，越可能被後面的區間重複利用）。交換論證：把任何解中較左的樹右移到區間右端，不影響當前約束、只會幫助後續。加速：BIT 查區間和 + 並查集找「最右空位」。同型進階：洛谷 P2894、CF 志願者/餵牛系列。

★★★CF 1526C2 Potions（multiset/堆反悔）反悔貪心multiset

從左到右喝藥水（可跳過），生命值不得為負，最多喝幾瓶？

思路

全喝，血量為負時反悔吐掉「已喝中最毒的一瓶」（小根堆取最小負值）。與 LC 630 同構 — 反悔貪心配上不同容器（heap / multiset）反覆出現在 CF Div2 C/D，是性價比最高的訓練題型之一。

常見誤解與除錯順序

  1. 只憑直覺挑最大或最小，沒有交換論證、領先論證或反證。
  2. 排序鍵選錯，或忘記相同主鍵時的 tie-break 會影響答案。
  3. 只測隨機資料，沒有刻意構造讓局部最佳與全域最佳衝突的小反例。

最小測資順序：先測空集合或最短輸入，再測單一元素、全部相同、嚴格遞增／遞減、 重複值與最大邊界；若答案可能累加，另外檢查 32-bit overflow。

複雜度與適用時機

不要只記正文最後的 Big-O。請把它拆成「狀態／節點／候選數量 × 每次轉移或操作成本」， 再代入題目的最大資料範圍。若乘積超過時間預算，就回頭找是否重算、能否排序、離線、 壓縮狀態或使用更合適的資料結構。
