2.5 雙序列 DP

💡 核心想法

dp[i][j] = 「序列 A 的前 i 個」與「序列 B 的前 j 個」的答案。轉移枚舉兩序列末端字元的處置方式（配對 / 各自跳過），永遠只有常數種，複雜度 O(nm)。

🔎 適用特徵
  • 兩條字串/序列之間的比較、對齊、匹配、轉換。
  • 關鍵不變量：處理完前綴 (i, j) 後，剩下的問題只跟後綴有關。
★★LC 1143 最長公共子序列（LCS）雙序列

兩字串的最長公共子序列長度。

思路

a[i]==b[j] → dp[i−1][j−1]+1；否則 max(dp[i−1][j], dp[i][j−1])。雙序列 DP 的原型。還原方案：從 dp[n][m] 倒走，看每步是哪個轉移貢獻的。

程式碼（C++） 
for (int i = 1; i <= n; i++)
    for (int j = 1; j <= m; j++)
        dp[i][j] = (a[i-1] == b[j-1]) ? dp[i-1][j-1] + 1
                                       : max(dp[i-1][j], dp[i][j-1]);
★★★洛谷 P1439 排列 LCS雙序列問題轉化

兩個 1..n 的排列求 LCS，n ≤ 10⁵——O(nm) 過不了。

思路

排列的特殊性：元素兩兩相異。把 B 的每個元素改寫成「它在 A 中的位置」，LCS 就變成 LIS（公共子序列 ⟺ 在兩邊都遞增的位置序列），套 O(n log n)。「利用輸入的特殊結構把模型換掉」是 ★★★ 之後的常態。

★★★LC 72 編輯距離雙序列

插入/刪除/替換，把 A 變成 B 的最少操作數。

思路

三種操作對應三個轉移：dp[i][j] = min(dp[i−1][j]+1（刪）, dp[i][j−1]+1（插）, dp[i−1][j−1]+[a≠b]（替換/免費）)。邊界 dp[i][0]=i、dp[0][j]=j。工業界（diff、拼寫校正）與競賽通吃的經典。

★★★LC 115 不同的子序列雙序列計數

A 的子序列中等於 B 的個數。

思路

計數版雙序列：dp[i][j] = A 前 i 個裡湊出 B 前 j 個的方案數 = dp[i−1][j]（不用 a[i]）+ [a[i]==b[j]]·dp[i−1][j−1]（用）。計數 DP 的鐵律：每個方案恰好被一條轉移路徑數到一次——這裡按「a[i] 用不用」分類，天然不重不漏。

★★★LC 44 萬用字元匹配雙序列匹配

模式含 ?（單字元）與 *（任意長度串），判斷全串匹配。

思路

dp[i][j] = s 前 i 個能否匹配 p 前 j 個。* 的轉移：dp[i][j] = dp[i][j−1]（* 匹配空）|| dp[i−1][j]（* 多吞一個字元）。難度再上一級的 LC 10 正則版：x* 是整體單位，轉移分「匹配零次」dp[i][j−2] 與「再吞一個」dp[i−1][j]（需 s[i] 與 x 相容）。

程式碼（C++） 
dp[0][0] = true;
for (int j = 1; j <= m; j++) dp[0][j] = dp[0][j-1] && p[j-1] == '*';
for (int i = 1; i <= n; i++)
    for (int j = 1; j <= m; j++)
        dp[i][j] = (p[j-1] == '*') ? dp[i][j-1] || dp[i-1][j]
                 : dp[i-1][j-1] && (p[j-1] == '?' || p[j-1] == s[i-1]);
★★★LC 97 交錯字串雙序列

C 能否由 A、B 保序交錯而成？

思路

狀態設計練習：C 的前 i+j 個由 A 前 i 個與 B 前 j 個組成——第三條序列的下標是導出量，不用進狀態。dp[i][j] = (dp[i−1][j] && a[i]==c[i+j]) || (dp[i][j−1] && b[j]==c[i+j])。貪心（誰匹配就先吃誰）在這題有反例，值得親手構造。
