3常見錯誤與除錯（假 DP 教材）

DP 的 bug 高度模式化。這一章按「發生頻率 × 殺傷力」排序，每個坑附最小反例與修法。

3.1 後效性翻車：狀態漏了資訊

⚠ 症狀

方程看起來很順，小測資就 WA。反例：網格圖四方向移動求最小路徑和，寫 dp[i][j] = min(上, 左) + a——但最優路徑可能繞回頭（往上、往左走），dp[i][j] 依賴 dp[i+1][j]，依賴成環。修法：依賴成環就不是 DP 填表，是最短路（Dijkstra/SPFA）；或把缺的資訊（方向、已用步數）加進狀態把環拆掉。判斷口訣：畫出依賴箭頭，有環就重設狀態。

3.2 背包枚舉方向寫反

⚠ 症狀

0/1 背包容量正序枚舉 → 同一件物品被重複選，答案偏大（悄悄變成完全背包）；完全背包倒序 → 每件只選一次，答案偏小。兩份程式只差一個 for 的方向，卻是兩個不同的問題——見 2.3。多測時背包陣列忘記清空是同級高頻坑。

3.3 「恰好裝滿」初始化錯誤

⚠ 症狀

求「恰好裝滿」時把 dp 全初始化成 0 → 不可達容量被當成「價值 0 的合法方案」參與轉移，答案錯。修法：dp[0] = 0，其餘 = −INF（計數版：dp[0] = 1，其餘 0）。−INF 要取夠大但不溢位的值（0x3f3f3f3f），判不可達用 dp[x] < -INF/2——轉移中 −INF 可能被加減過。

3.4 記憶化搜尋的狀態不完整

⚠ 症狀

dfs 的返回值還依賴某個沒進參數的全域量（目前路徑、剩餘次數），cache 卻只按參數存 → 第二次命中時抄了錯的答案。修法：返回值的所有依賴都必須是參數（= 狀態）；「初始值撞答案值」（用 0 當「沒算過」但答案本身可為 0）用獨立的 vis 陣列解決。

3.5 該 DP 卻用了貪心（或反之）

⚠ 症狀

帶權區間排程「按結束時間能選就選」→ 反例：一個大權區間 vs 三個小權區間。0/1 背包按性價比排序 → 反例 {1,3,4} 湊 6 型。修法：貪心需要交換論證，寫不出來就老實 DP；反方向，可以貪心的題硬上 DP 通常只是慢，不會錯——不確定時 DP 是安全側。姊妹站貪心指南第 3 章有完整的反例清單。

3.6 工程坑清單

  • 爆 int：計數取模前的乘法、路徑和上限 n·maxA，先估上界再定型別。
  • 取模出負數：退背包、容斥的減法後 (x % MOD + MOD) % MOD。
  • 多測未清空：memo、vis、鄰接表——清空範圍只到「本測資用到的大小」，避免 O(T·MAXN)。
  • 浮點期望比較：期望 DP 輸出精度用 printf("%.10f")，中途別過早四捨五入。
  • 遞迴爆棧：樹退化成鏈 + 記憶化搜尋 → 手動開棧或改遞推；CF 上 10⁶ 深度必炸。

3.7 除錯流程（照做就好）

  1. 印出 DP 表：小輸入下把整張表印出來，跟手算對——九成的錯在第一排/第一列就看得到。
  2. 寫暴力對拍：暴力遞迴（不記憶化）+ 隨機小測資，逐一比對（模板見 4.4）。找到最小反例後手動追蹤那條轉移。
  3. 檢查三件事：初始化語意（0 還是 −INF）、枚舉順序（依賴的都算完了嗎）、答案位置（要不要對整排取 max）。
