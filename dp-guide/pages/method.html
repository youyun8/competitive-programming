4狀態設計實戰方法論

4.1 從暴力遞迴到 DP 的機械化流程

不會做的 DP 題，永遠從暴力遞迴開始——狀態設計不是靈感，是重構：

  1. 寫暴力：定義 solve(完整描述目前局面) 回傳答案，枚舉當下所有決策遞迴。
  2. 削參數：檢查每個參數——未來用不到的刪掉；能由其他參數導出的刪掉（交錯字串裡 C 的下標 = i + j）。
  3. 找重疊：剩下的參數組合會不會重複出現？會 → 加 memo，DP 完成。
  4. 需要時改遞推：要滾動陣列或轉移優化時，把記憶化搜尋翻成 for 迴圈。
💡 示範一：打家劫舍

暴力：solve(i, 上一家偷了嗎)。削參數：「上一家偷了嗎」只影響「這一家能不能偷」，可以併進轉移（dp[i−2] + a[i] vs dp[i−1]）——參數剩一個 i，重疊顯然，完工。

💡 示範二：戳氣球

暴力：solve(剩餘氣球集合)——指數狀態，卡住。換枚舉對象：不枚舉「第一個戳誰」而枚舉「最後一個戳誰」，剩餘集合永遠是連續區間 → 參數變 (l, r)，狀態 O(n²)。卡在狀態爆炸時，先怀疑「枚舉的方向」而不是硬壓縮。

4.2 狀態設計四視角

視角觸發訊號代表
前 i 個（前綴）決策從左到右、答案只問總體背包、股票、編輯距離
以 i 結尾轉移需要知道「最後一個是誰/在哪」LIS、Kadane、值域 DP
區間 [l, r]操作發生在相鄰元素之間、子區間獨立石子合併、戳氣球、博弈取數
集合 Sn ≤ 20、順序任意、每個元素用一次TSP、配對計數、集合覆蓋

再各加一組常用「附加維」：處理到第幾行＋輪廓（棋盤）、目前節點＋子樹已合併（樹）、匹配到模式串第幾位（自動機）、用了幾次特權（分層）。

4.3 降維與資訊壓縮技巧清單

  • 滾動陣列：只依賴上一排 → 兩排交替；0/1 背包型 → 原地倒序。
  • 導出量不進狀態：能由其他維算出的（i+j = 常數、j = k − i）刪掉一維。
  • 相對量代替絕對量：排列計數記「相對名次」（AT DP T）；差值代替兩個絕對位置。
  • 對稱性合併：等價元素只記個數不記身分（Sushi 的 dp[a][b][c]）。
  • 值域與下標互換：價值範圍小、重量範圍大 → dp[價值] = 最小重量（經典逆轉）。
  • 模一個共同量：整除類條件只需 mod lcm（CF 55D 的 2520）。
  • 凸性/單調性外包：整個函數當 DP 值，用堆/平衡樹維護（slope trick）。

4.4 對拍驗證器（Python）

DP 最幸福的一點：暴力遞迴天然就是正確性基準。方程一寫完就對拍，別等大測資 WA。

import random

def brute(a):                       # 暴力：不記憶化的遞迴 = 規格書
    n = len(a)
    def go(i):
        if i < 0: return 0
        return max(go(i - 1), go(i - 2) + a[i])
    return go(n - 1)

def fast(a):                        # 待驗證的 DP
    p2 = p1 = 0
    for x in a:
        p2, p1 = p1, max(p1, p2 + x)
    return p1

for t in range(100000):
    a = [random.randint(-5, 20) for _ in range(random.randint(0, 8))]
    if brute(a) != fast(a):
        print("WA!", a, brute(a), fast(a)); break
else:
    print("100000 tests passed")

要點：測資小而刁（長度 0–8、含負數/零/重複）比大測資更會抓 bug；找到反例後把它縮到最小再除錯。
