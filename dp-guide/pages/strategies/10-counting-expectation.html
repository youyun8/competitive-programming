2.10 計數 DP 與期望 DP

💡 核心想法

計數：max 換成加法，鐵律是「每個方案恰好被數一次」——狀態設計要讓不同方案走不同的轉移路徑。期望：利用線性性 E[X+Y] = E[X] + E[Y] 拆解；「到終點的期望步數」類問題通常逆推（從終點往回），因為起點的期望才是問的東西、終點的期望是 0。

🔎 適用特徵
  • 「方案數 mod 998244353 / 10⁹+7」——計數；「期望值」「機率」——期望 DP。
  • 期望題若過程有隨機轉移（擲骰、隨機選），把「局面」設為狀態、期望值設為 dp 值。
★★LC 96 不同的二叉搜尋樹（卡特蘭數）計數

n 個節點能構成幾種 BST？

思路

枚舉根 = 枚舉「左子樹有幾個節點」：dp[n] = Σ dp[i]·dp[n−1−i]——卡特蘭數的遞推。乘法原理對應「獨立部分的方案數相乘」，這種「枚舉第一層結構 + 兩側獨立」的形狀在計數 DP 反覆出現（合法括號、三角剖分、出棧序列都是它）。

★★★洛谷 P1025 數的劃分計數劃分數

把 n 分成 k 個正整數之和（不計順序）的方案數。

思路

不計順序 → 強制份數非遞減來去重。兩種等價狀態：dp[n][k] = dp[n−1][k−1]（有一份是 1）+ dp[n−k][k]（每份都 ≥ 2，全體減 1）。「不計順序的計數 = 給方案定一個規範順序再數」是通用去重心法。

★★★★AtCoder DP T - Permutation計數前綴和優化

給定 </> 序列，數滿足大小關係的 1..n 排列數。

思路

妙狀態：dp[i][j] = 填好前 i 個數、且第 i 個數是「目前已用數字中第 j 小」的方案數——相對名次代替具體數值，未來只關心相對關係（無後效性）。轉移是對 j 的前綴/後綴和，O(n²)。這題把「狀態記相對量」與「前綴和優化轉移」（2.12.1）一次教完。

★★★洛谷 P4316 綠豆蛙的歸宿期望DAG

DAG 上每步等機率選一條出邊，求起點到終點的期望路徑長。

思路

逆推模板：E[u] = Σ (E[v] + w(u,v)) / outdeg(u)，E[終點] = 0，按反拓撲序遞推。為什麼逆推？「從 u 到終點的期望」與怎麼到 u 的無關（無後效性）；順推的「從起點到 u 的期望」還得除以到達機率，麻煩得多。

程式碼（C++） 
// 反圖拓撲序遞推
for (int u : topo_reverse) {
    if (u == T) { E[u] = 0; continue; }
    double s = 0;
    for (auto [v, w] : g[u]) s += E[v] + w;
    E[u] = s / g[u].size();
}
★★★AtCoder DP J - Sushi期望對稱性壓縮

n 個盤子各有 1–3 個壽司，每次隨機選盤、有就吃一個，期望幾次吃完？

思路

狀態不是 4ⁿ 個局面——盤子可交換，只有「剩 1/2/3 個的盤子各幾個」重要：dp[a][b][c]，O(n³)。轉移含「選到空盤」的自環，移項解出 dp。對稱性壓縮 + 自環移項，期望 DP 兩大手筋一次到位。

★★★洛谷 P1365 WJMZBMR 打 OSU!期望線性性

序列 o/x/?（? 各半機率），連續 o 段貢獻長度平方，求期望得分。

思路

平方不能直接線性拆 → 多維護一個輔助期望：len = 目前結尾連續 o 長度的期望。x² 展開 (len+1)² = len² + 2len + 1，得分增量的期望 = 2·E[len] + 1。「維護低次矩來推高次貢獻」是期望 DP 處理平方/立方得分的通法。

★★★★洛谷 P1850 換教室（NOIP 2016）期望決策維

n 節課可申請換教室（成功率 p_i），最多申請 m 次，最小化走動距離期望。

思路

先 Floyd 求全對最短路。dp[i][j][0/1] = 前 i 節課、用了 j 次申請、第 i 節是否申請 的最小期望。轉移枚舉上一節申請與否，期望 = 四種（成功×成功…）組合按機率加權。把「決策次數」與「上一步決策」都放進狀態，是期望 × 最優化的標準融合。

★★★★洛谷 P4550 收集郵票期望二次期望

n 種郵票等機率抽，第 k 次購買花費 k 元，求集滿的期望花費。

思路

費用跟「已買次數」相關 → 同時維護 f[i] = 已有 i 種時集滿的期望次數、g[i] = 期望花費。g 的轉移里會用到 f（未來每多等一次，後續所有次數的單價整體 +1）。「期望的期望」：當代價是次數的函數，就得帶著低階期望一起遞推。
