2.12 DP 優化技術（競賽分水嶺）

💡 核心想法

方程對了但 TLE，瓶頸幾乎都在「枚舉決策點」。優化的本質只有一句話：把對決策的線性枚舉，換成資料結構查詢或數學性質的直接定位。先把轉移整理成 dp[i] = min/max over j of { 只與 j 有關 } + { 只與 i 有關 } +（也許）{ i·j 交叉項 }，再對號入座：

轉移形態工具複雜度變化
決策 j 在滑動窗口內、無交叉項單調佇列（2.12.2）O(n²) → O(n)
有 i·j 交叉項（形如 −2·s_i·s_j）斜率優化 / Li Chao（2.12.3）O(n²) → O(n)／O(n log n)
決策點隨 i 單調（區間合併類）Knuth / 決策單調性（2.12.4–5）O(n³) → O(n²)；O(kn²) → O(kn log n)
「恰好選 k 個」＋答案關於 k 凸wqs 二分（2.12.6）去掉一整維
轉移是固定線性遞推、n 超大矩陣快速冪（2.12.7）O(n) → O(k³ log n)
決策 j 需滿足偏序（值域約束）BIT/線段樹（2.12.8）O(n²) → O(n log n)
bool 可行性 / 集合可達性bitset（2.12.9）除以 64

2.12.1 前綴和優化

★★★AtCoder DP M - Candies前綴和優化計數

n 個小孩分 K 顆糖，第 i 個最多拿 a_i 顆，方案數。

思路

dp[i][j] = 前 i 人分掉 j 顆 = Σ dp[i−1][j−t]（0 ≤ t ≤ a_i）——轉移是一段連續區間的和，維護上一排的前綴和即可 O(1) 轉移，O(nK) 總複雜度。最便宜也最常用的優化，永遠先想它。

2.12.2 單調佇列優化

★★★洛谷 P1725 琪露諾單調佇列

從格子 i 只能跳到 [i+L, i+R]，最大化途經權值和。

思路

dp[i] = max(dp[j]) + a[i]，j ∈ [i−R, i−L]——決策窗口隨 i 平移，單調遞減佇列維護窗口最大值：隊尾比新值差就彈掉（它永遠不會再當最大），隊首過期就出隊。攤還 O(1)。

程式碼（C++） 
deque<int> q;                       // 存下標，dp 值遞減
for (int i = L; i <= n; i++) {
    int j = i - L;                   // j 進入窗口
    while (!q.empty() && dp[q.back()] <= dp[j]) q.pop_back();
    q.push_back(j);
    while (q.front() < i - R) q.pop_front();
    dp[i] = dp[q.front()] + a[i];
}
★★★洛谷 P3572 [POI2014] Little Bird單調佇列

小鳥每次前跳 ≤ k 格，跳到不低於目前的樹會累 1 點，最小化疲勞。

思路

窗口 min 版，加了「同疲勞值時比樹高」的次級比較——單調佇列的比較器可以是多鍵的。多重背包的單調佇列優化（按余數分組後就是窗口 max）同屬此類，寶物篩選（P1776）可用它壓到 O(nW)。

2.12.3 斜率優化（Convex Hull Trick）

★★★★洛谷 P3195 玩具裝箱（HNOI2008）斜率優化

序列切段，每段代價 (len + c)²，最小化總代價。

思路

設 f_i = dp[i]、x_j = s_j（前綴函數），展開得 f_i = min_j { −2·A_i·x_j + (x_j² + f_j) } + 只含 i 的項——把每個決策 j 看成直線 y = −2x_j·t + (x_j²+f_j)，查詢點 t = A_i。A_i 與 x_j 都單調時：維護下凸殼，隊首兩線比較即可彈出過期最優，攤還 O(n)。推導三步：展開 → 分離 i/j → 交叉項當斜率。不單調時改用 Li Chao 線段樹或 CDQ 分治插線，O(n log n)。

程式碼（C++） 
// 維護下凸殼（斜率遞增），X(j), Y(j) 為決策 j 的座標
auto slope = [&](int a, int b) {
    return (double)(Y(b) - Y(a)) / (X(b) - X(a));
};
int h = 0, t = 0; q[t++] = 0;
for (int i = 1; i <= n; i++) {
    while (t - h >= 2 && slope(q[h], q[h+1]) <= K(i)) h++;   // 查詢斜率遞增：隊首彈出
    dp[i] = calc(i, q[h]);
    while (t - h >= 2 && slope(q[t-2], q[t-1]) >= slope(q[t-1], i)) t--; // 維護凸性
    q[t++] = i;
}
★★★★洛谷 P3628 特別行動隊（APIO 2010）斜率優化

切段代價是二次函數 a·x² + b·x + c（a < 0），最大化總和。

思路

玩具裝箱的孿生題，但求 max → 維護上凸殼、不等號全反。做完這兩題，斜率優化的「推式子 → 定凸殼方向 → 定隊首/隊尾操作」流程就固化了。

2.12.4 四邊形不等式（Knuth 優化）

★★★★POJ 1160 Post Office（IOI 2000）Knuth區間

直線上 n 個村莊建 m 個郵局，最小化總距離。

思路

dp[i][j] = 前 i 村建 j 局。代價 w(l,r)（區間到中位數的距離和）滿足四邊形不等式 w(a,c)+w(b,d) ≤ w(a,d)+w(b,c) → 決策點單調 opt[i][j−1] ≤ opt[i][j] ≤ opt[i+1][j]，枚舉範圍夾住後總量攤還 O(n²)。石子合併同理從 O(n³) 降到 O(n²)。實戰驗證法：先寫暴力打出 opt 表，肉眼確認單調再上優化。

2.12.5 分治優化（Divide & Conquer Optimization）

★★★★★CF 321E Ciel and Gondolas分治優化

n 人依序分成 k 組，組內兩兩「不熟度」和最小化（n ≤ 4000, k ≤ 800）。

思路

層轉移 dp[j][i] = min dp[j−1][t] + w(t+1, i)，決策單調但不方便夾邊界時用分治：solve(l, r, optl, optr) 對中點 mid 暴力找最優決策 opt(mid)，再遞迴兩半並收窄決策範圍——每層 O(n·(範圍)) 總計 O(kn log n)。w 用 2D 前綴和 O(1) 查。比 Knuth 更好寫、適用面更廣，是 CF 難題的常客。

2.12.6 wqs 二分（Aliens Trick）

★★★★LC 188 重做：恰好 k 筆交易wqs 二分

把「至多/恰好 k 筆」的次數維整個消掉。

思路

答案 g(k) 關於 k 是凸函數時：給每筆交易加罰款 λ，解無次數限制的問題（O(n) 狀態機），最優交易數隨 λ 增大而減少 → 二分 λ 使交易數恰為 k，答案 = 帶罰款最優值 + k·λ。幾何直觀：用斜率 λ 的直線去切凸函數。難點在凸性證明與「相同值取交易數最多/最少」的邊界處理。

★★★★★洛谷 P4383 林克卡特樹（八省聯考 2018）wqs 二分樹上

樹上刪 k 條邊再連 k 條 0 權邊，最大化直徑——等價於選 k+1 條點不相交路徑權和最大。

思路

「選恰好 k+1 條路徑」凸 → wqs 二分罰款，內層變成無約束的樹形 DP（每點三態：不在路徑/是端點/在中間）。wqs × 樹形 DP 是 NOI 級標配組合；IOI 2016 Aliens 是同名出處。

2.12.7 矩陣快速冪加速線性遞推

★★★洛谷 P1962 斐波那契數列（n ≤ 2⁶³）矩陣冪

求 F(n) mod 10⁹+7，n 大到不能遞推。

思路

轉移與 i 無關的線性遞推 → 寫成向量 × 固定矩陣，快速冪 O(k³ log n)。判別式：dp[i] 是 dp[i−1..i−k] 的固定線性組合（常數 ≤ 幾十）。GT 考試（2.6.4）= KMP 自動機 + 本技術。騎士撥號器 n 變大也是它。

2.12.8 資料結構優化轉移

★★★CF 597C SubsequencesBIT 優化計數

數長度 k+1 的遞增子序列個數（n ≤ 10⁵, k ≤ 10）。

思路

dp[i][t] = 以 i 結尾長 t 的方案數 = Σ dp[j][t−1]（j < i 且 a[j] < a[i]）——「下標在前、值更小」的偏序求和，對每層 t 開一個 BIT 按值域求前綴和，O(nk log n)。LIS 的 O(n log n) BIT 寫法（前綴 max）是同一招的最值版。

2.12.9 bitset 壓位

💡 模式

可行性背包 dp |= dp << w：一行完成一件物品的全部轉移，O(nW/64)。字串類（LCS 的 bitset 演算法）、傳遞閉包、可達性 DP 同理。當 dp 值只有 0/1 且轉移是平移/或/與，就想 bitset——它經常把「明顯超時」的 O(n²) 直接壓進時限。

bitset<100001> can; can[0] = 1;
for (int w : items) can |= can << w;    // 可行性 0/1 背包
